## Overview

The Loop block enables you to:

<Steps>
  <Step>
    <strong>Iterate over collections</strong>: Process arrays or objects one item at a time
  </Step>
  <Step>
    <strong>Repeat operations</strong>: Execute blocks a fixed number of times
  </Step>
  <Step>
    <strong>Build progressive workflows</strong>: Refine outputs through multiple iterations
  </Step>
  <Step>
    <strong>Handle batch processing</strong>: Process large datasets efficiently
  </Step>

## Configuration Options

### Loop Type

Choose between two types of loops:

<Tabs items={['For Loop', 'ForEach Loop']}>
  <Tab>
    A numeric loop that executes a fixed number of times. Use this when you need to repeat an operation a specific number of times.
    
    ```
    Example: Run 5 times
    - Iteration 1
    - Iteration 2
    - Iteration 3
    - Iteration 4
    - Iteration 5
    ```
  </Tab>
  <Tab>
    A collection-based loop that iterates over each item in an array or object. Use this when you need to process a collection of items.
    
    ```
    Example: Process ["apple", "banana", "orange"]
    - Iteration 1: Process "apple"
    - Iteration 2: Process "banana"
    - Iteration 3: Process "orange"
    ```
  </Tab>
</Tabs>

### Configuration Parameters

- **Iterations** (For Loop): Number of times to execute the loop (1-50)
- **Collection** (ForEach Loop): Array or object to iterate over

## How to Use Loops

### Creating a Loop

1. Drag a Loop block from the toolbar onto your canvas
2. Configure the loop type and parameters
3. Drag other blocks inside the loop container
4. Connect the blocks as needed

### Accessing Loop Data

Inside a loop, you can access special variables:

- **`<loop.currentItem>`**: The current item being processed (ForEach loops)
- **`<loop.index>`**: The current iteration number (0-based)
- **`<loop.items>`**: The full collection being iterated (ForEach loops)

## Advanced Features

### Nested Loops

You can place loops inside other loops for multi-dimensional iteration:

```
Outer Loop (countries)
  └── Inner Loop (cities in each country)
      └── Process city data
```

<Callout type="warning">
  Be cautious with nested loops as they can significantly increase execution time. A loop of 10 items containing another loop of 10 items will execute 100 times total.
</Callout>

### Dynamic Collections

The collection for a ForEach loop can be dynamically generated:

- From an API response: `<api.response.data>`
- From a function output: `<function.results>`
- From a parsed file: `<file.content.items>`

### Loop Control

Loops automatically handle:
- **Iteration management**: Tracks current position and manages state
- **Variable scoping**: Each iteration has access to its own item
- **Error handling**: Continues to next iteration if one fails
- **Performance**: Executes iterations sequentially to maintain order

## Best Practices

- **Set reasonable limits**: Keep iteration counts reasonable to avoid long execution times
- **Use ForEach for collections**: When processing arrays or objects, use ForEach instead of For loops
- **Access current item**: Use `<loop.currentItem>` to access the current item in ForEach loops
- **Track progress**: Use `<loop.index>` to track which iteration you're on
- **Handle errors gracefully**: Consider adding error handling inside loops for robust workflows

<Callout type="info">
  Loops execute sequentially, not in parallel. If you need concurrent execution, use the Parallel block instead.
</Callout>

## Tips and Tricks

- **Collect results**: Use a Function block after the loop to aggregate results from all iterations
- **Early termination**: Use a Condition block with a Router to exit the loop early if needed
- **Progress tracking**: Connect to external systems to report loop progress for long-running operations
- **Memory management**: Be mindful of memory usage when processing large collections
