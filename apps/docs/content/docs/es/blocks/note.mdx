El bloque Nota añade anotaciones contextuales directamente en tu lienzo de flujo de trabajo. Úsalo para documentar decisiones, compartir instrucciones o dejar contexto colaborativo sin afectar la ejecución del flujo de trabajo.

<Callout type="info">
  Los bloques Nota son puramente para documentación y colaboración—no se ejecutan durante las ejecuciones de flujo de trabajo y no afectan la lógica o salida del flujo de trabajo.
</Callout>

## Opciones de configuración

### Contenido

El contenido de la nota que se muestra en el lienzo. Las notas soportan renderización completa de Markdown, incluyendo:

- **Formato de texto**: Negrita, cursiva, enlaces, código en línea
- **Listas**: Puntos y listas numeradas
- **Bloques de código**: Ejemplos de código con sintaxis resaltada
- **Encabezados**: Múltiples niveles de encabezados para organización
- **Incrustaciones de YouTube**: Incrustar videos directamente usando sintaxis estándar de incrustación
- **Referencias de bloques**: Vincular a otros bloques en el flujo de trabajo usando nombres de bloques

### Ejemplo de Markdown

```markdown
## Estrategia de procesamiento

Este flujo de trabajo maneja tickets de soporte al cliente usando:
1. Enrutamiento por intención (FAQ vs. escalación)
2. Generar respuesta con Agente
3. Registrar en base de datos para análisis

**Importante**: Siempre incluye el ID del ticket en la respuesta.

[Más información](https://docs.example.com)
```

## Casos de uso

**Documentación de decisión de flujo de trabajo**
```
Añade una Nota explicando por qué se utiliza la lógica de enrutamiento específica, reglas comerciales o puntos de decisión que otros miembros del equipo deben entender
```

**Instrucciones para colaboradores**
```
Deja notas con instrucciones de configuración, requisitos de configuración o requisitos previos antes de ciertos bloques
```

**Recursos de aprendizaje integrados**
```
Incluye enlaces a documentación, tutoriales de YouTube o ejemplos que ayuden a nuevos miembros del equipo a entender el flujo de trabajo
```

## Mejores prácticas

- **Mantén las notas concisas**: Usa notas para complementar, no reemplazar, un diseño claro de flujo de trabajo. Si estás escribiendo documentación extensiva, considera recursos externos
- **Actualiza con cambios de flujo de trabajo**: Cuando modifiques la lógica del flujo de trabajo, actualiza las notas relacionadas para reflejar los cambios
- **Usa para comunicación de equipo**: Las notas son visibles para todos los colaboradores—úsalas para explicar decisiones de diseño no obvias
- **Vincula a recursos**: Incluye enlaces a documentación externa, FAQs o registros de decisiones para más contexto
