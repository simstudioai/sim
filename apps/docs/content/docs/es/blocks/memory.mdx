El bloque Memoria almacena y recupera la memoria de conversación para agentes de IA. Te permite añadir, recuperar, actualizar y eliminar entradas del historial de conversación, permitiendo interacciones multi-turno con estado más allá de los modos de memoria incorporados del bloque Agente.

<Callout type="warning">
  No uses el bloque Memoria a menos que sea explícitamente necesario. Para conversaciones naturales, usa los modos de memoria incorporados del bloque Agente (Conversación, Ventana deslizante por mensajes o Ventana deslizante por tokens). El bloque Memoria está destinado a inyectar memoria artificial en flujos de trabajo cuando el manejo nativo de memoria del Agente es insuficiente.
</Callout>

## Opciones de configuración

### Operación

Selecciona la operación de memoria a realizar:

- **Añadir memoria**: Añade un nuevo mensaje al historial de memoria de conversación
- **Obtener memoria**: Recupera una entrada de memoria específica por su ID
- **Obtener todas las memorias**: Recupera todas las memorias de una conversación
- **Eliminar memoria**: Elimina una entrada de memoria del almacenamiento

### ID de conversación

El identificador único de la conversación. Esta clave persiste la memoria entre ejecuciones de flujo de trabajo y permite compartir contexto entre múltiples ejecuciones.

- **Requerido**: Sí
- **Tipo**: Cadena de texto
- **Ejemplo**: `chat-session-123`, `user-456`, `conversation-abc`

### Rol (solo Añadir memoria)

Especifica el rol del hablante para la entrada de memoria cuando se utiliza la operación Añadir memoria:

- **user**: Mensaje del usuario final
- **assistant**: Mensaje del modelo de IA
- **system**: Instrucción o contexto a nivel de sistema

### Contenido (solo Añadir memoria)

El contenido del mensaje para almacenar cuando se utiliza la operación Añadir memoria:

- **Tipo**: Cadena de texto (Markdown soportado)
- **Longitud máxima**: Sin límite estricto, pero mantén un tamaño razonable para el rendimiento

## Salidas

- **`<memory.memories>`**: Array JSON de todas las memorias en la conversación (para Obtener todas las memorias)
- **`<memory.memory>`**: La entrada de memoria específica recuperada (para Obtener memoria)
- **`<memory.success>`**: Booleano indicando si la operación se completó exitosamente
- **`<memory.id>`**: El ID de conversación usado para la operación

## Casos de uso de ejemplo

**Conversación multi-turno con preservación de contexto**
```
Entrada del usuario → Memoria (Obtener todo) → Agente (con historial) → Memoria (Añadir mensaje usuario) → Agente → Memoria (Añadir mensaje asistente) → Respuesta
```

**Construcción de perfiles de usuario persistentes**
```
API (Datos del usuario) → Memoria (Añadir) → Agente (Mejorado con contexto del usuario) → API (Actualizar perfil)
```

**Análisis de conversación**
```
Disparador (Programado) → Memoria (Obtener todo, conversationId) → Función (Analizar sentimiento) → API (Registrar análisis)
```

## Mejores prácticas

- **Usa primero la memoria del Agente**: Los modos de memoria incorporados del bloque Agente manejan la mayoría de escenarios de conversación sin requerir el bloque Memoria
- **Gestiona el crecimiento de memoria**: Limpia periódicamente memorias antiguas para evitar almacenamiento excesivo y recuperación lenta
- **IDs de conversación consistentes**: Usa IDs significativos y consistentes para asegurar que las memorias se agrupen correctamente y sean recuperables
- **Combina con bloque Agente**: Para mejores resultados, usa Memoria junto con el bloque Agente para construir contexto de conversación completo
