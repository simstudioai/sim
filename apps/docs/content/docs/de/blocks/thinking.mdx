Der Thinking-Block zwingt ein KI-Modell dazu, seinen Gedankenprozess explizit zu skizzieren, bevor es fortfährt. Durch die Förderung von Schritt-für-Schritt-Analysen kann dieser Block die Reasoning-Qualität und Output-Genauigkeit bei komplexen Aufgaben verbessern.

<Callout type="info">

</Callout>

## Konfigurationsoptionen

### Thought

Der Gedankenprozess oder Reasoning-Schritt, den das Modell dokumentieren sollte. Dieses Feld ist in der UI verborgen, ist aber für die interne Verarbeitung erforderlich.

- **Typ**: String (Markdown unterstützt)
- **Zweck**: Erfasst das explizite Reasoning des Modells vor der Erstellung der endgültigen Ausgabe

## Outputs

- **`<thinking.acknowledgedThought>`**: Die Bestätigung und Verarbeitung des skizzierten Gedankenprozesses durch das Modell

## Wie es funktioniert

Der Thinking-Block funktioniert durch:

1. Annahme eines Gedankens oder Reasoning-Prompts
2. Anweisung an das KI-Modell, den Gedankenprozess explizit zu durcharbeiten
3. Rückgabe des bestätigten Gedankens als Ausgabe, die in nachfolgende Blöcke fließen kann

Dies ermutigt das Modell, komplexe Reasoning in explizite Schritte zu zerlegen, was zu genaueren und zuverlässigeren Ausgaben führen kann, besonders für:

- **Multi-Step-Problemlösung**: Aufgaben, die sequentielles Reasoning erfordern
- **Komplexe Analyse**: Situationen, in denen Zwischenschritte wichtig sind
- **Chain-of-Thought-Reasoning**: Workflows, die von expliziter Schritt-für-Schritt-Logik profitieren
- **Verifizierungsschritte**: Explizite Reasoning-Checks vor endgültigen Ausgaben

## Beispiel-Anwendungsfälle

**Komplexe Problemlösung mit Reasoning-Schritten**
```
Input → Thinking (outline approach) → Agent (execute with reasoning) → Response
```

**Qualitätssicherung mit expliziter Analyse**
```
Content → Agent (analyze) → Thinking (verify reasoning) → Function (validate) → Response
```

**Chain-of-Thought-Verifizierung**
```
Question → Thinking (break down problem) → Agent (solve step-by-step) → Response
```

## Best Practices

- **Verwende für komplexe Aufgaben**: Wende Thinking-Blöcke auf Workflows mit komplexem Reasoning, mehrstufiger Logik oder hohen Genauigkeitsanforderungen an
- **Kombiniere mit Agent-Blöcken**: Verwende Thinking in Kombination mit Agent-Blöcken, um ihre Reasoning-Fähigkeiten zu verbessern
- **Halte Gedanken fokussiert**: Schreibe klare, spezifische Gedanken, die das Modell zu besserem Reasoning leiten
- **Überwache die Leistung**: Teste Workflows mit und ohne Thinking-Blöcke, um die Auswirkung auf die Output-Qualität zu messen

<Callout type="tip">
  Für standardmäßige KI-Interaktionen ist der Agent-Block allein normalerweise ausreichend. Der Thinking-Block ist am wertvollsten bei komplexen Reasoning-Aufgaben, bei denen explizite Schritt-für-Schritt-Analyse nachweislich die Ergebnisse verbessert.
</Callout>
