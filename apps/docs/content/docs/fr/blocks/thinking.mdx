Le bloc Thinking force un modèle d'IA à explicitement décrire son processus de réflexion avant de continuer. En encourageant l'analyse étape par étape, ce bloc peut améliorer la qualité du raisonnement et la précision de la sortie sur des tâches complexes.

<Callout type="info">
  Le bloc Thinking est caché de la barre d'outils des blocs par défaut et est utilisé principalement dans les flux de travail avancés. Il ne s'affiche pas dans l'interface utilisateur de sélection de bloc standard mais peut être accédé par programmation ou par importation.
</Callout>

## Options de configuration

### Thought

Le processus de réflexion ou l'étape de raisonnement que le modèle doit documenter. Ce champ est caché de l'interface utilisateur mais est nécessaire pour le traitement interne.

- **Type** : String (Markdown supporté)
- **Objectif** : Capture le raisonnement explicite du modèle avant de générer la sortie finale

## Sorties

- **`<thinking.acknowledgedThought>`** : L'accusé de réception et le traitement du modèle du processus de réflexion décrit

## Comment cela fonctionne

Le bloc Thinking fonctionne en :

1. Acceptant une pensée ou un prompt de raisonnement
2. Instruisant le modèle d'IA d'explicitement travailler à travers le processus de réflexion
3. Retournant la pensée reconnue en tant que sortie qui peut alimenter les blocs suivants

Cela encourage le modèle à décomposer le raisonnement complexe en étapes explicites, ce qui peut mener à des sorties plus précises et fiables, particulièrement pour :

- **Résolution de problèmes multi-étapes** : Tâches nécessitant un raisonnement séquentiel
- **Analyse complexe** : Situations où les étapes intermédiaires importent
- **Raisonnement chaîne-de-pensée** : Flux de travail qui bénéficient d'une logique explicite étape par étape
- **Étapes de vérification** : Ajouter des vérifications de raisonnement explicite avant les sorties finales

## Cas d'utilisation courants

**Résolution de problèmes complexes avec étapes de raisonnement**
```
Input → Thinking (outline approach) → Agent (execute with reasoning) → Response
```

**Assurance qualité avec analyse explicite**
```
Content → Agent (analyze) → Thinking (verify reasoning) → Function (validate) → Response
```

**Vérification chaîne-de-pensée**
```
Question → Thinking (break down problem) → Agent (solve step-by-step) → Response
```

## Bonnes pratiques

- **Utilisez pour les tâches complexes** : Appliquez les blocs Thinking aux flux de travail impliquant un raisonnement complexe, une logique multi-étapes ou des exigences de haute précision
- **Combinez avec les blocs Agent** : Utilisez Thinking en conjonction avec les blocs Agent pour améliorer leurs capacités de raisonnement
- **Gardez les pensées ciblées** : Écrivez des pensées claires et spécifiques qui guident le modèle vers un meilleur raisonnement
- **Surveillez les performances** : Testez les flux de travail avec et sans blocs Thinking pour mesurer l'impact sur la qualité de la sortie

<Callout type="tip">
  Pour les interactions IA standard, le bloc Agent seul est généralement suffisant. Le bloc Thinking est plus précieux lorsqu'il s'agit de tâches de raisonnement complexe où l'analyse explicite étape par étape améliore mesurément les résultats.
</Callout>
