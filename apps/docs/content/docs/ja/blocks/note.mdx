ノートブロックはコンテキスト注釈をワークフローキャンバスに直接追加します。これを使用して、ワークフロー実行に影響を与えることなく、決定事項をドキュメント化し、指示を共有し、コラボレーティブなコンテキストを残します。

<Callout type="info">
  ノートブロックはドキュメント化とコラボレーション用のみです。ワークフロー実行中に実行されず、ワークフロー ロジックまたは出力に影響しません。
</Callout>

## 構成オプション

### コンテンツ

キャンバスに表示されるノートコンテンツ。ノートは完全なMarkdownレンダリングをサポートしており、以下が含まれます：

- **テキスト形式**：太字、斜体、リンク、インラインコード
- **リスト**：箇条書きと番号付きリスト
- **コードブロック**：構文強調表示されたコードサンプル
- **見出し**：組織化用の複数の見出しレベル
- **YouTubeの埋め込み**：標準の埋め込み構文を使用してビデオを直接埋め込む
- **ブロック参照**：ワークフロー内のブロック名を使用して他のブロックにリンク

### Markdownの例

```markdown
## 処理戦略

このワークフローは以下を使用して顧客サポートチケットを処理します：
1. 意図別にルーティング（FAQ対エスカレーション）
2. エージェントで応答を生成
3. 分析用にデータベースにログ

**重要**：常に応答にチケットIDを含めてください。

[詳細情報](https://docs.example.com)
```

## ユースケース

**ワークフロー決定ドキュメント**
```
特定のルーティングロジックが使用される理由、ビジネスルール、または他のチームメンバーが理解すべき決定ポイントを説明するノートを追加します
```

**コラボレーターへの指示**
```
セットアップ手順、設定要件、または特定のブロック前の前提条件を含むノートを残します
```

**組み込みの学習リソース**
```
新しいチームメンバーがワークフローを理解するのに役立つドキュメント、YouTubeチュートリアル、または例へのリンクを含めます
```

## ベストプラクティス

- **ノートを簡潔に保つ**：ノートを使用して、明確なワークフロー設計を補完しますが、置き換えません。広範なドキュメント化が必要な場合は、外部リソースを検討してください
- **ワークフロー変更でアップデート**：ワークフローロジックを変更するときに、関連するノートを更新して変更を反映させます
- **チーム通信に使用**：ノートはすべてのコラボレーターに表示されます。これを使用して、自明でない設計上の決定事項を説明します
- **リソースにリンク**：外部ドキュメント、FAQ、または決定ログへのリンクを含めて、より多くのコンテキストを取得します
