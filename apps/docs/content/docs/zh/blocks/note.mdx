Note 模块在工作流画布上直接添加上下文注解。使用它来记录决策、共享指令或留下协作上下文，而不影响工作流执行。

</Callout>

## 配置选项

### Content

显示在画布上的注解内容。注解支持完整的 Markdown 渲染，包括：

- **文本格式**：粗体、斜体、链接、代码内联
- **列表**：项目符号和编号列表
- **代码块**：语法高亮的代码示例
- **标题**：多个标题级别用于组织
- **YouTube 嵌入**：使用标准嵌入语法直接嵌入视频
- **模块引用**：使用模块名称链接到工作流中的其他模块

### Markdown 示例

```markdown
## 处理策略

此工作流使用以下方式处理客户支持工单：
1. 按意图路由（FAQ 对比升级）
2. 使用 Agent 生成响应
3. 记录到数据库用于分析

**重要**：始终在响应中包含工单 ID。

[更多信息](https://docs.example.com)
```

## 使用案例

**工作流决策文档**
```
添加一个 Note 解释为什么使用特定路由逻辑、业务规则或其他团队成员应该理解的决策点
```

**协作者指令**
```
留下关于设置指令、配置要求或某些模块之前的先决条件的注解
```

**嵌入学习资源**
```
包括帮助新团队成员理解工作流的文档、YouTube 教程或示例链接
```

## 最佳实践

- **保持注解简洁**：使用注解来补充而不是替代清晰的工作流设计。如果您编写大量文档，请考虑外部资源
- **随工作流更改更新**：当您修改工作流逻辑时，更新相关注解以反映更改
- **用于团队沟通**：注解对所有协作者可见——使用它们来解释非明显的设计决策
- **链接到资源**：包括指向外部文档、FAQs 或决策日志的链接以获得更多上下文
