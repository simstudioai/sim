IMAP provides 1 trigger for automating workflows based on events.

All triggers below are **polling-based** — they check for new data on a schedule rather than receiving push notifications.

## IMAP Email Trigger

Triggers when new emails are received via IMAP (works with any email provider)

### Configuration

| Field | Type | Required | Description |
| ----- | ---- | -------- | ----------- |
| `host` | short-input | Yes | IMAP server hostname \(e.g., imap.gmail.com, outlook.office365.com\) |
| `port` | short-input | Yes | IMAP port \(993 for SSL/TLS, 143 for STARTTLS\) |
| `secure` | switch | No | Enable SSL/TLS encryption \(recommended for port 993\) |
| `username` | short-input | Yes | Email address or username for authentication |
| `password` | short-input | Yes | Password or app-specific password \(for Gmail, use an App Password\) |
| `mailbox` | dropdown | No | Choose which mailbox/folder\(s\) to monitor for new emails. Leave empty to monitor INBOX. |
| `searchCriteria` | code | No | ImapFlow search criteria as JSON object. Default: unseen messages only. |
| `markAsRead` | switch | No | Automatically mark emails as read \(SEEN\) after processing |
| `includeAttachments` | switch | No | Download and include email attachments in the trigger payload |

### Output

| Field | Type | Description |
| ----- | ---- | ----------- |
| `email` | string | RFC Message-ID header |
| `timestamp` | string | Event timestamp |
