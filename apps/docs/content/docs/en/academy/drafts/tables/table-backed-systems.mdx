**Learning outcome.** The learner can decide when a use case should be table-backed.

## When to use a table-backed system

Reach for a table-backed system when:

- many similar records exist
- each row needs a status or output
- users need to compare results
- work should run per row
- results need to persist
- downstream steps depend on previous outputs

## Examples

```
GTM enrichment
support triage
recruiting enrichment
document extraction
market research
ops queues
```
