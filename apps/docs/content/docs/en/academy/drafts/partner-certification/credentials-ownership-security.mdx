**Learning outcome.** The partner understands how to handle customer credentials safely.

## Key beats

1. Credentials let workflows access external systems.
2. Customer-owned production credentials are preferred.
3. Avoid hardcoding secrets.
4. Distinguish personal credentials, workspace credentials, integration credentials, and API keys / secrets.
5. Handoff must include: who owns credentials, who can use them, what happens if the partner leaves, what appears in logs, and how to rotate / update secrets.

<Callout type="warn">
Open product questions: exact credential visibility / usage rules, logs redaction behavior, and personal vs. workspace credential behavior.
</Callout>
