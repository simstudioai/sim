**Purpose.** Show the learner what "using Sim" feels like at the most basic level.

**Learning outcome.** The learner understands how to run a workflow, where output appears, where to look if something fails, and that Sim systems are inspectable.

## What to say

- Once you have a workflow, you can run it.
- When it runs, Sim records what happened.
- You can inspect the output.
- If something fails, logs show where and why.
- This matters because AI systems should be inspectable, not just magical.

## Suggested order

1. Use the simple workflow from the previous video.
2. Enter a sample input.
3. Run it.
4. Show the output.
5. Open logs.
6. Show a successful run, block outputs, and where errors would appear.
7. Keep it light: *You don't need to master logs yet. Just know this is where you go to see what happened.*

## What not to do

- Don't create a long debugging session.
- Don't teach advanced logs.
- Don't inspect cost or model details unless it naturally appears.
- Don't turn this into an operations lesson.
