**Learning outcome.** The partner can update live systems safely.

## Key beats

1. Deployment creates a snapshot / version.
2. Editing a workflow does not automatically change the live deployment.
3. Promote a version to live intentionally.
4. Roll back by promoting a previous version.
5. For customers: test before promoting, know the current live version, and keep the rollback path clear.
6. Staging / prod pattern: duplicate workflows or folders if needed — confirm the official recommendation.
