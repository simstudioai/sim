## Why files

Files are the artifact layer: documents, images, markdown, PDFs, reports, and generated outputs. They're less conceptually central than workflows, tables, and knowledge bases, but important because many real systems consume or produce artifacts.

## Core thesis

> Files are Sim's artifact layer: documents and media that systems can store, reference, transform, and generate.

<Callout type="info">
Use a file when the document or media object itself matters. Use a knowledge base when the contents need semantic search. Use a table when outputs should become structured records.
</Callout>
