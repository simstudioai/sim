## Key beats

### 1. Workspace as system boundary

A workspace contains the resources for a system: workflows, tables, files, knowledge bases, integrations, credentials, deployments, and logs.

### 2. Personal vs. team / customer workspace

- **Personal** — your own experiments and resources.
- **Team / customer** — a shared environment where multiple people access and operate systems.

### 3. Workspace organization

Organize workflows and resources so systems are understandable. Hierarchy matters for bigger systems; the desired pattern is folders across all resource types to make subsystems easier to manage.

### 4. Workspace as customer app

For partners and enterprise, the workspace is effectively the customer's app environment — what they'll use, own, and maintain.
