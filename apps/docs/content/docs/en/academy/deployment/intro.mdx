**Learning outcome.** The learner understands what deployment means in Sim and the main deployment surfaces.

## Key beats

### 1. Start with the problem

A workflow on the canvas is internal. Deployment makes it usable externally.

### 2. Define deployment

A deployed workflow can be called or interacted with outside the editor. The main surfaces are **API**, **Chat**, and **MCP**.

### 3. Deployment snapshot

Deploying creates a version / snapshot. Updating the workflow doesn't necessarily mean the live deployment changed — important for reliability.

### 4. When deployments matter

Customer-facing systems, internal tools, API endpoints, chat apps, and workflows used by other AI tools.

## Avoid

- Don't deep-dive every API parameter in the intro.
- Don't turn it into partner versioning yet.
- Don't start with MCP if the audience is non-technical.
