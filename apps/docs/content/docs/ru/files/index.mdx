**Файл** – это документ, изображение, электронная таблица или PDF-файл в вашем рабочем пространстве. Файлы используются для перемещения документов и медиа между вашими агентами. Ваша команда загружает их, вы создаете их в редакторе, или рабочий процесс генерирует их, и все они хранятся в одном месте, доступном всем пользователям рабочего пространства.

Любой[рабочий процесс](/workflows)рабочий процесс/рабочие процессыможно прочитать файл или создать его. Вы можете загрузить договор для ознакомления агента, сгенерировать отчет из таблицы или передать документу рабочий процесс, который ему необходим для ответа на вопрос.

## Как файлы организуются в рабочем пространстве.
- **[Рабочие процессы](/рабочие_процессы)** позволяют читать файлы (например, PDF для создания сводки) и создавать их (например, отчёт), как если бы он был отрисован. Подробнее о работе с файлами в рабочих процессах: [использование файлов в рабочих процессах](/файлы/использование_в_рабочих_процессах).- **Базы знаний** создаются на основе файлов, которые вы загружаете, и превращают их содержимое в удобный для поиска архив.- **Развертывание** (под задачи / развертывания) может принимать файл в качестве входных данных и возвращать один файл в качестве выходных данных.
Используйте файл, когда важен сам документ или медиа-файл. Используйте...[таблица](/tables)таблица/таблицыкогда вам нужны структурированные строки и поля, а также...[база знаний](/knowledgebase)база знаний/база знанийкогда агенту необходимо искать информацию в большом количестве документов.
