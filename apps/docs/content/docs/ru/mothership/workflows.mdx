Опишите рабочий процесс, и Mothership его создаст. Укажите существующий по имени, и он его отредактирует. Навигация по холсту не требуется — каждое изменение отображается на панели ресурсов в реальном времени.

## Создание рабочих процессов

Опишите, что должен делать рабочий процесс — что его запускает, что он должен делать, какие интеграции нужны и что он должен возвращать. Mothership создаёт его и открывает холст на панели ресурсов.

- «Создай рабочий процесс, который принимает URL, парсит страницу, суммирует её с помощью Claude и отправляет сводку в канал Slack»
- «Создай рабочий процесс, запускаемый вебхуком, который извлекает данные счёта из PDF и записывает их в таблицу billing»
- «Создай исходящий рабочий процесс: прими название компании и домен, обогати фирмографическими данными, оцени соответствие и составь персонализированное холодное письмо»
- «Создай рабочий процесс обогащения лидов, который принимает email из отправки формы, ищет компанию и записывает обогащённую запись в таблицу leads»
- «Создай рабочий процесс онбординга клиентов: когда новый пользователь регистрируется, отправь приветственное письмо, создай контакт в HubSpot и опубликуй уведомление в #new-customers в Slack»

## Редактирование рабочих процессов

{/* TODO: Screenshot of Mothership with the Edit subagent active and a change applied to an open workflow — e.g., a new block added or a configuration updated, visible on the canvas in the resource panel. */}

Откройте существующий рабочий процесс с помощью `@имя-процесса` или меню **+**, затем опишите изменение. Mothership читает текущую структуру перед изменением — вам не нужно объяснять, что уже существует.

- «Добавь условие, которое направляет в другую ветку, если оценка уверенности ниже 0.7»
- «Замени модель GPT-4o на Claude Opus 4.6 в блоке суммирования»
- «Добавь уведомление в Slack в конце, которое включает результат»

## Запуск рабочих процессов

<Video src="mothership/run-workflow.mp4" width={700} height={450} />

Попросите Mothership запустить рабочий процесс, и он выполнит его:

- «Запусти рабочий процесс data sync»
- «Запусти обработчик счетов с этим PDF [прикрепите файл]»
- «Протестируй рабочий процесс оценки лидов с этими входными данными: name=Acme, score=0.4»

Выполнение транслируется обратно в чат. Рабочий процесс на панели ресурсов показывает состояние блоков в реальном времени.

## Чтение журналов

Mothership может извлекать и интерпретировать журналы выполнения для любого рабочего процесса в пространстве:

- «Покажи последние 10 запусков рабочего процесса pipeline»
- «Почему рабочий процесс invoice не удался вчера?»
- «Что вернул блок extractor в последнем запуске?»

Журналы включают состояние выполнения каждого блока, выводы, ошибки и время выполнения.

## Отладка

Когда рабочий процесс завершается с ошибкой, попросите Mothership отладить его:

- «Отладь последний неудачный запуск content pipeline»
- «Блок summarizer возвращает пустой вывод — выясни почему»

Mothership читает журналы ошибок, определяет причину, применяет исправление и может перезапустить для подтверждения.

## Развёртывание

Mothership может развернуть рабочий процесс в любом из трёх типов развёртывания:

| Тип развёртывания | Что создаёт |
|----------------|----------------|
| **API** | REST-эндпоинт по адресу `https://sim.ai/api/workflows/{id}/execute` |
| **Чат** | Размещённый диалоговый интерфейс с общедоступным URL |
| **Инструмент MCP** | MCP-сервер, предоставляющий рабочий процесс как инструмент |

Попросите: «Разверни рабочий процесс invoice как API и сгенерируй API-ключ.»

Mothership также может откатить: «Верни рабочий процесс billing к версии с прошлого вторника.»

См. [Развёртывание API](/workflows/deployment/api) и [Развёртывание чата](/workflows/deployment/chat) для полной информации о каждом типе развёртывания.

## Организация рабочих процессов

Mothership может создавать папки и управлять ими для поддержания порядка в рабочем пространстве.

**Папки:**
- «Создай папку 'Data Pipelines'»
- «Перемести рабочий процесс invoice в папку billing»
- «Перемести папку billing внутрь папки finance»
- «Удали папку old-experiments»

**Переименование и перемещение:**
- «Переименуй рабочий процесс 'test_v2' в 'lead-scorer'»
- «Перемести рабочий процесс summarizer в папку research»

## Переменные рабочего процесса

Mothership может задавать глобальные переменные для рабочего процесса — значения, доступные всем блокам в этом процессе во время выполнения:

- «Задай переменную API_ENDPOINT для рабочего процесса sync как 'https://api.example.com/v2'»
- «Обнови переменную MAX_RETRIES для рабочего процесса pipeline на 5»

Заданные таким образом переменные доступны через синтаксис `<variable.VARIABLE_NAME>` внутри любого блока рабочего процесса.

## Удаление рабочих процессов

- «Удали рабочий процесс old_api_prototype»
- «Удали все рабочие процессы в папке deprecated»
