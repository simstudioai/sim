Mothership может действовать от вашего имени прямо сейчас — отправить сообщение, создать задачу, вызвать API — или по расписанию, автоматически выполняя промпт каждый час, день или неделю. Он также может подключать интеграции, задавать переменные окружения, добавлять MCP-серверы и создавать пользовательские инструменты.

## Запланированные задания

Запланированное задание — это задача Mothership, которая выполняется по cron-расписанию. При каждом запуске Mothership считывает текущее состояние рабочего пространства и выполняет промпт задания так, как будто вы только что его отправили.

### Создание задания

Опишите повторяющуюся задачу и как часто она должна выполняться:

- «Каждое утро в 8:00 проверяй таблицу leads на новые записи и публикуй сводку в #sales в Slack»
- «Каждый понедельник в 9:00 собирай количество запусков рабочих процессов за прошлую неделю и записывай отчёт в рабочее пространство»
- «Запускай рабочий процесс data sync каждые 6 часов»
- «Первого числа каждого месяца экспортируй таблицу billing в CSV и отправляй на finance@example.com»
- «Каждый будний день в 7:00 проверяй новые объявления о финансировании от компаний из нашего ICP и публикуй топ-5 в #market-intel в Slack»
- «Каждое воскресенье вечером запускай рабочий процесс обогащения лидов для всех перспектив, добавленных за прошлую неделю, и обновляй их оценки в таблице»
- «Ежедневно в 6:00 собирай ошибки рабочих процессов за предыдущий день, суммируй основные проблемы и публикуй в #eng-alerts»

Mothership задаёт cron-выражение и сохраняет промпт задания. Первый запуск происходит в следующее запланированное время.

### Просмотр журналов заданий

- «Покажи последние 5 запусков задания weekly report»
- «Успешно ли выполнилось задание sync сегодня утром?»
- «Что делало задание Monday digest на прошлой неделе?»

Журналы показывают время выполнения, статус (completed, failed) и сводку того, что сделал агент.

### Управление заданиями

- «Приостанови задание morning summary»
- «Измени задание sync на запуск каждые 3 часа вместо 6»
- «Удали задание onboarding digest»
- «Какие запланированные задания сейчас активны?»

## Прямые действия

Для запросов, которые должны быть выполнены немедленно — без создания рабочего процесса — просто попросите. Mothership действует сразу, используя учётные данные, подключённые к вашему рабочему пространству.

{/* TODO: Screenshot of Mothership chat showing the "Taking action" subagent label active during a direct action — e.g., posting to Slack or sending an email. Shows the subagent inline in the chat thread. */}

| Запрос | Что происходит |
|---------|-------------|
| «Отправь сообщение в Slack в #eng, что развёртывание завершено» | Публикует в Slack немедленно |
| «Отправь отчёт Q3 на jane@example.com» | Отправляет через подключённый Gmail или Outlook |
| «Создай задачу на GitHub: токены аутентификации не сбрасываются при выходе» | Открывает задачу в указанном репозитории |
| «Добавь контакт в HubSpot: Acme Corp, ceo@acme.com» | Создаёт контакт через HubSpot API |
| «Вызови вебхук по адресу [URL] с этим JSON» | Выполняет HTTP-запрос |

Если интеграция не подключена, Mothership проведёт вас через процесс подключения.

## Подключение интеграций

Mothership может подключать новые OAuth-интеграции и API-ключи по запросу:

- «Подключи мой аккаунт Google»
- «Добавь рабочее пространство Slack для нашей команды»
- «Настрой GitHub с моим персональным токеном доступа»

{/* TODO: Screenshot of Mothership walking through connecting an integration — e.g., the Integration subagent active with an OAuth prompt or confirmation that a credential was connected. */}

После подключения учётные данные доступны Mothership для прямых действий и запланированных заданий, а также всем рабочим процессам в рабочем пространстве.

<Callout type="info">
  Подключённые учётные данные являются общими для всего рабочего пространства. Любой рабочий процесс, использующий ту же интеграцию, будет автоматически использовать те же учётные данные.
</Callout>

См. [Учётные данные](/platform/credentials) для управления подключёнными аккаунтами.

## Переменные окружения

Переменные окружения — это значения уровня рабочего пространства: API-ключи, строки подключения и конфигурация, на которые рабочие процессы ссылаются через синтаксис `{{ENV_VAR}}` вместо жёсткого кодирования. Задайте их один раз, и каждый рабочий процесс в пространстве сможет их использовать.

- «Задай переменную окружения DATABASE_URL как 'postgres://...'»
- «Добавь переменную окружения OPENAI_API_KEY»
- «Добавь переменную WEBHOOK_SECRET для рабочего процесса inbound webhook»
- «Обнови переменную SCORING_API_URL, чтобы она указывала на новый эндпоинт»
- «Какие переменные окружения сейчас заданы?»

## MCP-серверы

MCP-серверы (Model Context Protocol) предоставляют инструменты от внешних сервисов, которые блоки Agent могут вызывать внутри рабочих процессов. Подключение MCP-сервера делает все его инструменты доступными в выборе инструментов редактора рабочих процессов — без необходимости писать код интеграции.

Mothership может добавлять и управлять MCP-серверами, подключёнными к вашему рабочему пространству:

- «Добавь MCP-сервер Stripe, используя мой API-ключ»
- «Удали старый MCP-сервер аналитики»
- «Какие MCP-серверы подключены к этому рабочему пространству?»
- «Обнови эндпоинт для MCP-сервера internal tools на [URL]»

После добавления инструменты MCP появляются в выборе инструментов редактора рабочих процессов и могут быть вызваны из любого блока Agent.

## Пользовательские инструменты

Пользовательские инструменты — это отдельные HTTP-эндпоинты, которые вы определяете вручную — полезно для внутренних API и сервисов, у которых нет встроенной интеграции Sim или MCP-сервера. После создания они появляются в редакторе рабочих процессов наряду со встроенными инструментами и могут быть вызваны из любого блока Agent.

Mothership может создавать пользовательские инструменты по описанию:

- «Создай пользовательский инструмент, который вызывает наш внутренний API оценки по адресу [URL] с POST-запросом и возвращает поле score»
- «Создай инструмент для нашего Zendesk, который создаёт тикет с темой и содержимым»
- «Создай инструмент, который обращается к нашему внутреннему API обогащения с доменом и возвращает размер компании, отрасль и стадию финансирования»
- «Добавь инструмент, который вызывает REST API нашей CRM для поиска контакта по email и возвращает его ответственного менеджера»

Пользовательские инструменты появляются в редакторе рабочих процессов и могут вызываться из любого блока Agent наряду со встроенными инструментами.
