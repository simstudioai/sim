Создайте базу знаний, добавьте в неё документы и запрашивайте её на естественном языке — всё через общение. Базы знаний, созданные вами в Mothership, сразу же доступны блокам Agent в любом рабочем процессе.

## Создание баз знаний

Опишите базу знаний, и Mothership создаст её:

- «Создай базу знаний под названием 'Product Docs'»
- «Настрой базу знаний для нашей команды поддержки — назови 'Support KB'»
- «Создай базу знаний конкурентной разведки»
- «Создай базу знаний из нашего учебника по продажам и прикрепи её к рабочему процессу outbound agent»
- «Настрой базу знаний для отдела поддержки клиентов — я добавлю в неё наши руководства по онбордингу и прошлые кейсы»

## Добавление документов

Добавляйте документы, прикрепляя файлы к сообщению, вставляя текст или указывая Mothership на URL:

- «Добавь этот PDF в базу знаний Product Docs [прикрепите файл]»
- «Добавь следующий текст в Support KB как новый документ: [вставьте содержимое]»
- «Загрузи страницу по адресу [URL] и добавь её в базу знаний конкурентной разведки»
- «Добавь эти три загруженных кейса в базу знаний поддержки клиентов»

Mothership обрабатывает и индексирует каждый документ автоматически. После индексации содержимое доступно для поиска любому блоку Agent, к которому прикреплена база знаний.

## Запрос к базам знаний

Задайте Mothership вопрос, и он выполнит поиск в указанной базе знаний для ответа:

- «Что говорит база знаний Product Docs о нашей политике возврата?»
- «Найди в Support KB всё, что связано с ошибками настройки SSO»
- «Какие ключевые различия между нашими тарифами Pro и Enterprise, согласно документации продукта?»
- «Найди всё в базе знаний конкурентной разведки о ценах [конкурента]»

## Коннекторы

Для баз знаний, которые должны оставаться актуальными автоматически, коннекторы синхронизируют содержимое из внешних сервисов по расписанию — ручная загрузка не требуется. Новое содержимое добавляется, изменённое — переобрабатывается, а удалённое — убирается при каждом запуске.

Коннекторы настраиваются через параметры базы знаний, а не через чат Mothership. После подключения всё синхронизированное содержимое сразу доступно для поиска Mothership и любому блоку Agent с прикреплённой базой знаний.

Sim поставляется с 49 встроенными коннекторами, включая Notion, Google Drive, Slack, GitHub, Confluence, HubSpot, Salesforce, Gmail и другие.

Примеры того, что можно синхронизировать:

- **Notion** — синхронизация рабочего пространства, базы данных или определённого дерева страниц
- **Google Drive / Dropbox / OneDrive** — синхронизация документов из облачного хранилища
- **GitHub** — синхронизация markdown и файлов кода репозитория
- **Slack** — синхронизация истории канала
- **Confluence / Jira** — синхронизация внутренней вики или трекера задач
- **HubSpot / Salesforce** — синхронизация записей CRM в доступную для поиска базу знаний

См. [Коннекторы](/knowledgebase/connectors) для шагов настройки, опций частоты синхронизации и управления статусом коннекторов.

## Управление базами знаний

Просматривайте, проверяйте и очищайте базы знаний на естественном языке:

- «Какие базы знаний есть в этом рабочем пространстве?»
- «Сколько документов в Support KB?»
- «Удали устаревший документ о ценах из базы знаний Product Docs»
- «Удали базу знаний old-competitive-intel»

## Использование баз знаний в рабочих процессах

Базы знаний, созданные в Mothership, сразу же доступны блокам Agent в любом рабочем процессе. Прикрепите базу знаний к блоку Agent, и он будет использовать семантический поиск для извлечения релевантного содержимого во время выполнения.

См. [База знаний](/knowledgebase) для полной информации о настройках обработки документов, конфигурации поиска и синхронизации коннекторов.
