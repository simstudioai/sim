Для доступа к API Sim вам потребуется ключ API. API Sim поддерживает два типа ключей – **личные ключи** и **ключи для рабочих пространств**, каждый из которых имеет различные правила выставления счетов и доступа.

## Основные типы
|  |  | **Личные ключи** | **Ключи для рабочего пространства** |  || --- | --- | --- |
|  | **Оплачено на имя** | Ваш личный аккаунт | Владелец рабочего пространства |  ||  | **Область применения** | В рамках различных рабочих пространств у вас есть доступ к... | Доступно во всей рабочей среде |  ||  | Управляется | Каждый пользователь индивидуально | Администраторы рабочих пространств |  ||  | **Разрешения** | Должно быть включено на уровне рабочего пространства. | Требуются права администратора |  |
<Callout type="info">
  Workspace admins can disable personal API key usage for their workspace. If disabled, only workspace keys can be used.
</Callout>

## Генерация ключей API
Чтобы сгенерировать ключ, откройте панель управления Sim и перейдите в раздел **Настройки**, затем перейдите в раздел **Ключи Sim** и нажмите кнопку **Создать**.

<Callout type="warn">
  API keys are only shown once when generated. Store your key securely — you will not be able to view it again.
</Callout>

## Использование ключей API
Передавайте свой ключ API в заголовке `X-API-Key` при каждом запросе:

<Tabs items={['curl', 'TypeScript', 'Python']}>
  <Tab value="curl">
    ```bash
    curl -X POST https://www.sim.ai/api/workflows/{workflowId}/execute \
      -H "Content-Type: application/json" \
      -H "X-API-Key: YOUR_API_KEY" \
      -d '{"inputs": {}}'
    ```
  </Tab>
  <Tab value="TypeScript">
    ```typescript
    const response = await fetch(
      'https://www.sim.ai/api/workflows/{workflowId}/execute',
      {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'X-API-Key': process.env.SIM_API_KEY!,
        },
        body: JSON.stringify({ inputs: {} }),
      }
    )
    ```
  </Tab>
  <Tab value="Python">
    ```python
    import requests

    response = requests.post(
        "https://www.sim.ai/api/workflows/{workflowId}/execute",
        headers={
            "Content-Type": "application/json",
            "X-API-Key": os.environ["SIM_API_KEY"],
        },
        json={"inputs": {}},
    )
    ```
  </Tab>
</Tabs>

## Где используются ключи?
Ключи API используются для аутентификации доступа к:

- **Выполнение рабочих процессов** – запуск развернутых рабочих процессов через API.- **API для журналов** – позволяет запрашивать логи выполнения рабочих процессов и метрики.- **Серверы MCP** – аутентифицируют соединения с развернутыми серверами MCP.- **SDK:** Python ([`/api-reference/python`]) и TypeScript ([`/api-reference/typescript`]) SDK используют ключи API для всех операций.
## Безопасность
- Ключи используют префикс "`sk-sim-`" и зашифрованы при хранении.- Ключи можно отозвать в любое время из панели управления.- Используйте переменные окружения для хранения ключей – никогда не храните их непосредственно в исходном коде.- Для приложений, работающих в браузере, используйте прокси-сервер на стороне сервера, чтобы избежать раскрытия ключей для клиента.
<Callout type="warn">
  Never expose your API key in client-side code. Use a server-side proxy to make authenticated requests on behalf of your frontend.
</Callout>
