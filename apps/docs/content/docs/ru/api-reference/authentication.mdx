Для доступа к Sim API вам необходим API-ключ. Sim поддерживает два типа API-ключей — **персональные ключи** и **ключи рабочей области** — каждый из которых имеет различное поведение в отношении биллинга и доступа.

## Типы ключей

|  | **Персональные ключи** | **Ключи рабочей области** |
| --- | --- | --- |
| **Оплата через** | Ваш индивидуальный аккаунт | Владелец рабочей области |
| **Область действия** | Все рабочие области, к которым у вас есть доступ | Общая для всей рабочей области |
| **Управление** | Каждый пользователь самостоятельно | Администраторы рабочей области |
| **Разрешения** | Должны быть включены на уровне рабочей области | Требуют прав администратора |

<Callout type="info">
  Администраторы рабочей области могут отключить использование персональных API-ключей для своей рабочей области. Если использование отключено, можно применять только ключи рабочей области.
</Callout>

## Создание API-ключей

Чтобы создать ключ, откройте панель управления Sim и перейдите в **Settings**, затем в **Sim Keys** и нажмите **Create**.

<Callout type="warn">
  API-ключи отображаются только один раз при создании. Храните ключ в безопасном месте — вы не сможете просмотреть его снова.
</Callout>

## Использование API-ключей

Передавайте ваш API-ключ в заголовке `X-API-Key` с каждым запросом:

<Tabs items={['curl', 'TypeScript', 'Python']}>
  <Tab value="curl">
    ```bash
    curl -X POST https://www.sim.ai/api/workflows/{workflowId}/execute \
      -H "Content-Type: application/json" \
      -H "X-API-Key: YOUR_API_KEY" \
      -d '{"inputs": {}}'
    ```
  </Tab>
  <Tab value="TypeScript">
    ```typescript
    const response = await fetch(
      'https://www.sim.ai/api/workflows/{workflowId}/execute',
      {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'X-API-Key': process.env.SIM_API_KEY!,
        },
        body: JSON.stringify({ inputs: {} }),
      }
    )
    ```
  </Tab>
  <Tab value="Python">
    ```python
    import requests

    response = requests.post(
        "https://www.sim.ai/api/workflows/{workflowId}/execute",
        headers={
            "Content-Type": "application/json",
            "X-API-Key": os.environ["SIM_API_KEY"],
        },
        json={"inputs": {}},
    )
    ```
  </Tab>
</Tabs>

## Где используются ключи

API-ключи обеспечивают аутентификацию доступа к:

- **Выполнение рабочих процессов** — запуск развёрнутых рабочих процессов через API
- **API журналов** — запрос журналов выполнения рабочих процессов и метрик
- **MCP-серверы** — аутентификация подключений к развёрнутым MCP-серверам
- **SDK** — [Python](/api-reference/python) и [TypeScript](/api-reference/typescript) SDK используют API-ключи для всех операций

## Безопасность

- Ключи используют префикс `sk-sim-` и хранятся в зашифрованном виде
- Ключи можно отозвать в любое время из панели управления
- Используйте переменные окружения для хранения ключей — никогда не вставляйте их напрямую в исходный код
- Для приложений на основе браузера используйте бэкенд-прокси, чтобы избежать раскрытия ключей клиенту

<Callout type="warn">
  Никогда не раскрывайте ваш API-ключ в клиентском коде. Используйте серверный прокси для выполнения аутентифицированных запросов от имени вашего фронтенда.
</Callout>
