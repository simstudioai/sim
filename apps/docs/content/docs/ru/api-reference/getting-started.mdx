## Базовый URL
Все запросы к API отправляются по адресу:

```
https://www.sim.ai
```

## Быстрый старт
<Steps>

<Step>
### Get your API key

Go to the Sim platform and navigate to **Settings**, then go to **Sim Keys** and click **Create**. See [Authentication](/api-reference/authentication) for details on key types.
</Step>

<Step>
### Find your workflow ID

Open a workflow in the Sim editor. The workflow ID is in the URL:

```
https://www.sim.ai/workspace/{workspaceId}/w/{workflowId}
```

You can also use the [List Workflows](/api-reference/workflows/listWorkflows) endpoint to get all workflow IDs in a workspace.
</Step>

<Step>
### Deploy your workflow

A workflow must be deployed before it can be executed via the API. Click the **Deploy** button in the editor toolbar, or use the dashboard to manage deployments.
</Step>

<Step>
### Make your first request

<Tabs items={['curl', 'TypeScript', 'Python']}>
  <Tab value="curl">

    ```bash
    curl -X POST https://www.sim.ai/api/workflows/{workflowId}/execute \
      -H "Content-Type: application/json" \
      -H "X-API-Key: YOUR_API_KEY" \
      -d '{"inputs": {}}'
    ```
  </Tab>
  <Tab value="TypeScript">

## Синхронное выполнение против асинхронного выполнения
По умолчанию, выполнение рабочих процессов является **синхронным**: API ждет завершения рабочего процесса и возвращает результат напрямую.

Для длительных рабочих процессов используйте **асинхронное выполнение**, передавая параметр ``async: true``:

```bash
curl -X POST https://www.sim.ai/api/workflows/{workflowId}/execute \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_API_KEY" \
  -d '{"inputs": {}, "async": true}'
```

Это возвращает результат немедленно с кодами `jobId` и `statusUrl`:

```json
{
  "success": true,
  "jobId": "job_abc123",
  "statusUrl": "https://www.sim.ai/api/jobs/job_abc123",
  "message": "Workflow execution started",
  "async": true
}
```

Проведите опрос[Узнать статус работы](/api-reference/workflows/getJobStatus)Узнать статус работы/api-reference/рабочие_процессы/получить_статус_задачиконец, пока статус не станет `completed` или `failed`:

```bash
curl https://www.sim.ai/api/jobs/{jobId} \
  -H "X-API-Key: YOUR_API_KEY"
```

<Callout type="info">
  Job status transitions follow: `queued` → `processing` → `completed` or `failed`. The `output` field is only present when status is `completed`.
</Callout>

## Формат ответа
Успешные ответы включают объект `output` с результатами вашей работы и объект `limits` с текущим лимитом использования и статусом.

```json
{
  "success": true,
  "output": {
    "result": "Hello, world!"
  },
  "limits": {
    "workflowExecutionRateLimit": {
      "sync": {
        "requestsPerMinute": 60,
        "maxBurst": 10,
        "remaining": 59,
        "resetAt": "2025-01-01T00:01:00Z"
      },
      "async": {
        "requestsPerMinute": 30,
        "maxBurst": 5,
        "remaining": 30,
        "resetAt": "2025-01-01T00:01:00Z"
      }
    },
    "usage": {
      "currentPeriodCost": 1.25,
      "limit": 50.00,
      "plan": "pro",
      "isExceeded": false
    }
  }
}
```

## Обработка ошибок
API использует стандартные коды состояния HTTP. Ответы об ошибках содержат понятное для человека сообщение `error`:

```json
{
  "error": "Workflow not found"
}
```

|  | Статус | Значение | Что делать? |  || --- | --- | --- |
|  | `400` | Неверные параметры запроса | Проверьте массив `details` на наличие конкретных ошибок в полях. |  ||  | `401` | Отсутствующий или недействительный ключ API | Проверьте заголовок `X-API-Key`. |  ||  | `403` | Доступ запрещен | Убедитесь, что у вас есть необходимые разрешения для доступа к этому ресурсу. |  ||  | `404` | Ресурс не найден | Убедитесь, что идентификатор существует и принадлежит вашему рабочему пространству. |  ||  | `429` | Превышен лимит запросов | Дождитесь указанного времени в заголовке "`Retry-After`" |  |
<Callout type="info">
  Use the [Get Usage Limits](/api-reference/usage/getUsageLimits) endpoint to check your current rate limit status and billing usage at any time.
</Callout>

## Ограничения по количеству запросов
Ограничения на количество запросов зависят от вашего тарифного плана и применяются отдельно для синхронных и асинхронных операций. Каждый ответ на запрос включает объект ``limits``, который показывает текущий статус ваших ограничений.

При ограничении скорости, API возвращает ответ с кодом `429` и заголовком `Retry-After`, который указывает, сколько секунд нужно подождать перед повторной попыткой.

## Разбиение на страницы
Конечные точки для списков (рабочие процессы, журналы, журналы аудита) используют **страничную навигацию с помощью курсора**:

```bash
# First page
curl "https://www.sim.ai/api/v1/logs?limit=20" \
  -H "X-API-Key: YOUR_API_KEY"

# Next page — use the nextCursor from the previous response
curl "https://www.sim.ai/api/v1/logs?limit=20&cursor=abc123" \
  -H "X-API-Key: YOUR_API_KEY"
```

Ответ включает поле ``nextCursor``. Если поле ``nextCursor`` отсутствует или имеет значение ``null``, вы перешли к последней странице.
