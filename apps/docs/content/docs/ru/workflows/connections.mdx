**Соединение** связывает один блок со следующим и задаёт порядок их выполнения. Раздел [Как блоки передают данные](/workflows/data-flow) описывает концепцию: когда блок выполнен, его output сохраняется под именем блока, и любой последующий блок читает его по ссылке. Эта страница — справочник по синтаксису **тегов соединения** и структурам output'ов, к которым вы будете обращаться.

## Синтаксис тегов

Тег соединения ссылается на значение из output'а предыдущего блока:

```
<имяБлока.путь.к.данным>
```

- `имяБлока` — это имя блока-источника. Имена блоков нормализуются перед сопоставлением: приводятся к нижнему регистру, пробелы удаляются. `<My Agent.content>` и `<myagent.content>` ссылаются на один и тот же блок.
- `путь.к.данным` — это путь к значению внутри output'а этого блока. Путь к полям чувствителен к регистру.

Тот же синтаксис с угловыми скобками также читает [переменные воркфлоу](/workflows/variables) (`<variable.name>`) и контекст цикла или параллельного выполнения (`<loop.index>`); тег соединения — это именно случай с output'ом блока.

Вы редко вводите тег вручную. В любом поле ввода наберите `<`, чтобы открыть выпадающий список доступных значений от предыдущих блоков и выбрать нужное, или перетащите значение из подключённого блока прямо в поле.

## Вложенные поля и поля-массивы

Используйте цепочку точечной нотации для доступа к вложенным объектам и индексы в квадратных скобках для массивов. Оба варианта комбинируются в одном пути:

```
<agent1.tokens.total>
<api1.data.results[0].id>
<api2.data.users[0].name>
```

Резолвер проходит по пути во время выполнения и подставляет значение. Несколько уровней индексации, например `items[0][1]`, также работают.

## Использование тегов в тексте и коде

Теги разрешаются во время выполнения и могут находиться в любом месте поля, вперемешку со статическим текстом:

```javascript
// В тексте
"Имя пользователя: <userBlock.name>"

// В JSON
{
  "userName": "<userBlock.name>",
  "orderTotal": <apiBlock.data.total>
}

// В блоке Function
const greeting = `Здравствуйте, ${"<userBlock.name>"}!`
const total = <apiBlock.data.total> * 1.1
```

Значения форматируются в соответствии с контекстом. В блоке [Function](/workflows/blocks/function) разрешённое значение становится литералом кода: строки заключаются в кавычки, объекты становятся JSON, а `null` остаётся `null` — готовым к использованию в JavaScript или Python. В других блоках объекты преобразуются в JSON-строку, а примитивы становятся обычными строками.

<Callout type="warning">
В числовом контексте убедитесь, что значение по ссылке действительно является числом. Тег, разрешающийся в текст, не будет вести себя как число.
</Callout>

Когда блок, на который ссылаются, не выполнялся на текущем пути — например, блок на невыбранной ветке — его тег разрешается в пустое значение. В большинстве блоков это пустая строка; в блоках Function — `null`.

## Типовые output'ы блоков

Путь в теге зависит от того, что произвёл блок-источник. Ниже приведены стандартные структуры для блоков, на которые вы будете ссылаться чаще всего. Панель output'а показывает актуальную структуру для любого блока.

<Tabs items={['Agent', 'API', 'Function', 'Condition', 'Router', 'Evaluator']}>
  <Tab>
    ```json
    {
      "content": "Сгенерированный текстовый ответ",
      "model": "claude-sonnet-4-6",
      "tokens": { "input": 120, "output": 85, "total": 205 },
      "toolCalls": []
    }
    ```

    `content` — основной ответ. `tokens` — объект с количеством использованных токенов. `toolCalls` — список инструментов, вызванных агентом, с их входными данными и результатами. При настроенном формате ответа поля вашей схемы отображаются как отдельные output'ы.
  </Tab>
  <Tab>
    ```json
    {
      "data": "Тело ответа",
      "status": 200,
      "headers": { "content-type": "application/json" }
    }
    ```

    `data` — тело ответа, может быть любого типа. `status` — HTTP-код состояния. `headers` — заголовки ответа.
  </Tab>
  <Tab>
    ```json
    {
      "result": "Возвращаемое значение функции",
      "stdout": "Вывод в консоль"
    }
    ```

    `result` — то, что вернула ваша функция, может быть любого типа. `stdout` — всё, что код вывел в лог.
  </Tab>
  <Tab>
    ```json
    {
      "conditionResult": true,
      "selectedPath": {
        "blockId": "2acd9007-27e8-4510-a487-73d3b825e7c1",
        "blockType": "agent",
        "blockTitle": "Follow-up Agent"
      },
      "selectedOption": "condition-1"
    }
    ```

    `conditionResult` — булев результат. `selectedPath` описывает следующий блок. `selectedOption` — идентификатор совпавшего условия.
  </Tab>
  <Tab>
    ```json
    {
      "content": "Решение маршрутизации",
      "model": "claude-sonnet-4-6",
      "tokens": { "input": 120, "output": 85, "total": 205 },
      "selectedPath": {
        "blockId": "2acd9007-27e8-4510-a487-73d3b825e7c1",
        "blockType": "agent",
        "blockTitle": "Customer Service Agent"
      }
    }
    ```

    `content` — текст решения маршрутизации. `selectedPath` описывает выбранный целевой блок.
  </Tab>
  <Tab>
    ```json
    {
      "content": "Сводка оценки",
      "model": "claude-sonnet-4-6",
      "tokens": { "input": 120, "output": 85, "total": 205 },
      "metric1": 8.5,
      "metric2": 7.2
    }
    ```

    `content` — сводка оценки. Каждая определённая вами метрика отображается как отдельное поле с числовой оценкой.
  </Tab>
</Tabs>

`result` блока Function, `data` блока API и Agent с форматом ответа — все возвращают то, что произвели ваш код, удалённый сервис или ваша схема, поэтому проверяйте панель output'а при разработке, чтобы подтвердить путь, на который вы ссылаетесь.
