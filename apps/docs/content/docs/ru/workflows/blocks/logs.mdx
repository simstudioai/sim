Блок Logs запрашивает логи запусков воркфлоу в текущем рабочем пространстве и получает полные детали отдельных запусков — те же данные, которые вы видите на странице Logs, доступные вашим воркфлоу.

## Операции

## Типичный сценарий

Запросите интересующие вас запуски, затем выполните цикл по возвращённым идентификаторам и получите детали каждого:

1. **Query Logs** с `Status: error` и `Time Range: Past 24 hours` → `<logs1.runIds>`
2. Цикл по идентификаторам и вызов **Get Run Details** → проверьте `<logs2.traceSpans>`, чтобы найти упавший блок
3. Действуйте — отправьте сводку в Slack, создайте тикет или передайте трассировку блоку Agent для диагностики

Это естественно сочетается с [триггером Sim](/workflows/triggers/sim): триггер передаёт вам `runId`, вызвавший событие, а Get Run Details даёт полную трассировку.

<Callout type="info">
  Блок всегда работает с текущим рабочим пространством. Стоимость выражается в кредитах как для фильтра стоимости, так и для вывода стоимости.
</Callout>
