**Блок Credential** передаёт последующему блоку OAuth-учётные данные для использования, не раскрывая секрет. У него две операции: **Select Credential** выдаёт ссылку на идентификатор одних учётных данных для использования другими блоками; **List Credentials** возвращает все OAuth-учётные данные рабочего пространства (опционально отфильтрованные по провайдеру) в виде массива для итерации.

<Callout>
  Блок Credential выдаёт **ссылки на идентификаторы** учётных данных, а не секреты. Последующие блоки получают идентификатор и безопасно разрешают фактический OAuth-токен во время своего выполнения.

## Конфигурация

### Operation

| Значение | Описание |
|---|---|
| **Select Credential** | Выбрать одни OAuth-учётные данные и выдать их ссылку — используйте для передачи одних учётных данных в последующие блоки |
| **List Credentials** | Вернуть все OAuth-учётные данные рабочего пространства в виде массива — используйте с циклом ForEach |

### Credential (операция Select)

Выберите OAuth-учётные данные из вашего рабочего пространства. Выпадающий список показывает все подключённые OAuth-аккаунты (Google, GitHub, Slack и т.д.).

В расширенном режиме вставьте идентификатор учётных данных напрямую. Вы можете скопировать идентификатор со страницы настроек учётных данных рабочего пространства.

### Provider (операция List)

Отфильтруйте возвращаемые OAuth-учётные данные по провайдеру. Выберите одного или нескольких провайдеров из выпадающего списка — отображаются только те провайдеры, для которых у вас есть учётные данные. Оставьте пустым, чтобы вернуть все OAuth-учётные данные.

| Пример | Возвращает |
|---|---|
| Gmail | Только учётные данные Gmail |
| Slack | Только учётные данные Slack |
| Gmail + Slack | Учётные данные Gmail и Slack |

## Как подключить блок Credential

### Select Credential

1. Добавьте блок **Credential** и выберите OAuth-учётные данные из списка
2. В последующем блоке переключитесь в **расширенный режим** для поля учётных данных
3. Введите `<имяБлокаCredential.credentialId>` как значение

Та же ссылка работает для любого OAuth-блока. В поле учётных данных блока Gmail или Slack (расширенный режим) введите `<myCredential.credentialId>`.

### List Credentials

1. Добавьте блок **Credential**, установите Operation в **List Credentials**
2. Опционально выберите одного или нескольких **провайдеров** для сужения результатов (отображаются только ваши подключённые провайдеры)
3. Подключите `<имяБлокаCredential.credentials>` к циклу **ForEach** как источник элементов
4. Внутри цикла ссылайтесь на `<loop.currentItem.credentialId>` в полях учётных данных последующих блоков

## Рекомендации

- **Определите один раз, ссылайтесь многократно**: Когда пять блоков используют один и тот же аккаунт Google, используйте один блок Credential и подключите все пять к `<credential.credentialId>` вместо выбора аккаунта пять раз
- **Output'ы безопасны для логов**: Output `credentialId` — это UUID-ссылка, а не секрет. Его безопасно просматривать в логах выполнения
- **Используйте для переключения окружений**: Совместите с блоком Condition для маршрутизации к production- или staging-OAuth-учётным данным на основе переменной воркфлоу
- **Требуется расширенный режим**: Последующие блоки должны быть в расширенном режиме для поля учётных данных, чтобы принять динамическую ссылку
- **Используйте List + ForEach для разветвления**: Когда нужно выполнить одно действие для всех аккаунтов провайдера, List Credentials естественно подаётся в цикл ForEach
- **Сужайте по провайдеру**: Используйте мультиселект провайдера для фильтрации по конкретным сервисам — отображаются только те провайдеры, для которых у вас есть учётные данные
