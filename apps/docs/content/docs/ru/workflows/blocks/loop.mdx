**Блок Loop** — это контейнер, который многократно выполняет блоки внутри себя — по каждому элементу коллекции, фиксированное количество раз или пока выполняется условие. Поместите блоки внутрь, и они выполняются один раз за итерацию. Итерации выполняются **одна за другой**; для параллельной работы используйте блок [Parallel](/workflows/blocks/parallel).

## Конфигурация

Задайте тип цикла, затем нужное ему значение:

- **Iterations** — сколько раз выполнить (циклы For)
- **Collection** — массив или объект для итерации (циклы ForEach)
- **Condition** — булево выражение для проверки (циклы While и Do-While)

## Ссылки на данные цикла

Ссылки различаются внутри цикла и после него.

<Tabs items={['Внутри цикла', 'После цикла']}>
  <Tab>
    Внутри цикла читайте текущую итерацию через `<loop.*>`:

    - `<loop.index>` — номер итерации, начиная с 0
    - `<loop.currentItem>` — текущий элемент (только ForEach)
    - `<loop.items>` — полная коллекция (только ForEach)

    ```javascript
    const idx = <loop.index>;        // 0, 1, 2, ...
    const item = <loop.currentItem>; // текущий элемент
    ```

    <Callout type="info">
      Эти ссылки доступны только блокам внутри контейнера цикла.
    </Callout>
  </Tab>
  <Tab>
    После цикла читайте собранные результаты по имени блока цикла — не `<loop.*>`:

    ```javascript
    // цикл с именем "Process Items"
    const all = <processitems.results>; // [result1, result2, ...]
    ```

    <Callout type="info">
      Имя нормализуется — нижний регистр, без пробелов. `<loop.*>` работает только внутри.
    </Callout>
  </Tab>
</Tabs>

## Примеры

Цикл ForEach оценивает каждый отзыв и собирает результаты; после завершения цикла Agent обобщает `<loop.results>`. Тот же контейнер содержит блоки для любого типа цикла:

- **Обработка результатов API** — `API (Fetch) → Loop (ForEach) → Agent (Analyze) → Function (Store)`
- **Генерация вариантов** — `Loop (5×) → Agent (Generate) → Evaluator (Score) → Function (Select best)`
- **Счётчик с While** — `Variables (i=0) → Loop (While i<10) → Agent (Process) → Variables (i++)`

## Вложенность и ограничения

<Callout type="info">
  Контейнеры вкладываются. Вы можете размещать циклы внутри циклов, параллельные блоки внутри циклов и любые комбинации для построения многомерных воркфлоу.
</Callout>

Циклы For и ForEach ограничены **1 000 итераций**. Циклы While и Do-While выполняются, пока условие не станет ложным, без жёсткого ограничения — убедитесь, что условие в конечном итоге станет ложным, чтобы цикл завершился.

## Рекомендации

- **Используйте ForEach для коллекций.** Когда у вас массив или объект, ForEach понятнее, чем цикл For с ручной индексацией.
- **Держите количество итераций разумным.** Каждая итерация увеличивает общее время выполнения, поскольку цикл выполняется последовательно.
- **Используйте Parallel, когда порядок не важен.** Независимая работа над коллекцией выполняется быстрее в блоке [Parallel](/workflows/blocks/parallel).
- **Гарантируйте выход.** Для циклов While и Do-While убедитесь, что условие станет ложным.
