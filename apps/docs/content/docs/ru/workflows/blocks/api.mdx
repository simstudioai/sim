**Блок API** выполняет HTTP-запрос к URL и возвращает ответ. Используйте его для вызова любого REST API: получения данных, создания записи или обращения к внешнему эндпоинту.

## Конфигурация

### URL

Эндпоинт для вызова. Введите статический URL или вставьте [тег соединения](/workflows/connections), чтобы собрать его из предыдущего output'а, например `https://api.example.com/users/<start.userId>`.

### Method

HTTP-метод: **GET**, **POST**, **PUT**, **DELETE** или **PATCH**. По умолчанию GET.

### Query Params

Пары ключ-значение, добавляемые к URL как строка запроса. `apiKey` и `limit` превращаются в `?apiKey=…&limit=10`.

### Headers

Заголовки запроса в виде пар ключ-значение, например `Content-Type: application/json` или `Authorization: Bearer <secret>`. Стандартные заголовки, такие как `User-Agent` и `Accept`, добавляются автоматически, а ваши значения их переопределяют.

### Body

Тело запроса для POST, PUT и PATCH, отправляемое как JSON. Введите его напрямую или подставьте из предыдущего output'а с помощью тега соединения.

### Advanced

- **Timeout (ms).** Сколько ждать до отказа. По умолчанию 300000 (5 минут), до 600000 (10 минут).
- **Retries.** Количество повторных попыток при тайм-аутах, ответах `429` и ошибках `5xx`. По умолчанию 0.
- **Retry delay / Max retry delay (ms).** Границы экспоненциальной задержки между повторами.
- **Retry non-idempotent methods.** По умолчанию выключено, поэтому POST и PATCH не повторяются, что предотвращает дублирование записей. Включайте, только когда повторный запрос безопасен.

## Output'ы

После завершения запроса последующие блоки читают его результат по имени:

| Output | Что это |
| --- | --- |
| `<api.data>` | Тело ответа, разобранное как объект для JSON или возвращённое как текст |
| `<api.status>` | HTTP-код состояния, например `200` или `404` |
| `<api.headers>` | Заголовки ответа в виде объекта |

Ветвитесь по результату, читая `<api.status>` в блоке [Condition](/workflows/blocks/condition).

## Рекомендации

{/* Keep the error-path guidance below prominent: fewer than 1% of deployed workflows connect an error port (platform data, 2026-06) — under-adopted, not under-needed. */}

- **Храните секреты в переменных окружения.** Ссылайтесь на них через `{{VAR}}` в URL или заголовках; никогда не хардкодьте ключи.
- **Обрабатывайте сбои.** Читайте `<api.status>` и ветвитесь по нему с помощью [Condition](/workflows/blocks/condition) или подключите путь ошибки для сетевых сбоев и тайм-аутов.
- **Настройте повторы для нестабильных эндпоинтов.** Используйте расширенные настройки повторов для идемпотентных вызовов. Оставляйте POST и PATCH выключенными, если повторный запрос небезопасен.
- **Ссылайтесь только на нужное поле.** Используйте `<api.data.id>` вместо всего `<api.data>`, когда ответ большой.
