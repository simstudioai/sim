**Переменная** — это значение, которое вы задаёте сами и переиспользуете во всём воркфлоу. В Sim есть два вида. **Переменные воркфлоу** хранят данные во время запуска, такие как счётчик или объект конфигурации, и вы ссылаетесь на них через `<variable.name>`. **Переменные окружения** хранят секреты и конфигурацию, такие как API-ключи, и вы ссылаетесь на них через `{{KEY}}`.

## Один синтаксис, именованные источники

Всё в угловых скобках работает по одному механизму: `<имя.путь>` читает значение по имени во время выполнения. Первый сегмент говорит, откуда берётся значение. **`variable` — это буквально** — каждая переменная воркфлоу читается через него. [Тег соединения](/workflows/connections) начинается с собственного имени блока, а внутри контейнера `loop` и `parallel` зарезервированы для его контекста. `{{KEY}}` — это единственный другой синтаксис, для переменных окружения.

| Синтаксис | Читает | Пример |
|--------|-------|---------|
| `<loop.index>`, `<parallel.index>` | Текущая итерация [Loop](/workflows/blocks/loop) или экземпляр [Parallel](/workflows/blocks/parallel) внутри этого контейнера | `<loop.currentItem>` |
| `<variable.name>` | Переменная воркфлоу | `<variable.counter>` |
| `{{KEY}}` | Переменная окружения | `{{OPENAI_API_KEY}}` |
| `<имяБлока.поле>` | Output блока — тег соединения, не переменная | `<agent.content>` |

Таблица приведена в порядке разрешения: когда ссылка в угловых скобках может соответствовать более чем одному источнику — скажем, блоку с именем `variable` — контекст цикла и параллельного выполнения побеждает, затем переменные воркфлоу, затем output'ы блоков. (`{{KEY}}` не может конфликтовать; это другой синтаксис.) Если ничего не совпадает, исходная ссылка остаётся на месте без подстановки, что обычно ломает блок, который её читает — поэтому давайте переменным и блокам разные имена.

Наберите `<` в любом текстовом поле блока, чтобы открыть выпадающий список и просмотреть всё, на что можно сослаться.

## Переменные воркфлоу

Переменные воркфлоу — это глобальное хранилище, которое любой блок может читать или обновлять во время запуска. Откройте панель через **⋯ → Variables** в правом верхнем углу редактора. Каждая переменная имеет **имя**, **тип** и **значение**.

Ссылайтесь на переменную везде, где поле принимает ввод. Для переменной типа object или array используйте точечную нотацию для доступа к вложенному значению, например `<variable.config.timeout>`. Сопоставление имён игнорирует регистр и пробелы, поэтому переменная с именем `My Counter` разрешается через `<variable.mycounter>` — но имя не может содержать точку: всё после первой точки читается как путь к значению, поэтому `<variable.config.timeout>` всегда означает поле `timeout` объекта `config`, а не переменную с именем `config.timeout`. Единообразные имена, такие как `camelCase`, наиболее понятны.

### Область видимости

Переменные воркфлоу глобальны: каждый блок воркфлоу может читать и изменять их. Используйте [блок Variables](/workflows/blocks/variables) для обновления значения во время выполнения.

Каждый запуск начинается заново со значений, определённых на панели. Изменение, сделанное во время запуска, видно последующим блокам в этом же запуске, но оно никогда не изменяет сохранённое начальное значение для будущих запусков.

## Переменные окружения

Переменные окружения хранят конфиденциальные значения, такие как API-ключи, токены и конфигурацию, которые никогда не должны появляться в логах или на холсте. Создайте их в разделе **Settings → Secrets**, добавив пару ключ-значение.

Ссылайтесь на них с помощью двойных фигурных скобок в любом поле блока, включая системные промпты Agent и код блока Function. Значение подставляется до выполнения блока, а API-ключи скрываются в логах.

```
{{API_KEY}}
```

<Callout type="info">
  Имена переменных окружения должны начинаться с буквы или подчёркивания и содержать только буквы, цифры и подчёркивания, например `MY_API_KEY`.
</Callout>

### Личные и рабочие переменные

| Область | Видна | Используйте для |
|-------|-----------|---------|
| **Рабочая (Workspace)** | Всем участникам рабочего пространства, включая внешних участников | Общие API-ключи, командная конфигурация |
| **Личная (Personal)** | Только вам | Ваши личные токены, учётные данные разработчика |

Когда рабочая и личная переменные имеют одинаковый ключ, приоритет имеет значение рабочей переменной.

Используйте переменную окружения для сырого API-ключа или значения конфигурации. Когда сервису требуется OAuth или управляемое обновление токенов (Slack, GitHub, Google), используйте [учётные данные](/platform/credentials), выбираемые из выпадающего списка в настройках блока.
