[Algolia](https://www.algolia.com/) — это мощная хостинговая поисковая платформа, которая позволяет разработчикам и командам создавать быстрый и релевантный поиск в своих приложениях и на веб-сайтах. Algolia предоставляет полнотекстовый, фасетный и фильтруемый поиск, а также аналитику и расширенные возможности ранжирования.

С помощью Algolia вы можете:

- **Обеспечить молниеносный поиск**: Предоставлять мгновенные результаты поиска по мере ввода с устойчивостью к опечаткам и поддержкой синонимов
- **Управлять и обновлять записи**: Легко добавлять, обновлять или удалять объекты/записи в ваших индексах
- **Выполнять расширенную фильтрацию**: Использовать фильтры, фасеты и пользовательское ранжирование для уточнения и организации результатов поиска
- **Настраивать параметры индекса**: Регулировать релевантность, ранжирование, атрибуты для поиска и многое другое для оптимизации пользовательского опыта
- **Уверенно масштабироваться**: Algolia обрабатывает огромный трафик и объёмы данных с глобально распределённой инфраструктурой
- **Получать аналитику**: Отслеживать аналитику, поисковые паттерны и вовлечённость пользователей

В Sim интеграция Algolia позволяет вашим агентам искать, управлять и настраивать индексы Algolia непосредственно в ваших рабочих процессах. Используйте Algolia для динамического исследования данных, автоматизации обновления записей, выполнения пакетных операций и многого другого — всё из одного инструмента в вашем рабочем пространстве.

## Usage Instructions

Интегрируйте Algolia в ваш рабочий процесс. Ищите в индексах, управляйте записями (добавление, обновление, удаление, просмотр), настраивайте параметры индекса и выполняйте пакетные операции.



## Actions

### `algolia_search`

Поиск в индексе Algolia

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `applicationId` | string | Yes | ID приложения Algolia |
| `apiKey` | string | Yes | API-ключ Algolia |
| `indexName` | string | Yes | Имя индекса Algolia для поиска |
| `query` | string | Yes | Текст поискового запроса |
| `hitsPerPage` | number | No | Количество результатов на странице \(по умолчанию: 20\) |
| `page` | number | No | Номер страницы для получения \(по умолчанию: 0\) |
| `filters` | string | No | Строка фильтра \(например, "category:electronics AND price &lt; 100"\) |
| `attributesToRetrieve` | string | No | Список атрибутов для получения через запятую |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `hits` | array | Массив совпадающих записей |
|   ↳ `objectID` | string | Уникальный идентификатор записи |
|   ↳ `_highlightResult` | object | Выделенные атрибуты, соответствующие запросу. Каждый атрибут имеет value, matchLevel \(none, partial, full\) и matchedWords |
|   ↳ `_snippetResult` | object | Фрагментированные атрибуты, соответствующие запросу. Каждый атрибут имеет value и matchLevel |
|   ↳ `_rankingInfo` | object | Информация о ранжировании для результата. Присутствует только при включённом getRankingInfo |
|     ↳ `nbTypos` | number | Количество опечаток в совпадении запроса |
|     ↳ `firstMatchedWord` | number | Позиция первого совпавшего слова |
|     ↳ `geoDistance` | number | Расстояние в метрах для результатов геопоиска |
|     ↳ `nbExactWords` | number | Количество точно совпавших слов |
|     ↳ `userScore` | number | Пользовательская оценка ранжирования |
|     ↳ `words` | number | Количество совпавших слов |
| `nbHits` | number | Общее количество совпадающих результатов |
| `page` | number | Текущий номер страницы \(начиная с нуля\) |
| `nbPages` | number | Общее количество доступных страниц |
| `hitsPerPage` | number | Количество результатов на странице \(1-1000, по умолчанию 20\) |
| `processingTimeMS` | number | Время обработки на стороне сервера в миллисекундах |
| `query` | string | Выполненный поисковый запрос |
| `parsedQuery` | string | Строка запроса после нормализации и удаления стоп-слов |
| `facets` | object | Подсчёты фасетов, сгруппированные по имени фасета, каждый содержит пары значение-количество |
| `facets_stats` | object | Статистика \(min, max, avg, sum\) для числовых фасетов |
| `exhaustive` | object | Флаги полноты для facetsCount, facetValues, nbHits, rulesMatch и typo |

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `applicationId` | string | Yes | ID приложения Algolia |

| `createdAt` | string | Метка времени создания записи \(присутствует только при автоматической генерации objectID\) |

### `algolia_browse_records`

Просматривать и итерировать все записи в индексе Algolia с использованием курсорной пагинации

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `applicationId` | string | Yes | ID приложения Algolia |
| `apiKey` | string | Yes | API-ключ Algolia \(должен иметь ACL browse\) |
| `indexName` | string | Yes | Имя индекса Algolia для просмотра |
| `query` | string | No | Поисковый запрос для фильтрации просматриваемых записей |
| `filters` | string | No | Строка фильтра для сужения результатов |
| `attributesToRetrieve` | string | No | Список атрибутов для получения через запятую |
| `hitsPerPage` | number | No | Количество результатов на странице \(по умолчанию: 1000, макс: 1000\) |
| `cursor` | string | No | Курсор из предыдущего ответа browse для пагинации |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `hits` | array | Массив записей из индекса \(до 1000 за запрос\) |
|   ↳ `objectID` | string | Уникальный идентификатор записи |
| `cursor` | string | Непрозрачная строка курсора для получения следующей страницы результатов. Отсутствует, когда результатов больше нет. |
| `nbHits` | number | Общее количество записей, соответствующих критериям просмотра |
| `page` | number | Текущий номер страницы \(начиная с нуля\) |
| `nbPages` | number | Общее количество доступных страниц |
| `hitsPerPage` | number | Количество результатов на странице \(1-1000, по умолчанию 1000 для browse\) |
| `processingTimeMS` | number | Время обработки на стороне сервера в миллисекундах |

### `algolia_delete_by_filter`

Удалить все записи, соответствующие фильтру, из индекса Algolia

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `applicationId` | string | Yes | ID приложения Algolia |
| `apiKey` | string | Yes | Admin API-ключ Algolia \(должен иметь ACL deleteIndex\) |
| `indexName` | string | Yes | Имя индекса Algolia |
| `filters` | string | No | Выражение фильтра для выбора записей для удаления \(например, "category:outdated"\) |
| `facetFilters` | json | No | Массив фасетных фильтров \(например, \["brand:Acme"\]\) |
| `numericFilters` | json | No | Массив числовых фильтров \(например, \["price &gt; 100"\]\) |

| `insideBoundingBox` | json | No | Координаты ограничивающего прямоугольника как \[\[lat1, lng1, lat2, lng2\]\] для фильтра геопоиска |
| `insidePolygon` | json | No | Координаты многоугольника как \[\[lat1, lng1, lat2, lng2, lat3, lng3, ...\]\] для фильтра геопоиска |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `taskID` | number | ID задачи Algolia для отслеживания операции удаления по фильтру |
| `updatedAt` | string | Метка времени выполнения операции |
