## Инструкции по использованию

Интегрируйте UptimeRobot в ваш рабочий процесс. Create and manage monitors, inspect incidents, schedule maintenance windows, manage alert contacts, and publish public status pages using the UptimeRobot v3 API.



## Действия

### `uptimerobot_list_monitors`

Получение списка monitors in your UptimeRobot account, with optional filters and pagination

#### Вход

| Параметр | Тип | Обязательный | Описание |
| --------- | ---- | -------- | ----------- |
| `apiKey` | string | Да | UptimeRobot Ключ API |
| `limit` | number | Нет | Number of monitors per page \(1-200, по умолчанию 50\) |
| `status` | string | Нет | Comma-separated statuses to filter by \(PAUSED, STARTED, UP, LOOKS_DOWN, DOWN\) |
| `name` | string | Нет | Partial friendly-name filter |
| `url` | string | Нет | Partial URL filter |
| `tags` | string | Нет | Comma-separated tags to filter by \(case-sensitive, OR logic\) |
| `groupId` | number | Нет | Monitor group ID to filter by |
| `cursor` | number | Нет | Pagination cursor returned by a previous request |

#### Выход

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `monitors` | array | Получение списка of monitors |
|   ↳ `id` | number | Monitor ID |
|   ↳ `friendlyName` | string | Friendly name of the monitor |
|   ↳ `url` | string | Monitored URL or host |
|   ↳ `type` | string | Monitor type \(HTTP, KEYWORD, PING, PORT, HEARTBEAT, DNS, API, UDP\) |
|   ↳ `status` | string | Current status \(UP, DOWN, PAUSED, etc.\) |
|   ↳ `interval` | number | Check interval in seconds |
|   ↳ `timeout` | number | Check timeout in seconds |
|   ↳ `port` | number | Port for Port/UDP monitors |
|   ↳ `keywordType` | string | Keyword match type for Keyword monitors |
|   ↳ `keywordValue` | string | Keyword to match for Keyword monitors |
|   ↳ `httpMethodType` | string | HTTP method used for the check |
|   ↳ `authType` | string | HTTP authentication method |
|   ↳ `successHttpResponseCodes` | array | HTTP response codes treated as success |
|   ↳ `checkSSLErrors` | boolean | Whether SSL/domain expiration errors are checked |
|   ↳ `followRedirections` | boolean | Whether redirects are followed |
|   ↳ `sslExpirationReminder` | boolean | Whether SSL expiration reminders are enabled |
|   ↳ `domainExpirationReminder` | boolean | Whether domain expiration reminders are enabled |
|   ↳ `responseTimeThreshold` | number | Response time threshold in milliseconds |
|   ↳ `currentStateDuration` | number | Seconds spent in the current state |
|   ↳ `lastIncidentId` | string | ID of the most recent incident |
|   ↳ `groupId` | number | Monitor group ID \(0 if ungrouped\) |
|   ↳ `createDateTime` | string | When the monitor was created |
|   ↳ `tags` | array | Tags assigned to the monitor |
|     ↳ `id` | number | Tag ID |
|     ↳ `name` | string | Tag name |
|     ↳ `color` | string | Tag color |
|   ↳ `assignedAlertContacts` | array | Alert contacts assigned to the monitor |
|     ↳ `alertContactId` | number | Alert contact ID |
|     ↳ `threshold` | number | Notification delay threshold in minutes |
|     ↳ `recurrence` | number | Repeat notification interval in minutes |
|   ↳ `lastIncident` | object | Details of the most recent incident |
|     ↳ `id` | string | Incident ID |
|     ↳ `status` | string | Incident status |
|     ↳ `cause` | number | Incident cause code |
|     ↳ `reason` | string | Incident reason |
|     ↳ `startedAt` | string | When the incident started |
|     ↳ `duration` | number | Incident duration in seconds |
| `nextLink` | string | URL for the next page of results, or null on the last page |

### `uptimerobot_create_monitor`

Создание нового monitor in UptimeRobot

#### Вход

| Параметр | Тип | Обязательный | Описание |
| --------- | ---- | -------- | ----------- |
| `apiKey` | string | Да | UptimeRobot Ключ API |
| `friendlyName` | string | Да | Friendly name of the monitor |
| `type` | string | Да | Monitor type: HTTP, KEYWORD, PING, PORT, HEARTBEAT, DNS, API, or UDP |
| `url` | string | Нет | URL or host to monitor \(not required for Heartbeat monitors\) |
| `interval` | number | Да | Check interval in seconds \(minimum 30\) |
| `timeout` | number | Нет | Check timeout in seconds, 0-60 \(HTTP, Keyword and Port monitors only\) |
| `port` | number | Нет | Port to check, 1-65535 \(required for Port and UDP monitors\) |
| `keywordType` | string | Нет | Keyword match type for Keyword monitors: ALERT_EXISTS or ALERT_NOT_EXISTS |
| `keywordValue` | string | Нет | Keyword to look for \(Keyword monitors only\) |
| `keywordCaseType` | number | Нет | Keyword case sensitivity: 0 \(case-sensitive\) or 1 \(case-insensitive\) |
| `httpMethodType` | string | Нет | HTTP method: HEAD, GET, POST, PUT, PATCH, DELETE, or OPTIONS \(defaults to HEAD\) |
| `authType` | string | Нет | HTTP authentication: NONE, HTTP_BASIC, DIGEST, or BEARER |
| `httpUsername` | string | Нет | Username for HTTP authentication |
| `httpPassword` | string | Нет | Password for HTTP authentication |
| `gracePeriod` | number | Нет | Grace period in seconds, 0-86400 \(Heartbeat monitors only\) |
| `successHttpResponseCodes` | string | Нет | Comma-separated success HTTP response codes (например, "2xx,3xx"\) |
