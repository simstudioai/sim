[AWS AppConfig](https://docs.aws.amazon.com/appconfig/) — это возможность AWS Systems Manager, которая позволяет создавать, управлять и развёртывать конфигурацию приложений и флаги функций независимо от вашего кода. Вы организуете конфигурацию в приложения, окружения и профили конфигурации, храните версионные данные конфигурации и разворачиваете их постепенно с помощью стратегий развёртывания, которые могут отслеживать и автоматически откатывать при ошибках.

С помощью интеграции AppConfig вы можете:

- **Получать конфигурацию**: Извлекать последнюю развёрнутую конфигурацию для приложения, окружения и профиля во время выполнения, используя плоскость данных AppConfig для управления флагами функций и динамическими настройками в ваших рабочих процессах
- **Управлять приложениями**: Просматривать существующие приложения или создавать новые для группировки связанной конфигурации
- **Управлять окружениями**: Просматривать или создавать окружения \(такие как `development`, `staging` и `production`\), в которые разворачивается конфигурация
- **Управлять профилями конфигурации**: Просматривать или создавать профили, описывающие, где хранится конфигурация — размещённая в AppConfig, SSM или S3 — и является ли она свободной формой или профилем флагов функций
- **Версионировать размещённую конфигурацию**: Создавать новую версию размещённой конфигурации из JSON, YAML или текстового документа или читать содержимое конкретной версии
- **Выполнять развёртывания**: Запускать развёртывание версии конфигурации в окружение с выбранной стратегией развёртывания, проверять статус и историю развёртывания, а также останавливать выполняющееся развёртывание

В Sim интеграция AppConfig позволяет вашим агентам читать актуальную конфигурацию для ветвления поведения, публиковать новые версии конфигурации и оркестрировать безопасные, постепенные развёртывания. Это естественно сочетается с CloudWatch для мониторинга состояния развёртывания и Slack для шлюзов утверждения и оповещений о развёртывании, обеспечивая сквозную автоматизацию конфигурации и флагов функций.

Аутентификация использует AWS access key ID и secret access key. Связанный принципал IAM должен иметь соответствующие разрешения `appconfig:*` \(например, `appconfig:GetLatestConfiguration` и `appconfig:StartConfigurationSession` для получения и `appconfig:StartDeployment` для развёртываний\).

## Usage Instructions

Интегрируйте AWS AppConfig в рабочие процессы. Управляйте приложениями, окружениями и профилями конфигурации, создавайте и читайте версии размещённой конфигурации, выполняйте и проверяйте развёртывания, а также получайте последнюю развёрнутую конфигурацию во время выполнения. Требуется AWS access key и secret access key.



## Actions

### `appconfig_get_configuration`

Получить последнюю развёрнутую конфигурацию для приложения, окружения и профиля AppConfig

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `region` | string | Yes | Регион AWS \(например, us-east-1\) |
| `accessKeyId` | string | Yes | AWS access key ID |
| `secretAccessKey` | string | Yes | AWS secret access key |
| `applicationId` | string | Yes | ID или имя приложения для получения конфигурации |
| `environmentId` | string | Yes | ID или имя окружения для получения конфигурации |
| `configurationProfileId` | string | Yes | ID или имя профиля конфигурации для получения |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `configuration` | string | Содержимое развёрнутой конфигурации |
| `contentType` | string | Тип содержимого конфигурации |
| `versionLabel` | string | Метка полученной версии конфигурации |
