## Инструкции по использованию

Отправляйте письма через любой SMTP-сервер (Gmail, Outlook, собственные серверы и т.д.). Настройте параметры подключения SMTP и отправляйте письма с полным контролем над содержимым, получателями и вложениями.



## Действия

### `smtp_send_mail`

Отправка писем через SMTP-сервер

#### Вход

| Параметр | Тип | Обязательный | Описание |
| --------- | ---- | -------- | ----------- |
| `smtpHost` | string | Да | Имя хоста SMTP-сервера \(например, smtp.gmail.com\) |
| `smtpPort` | number | Да | Порт SMTP-сервера \(587 для TLS, 465 для SSL\) |
| `smtpUsername` | string | Да | Имя пользователя для аутентификации SMTP |
| `smtpPassword` | string | Да | Пароль для аутентификации SMTP |
| `smtpSecure` | string | Да | Протокол безопасности \(TLS, SSL или None\) |
| `from` | string | Да | Адрес электронной почты отправителя |
| `to` | string | Да | Адрес электронной почты получателя |
| `subject` | string | Да | Тема письма |
| `body` | string | Да | Содержимое письма |
| `contentType` | string | Нет | Тип содержимого \(text или html\) |
| `fromName` | string | Нет | Отображаемое имя отправителя |
| `cc` | string | Нет | Получатели копии \(через запятую\) |
| `bcc` | string | Нет | Получатели скрытой копии \(через запятую\) |
| `replyTo` | string | Нет | Адрес для ответа |
| `attachments` | file[] | Нет | Файлы для прикрепления к письму |

#### Выход

| Параметр | Тип | Описание |
| --------- | ---- | ----------- |
| `success` | boolean | Успешно ли отправлено письмо |
| `messageId` | string | Идентификатор сообщения от SMTP-сервера |
| `to` | string | Адрес электронной почты получателя |
| `subject` | string | Тема письма |
| `error` | string | Сообщение об ошибке, если отправка не удалась |
