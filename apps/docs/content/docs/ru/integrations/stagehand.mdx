[Stagehand](https://stagehand.com) — это инструмент, который позволяет как извлекать структурированные данные с веб-страниц, так и выполнять автономную веб-автоматизацию с использованием Browserbase и современных LLM (OpenAI или Anthropic).

Stagehand предлагает две основные возможности в Sim:

- **stagehand_extract**: Извлечение структурированных данных с одной веб-страницы. Вы указываете, что хотите получить (схему), и AI извлекает и разбирает данные в этой форме со страницы. Это лучше всего подходит для извлечения списков, полей или объектов, когда вы точно знаете, какая информация вам нужна и где её получить.

- **stagehand_agent**: Запуск автономного веб-агента, способного выполнять многошаговые задачи, взаимодействовать с элементами, перемещаться между страницами и возвращать структурированные результаты. Это гораздо более гибкий вариант: агент может выполнять такие действия, как вход в систему, поиск, заполнение форм, сбор данных из нескольких мест и вывод конечного результата в соответствии с запрошенной схемой.

**Ключевые различия:**

- *stagehand_extract* — это быстрая операция «извлеки эти данные с этой страницы». Лучше всего работает для прямых, одношаговых задач извлечения.
- *stagehand_agent* выполняет сложные, многошаговые автономные задачи в интернете — такие как навигация, поиск или даже транзакции — и может динамически извлекать данные в соответствии с вашими инструкциями и опциональной схемой.

На практике используйте **stagehand_extract**, когда вы знаете, что и где хотите получить, и используйте **stagehand_agent**, когда вам нужен бот, который продумывает и выполняет интерактивные рабочие процессы.

Интегрируя Stagehand, агенты Sim могут автоматизировать сбор данных, анализ и выполнение рабочих процессов в интернете: обновление баз данных, организацию информации и создание пользовательских отчётов — бесшовно и автономно.

## Инструкции по использованию

Интегрируйте Stagehand в рабочий процесс. Можно извлекать структурированные данные с веб-страниц или запускать автономного агента для выполнения задач.



## Действия

### `stagehand_extract`

Извлечение структурированных данных с веб-страницы с помощью Stagehand

#### Вход

| Параметр | Тип | Обязательный | Описание |
| --------- | ---- | -------- | ----------- |
| `url` | string | Да | URL веб-страницы для извлечения данных |
| `instruction` | string | Да | Инструкции по извлечению |
| `provider` | string | Нет | Используемый AI-провайдер: openai или anthropic |
| `apiKey` | string | Да | Ключ API для выбранного провайдера |
| `schema` | json | Да | JSON-схема, определяющая структуру извлекаемых данных |

#### Выход

| Параметр | Тип | Описание |
| --------- | ---- | ----------- |
| `data` | object | Извлечённые структурированные данные, соответствующие предоставленной схеме |

### `stagehand_agent`

Запуск автономного веб-агента для выполнения задач и извлечения структурированных данных

#### Вход

| Параметр | Тип | Обязательный | Описание |
| --------- | ---- | -------- | ----------- |
| `startUrl` | string | Да | URL веб-страницы, с которой агент начинает работу |
| `task` | string | Да | Задача для выполнения или цель для достижения на веб-сайте |
| `variables` | json | Нет | Опциональные переменные для подстановки в задачу \(формат: \{key: value\}\). Ссылайтесь в задаче через %key% |
| `provider` | string | Нет | Используемый AI-провайдер: openai или anthropic |
| `apiKey` | string | Да | Ключ API для выбранного провайдера |
| `outputSchema` | json | Нет | Опциональная JSON-схема, определяющая структуру данных, которые должен вернуть агент |
| `mode` | string | Нет | Режим инструментов агента: dom \(по умолчанию\), hybrid или cua |
| `maxSteps` | number | Нет | Максимальное количество шагов агента \(по умолчанию 20, макс. 200\) |

#### Выход

| Параметр | Тип | Описание |
| --------- | ---- | ----------- |
| `agentResult` | object | Результат выполнения агента Stagehand |
|   ↳ `success` | boolean | Успешно ли завершилась задача агента без ошибок |
|   ↳ `completed` | boolean | Завершил ли агент выполнение \(может быть false, если достигнуто максимальное количество шагов\) |
|   ↳ `message` | string | Итоговое сообщение о статусе или резюме результата от агента |
|   ↳ `actions` | array | Список всех действий, выполненных агентом в ходе выполнения задачи |
|     ↳ `type` | string | Тип выполненного действия \(например, "act", "observe", "ariaTree", "close", "wait", "navigate"\) |
|     ↳ `reasoning` | string | Обоснование AI, почему было предпринято это действие |
|     ↳ `taskCompleted` | boolean | Была ли задача завершена после этого действия |
|     ↳ `action` | string | Описание предпринятого действия \(например, "click the submit button"\) |
|     ↳ `instruction` | string | Инструкция, вызвавшая это действие |
|     ↳ `pageUrl` | string | URL страницы на момент выполнения этого действия |
|     ↳ `pageText` | string | Текстовое содержимое страницы \(для действий ariaTree\) |
|     ↳ `timestamp` | number | Unix-временная метка выполнения действия |
|     ↳ `timeMs` | number | Время в миллисекундах \(для действий wait\) |
| `structuredOutput` | object | Извлечённые данные, соответствующие предоставленной выходной схеме |
| `liveViewUrl` | string | Встраиваемый URL живого просмотра Browserbase \(активен только во время сессии\) |
| `sessionId` | string | Идентификатор сессии Browserbase |
