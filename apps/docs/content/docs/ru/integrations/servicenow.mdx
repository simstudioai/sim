[ServiceNow](https://www.servicenow.com/) is a powerful cloud platform designed to streamline and automate IT service management (ITSM), workflows, and business processes across your organization. ServiceNow enables you to manage incidents, requests, tasks, users, and more using its extensive API.

With ServiceNow, you can:

- **Automate IT workflows**: Create, read, update, and delete records in any ServiceNow table, such as incidents, tasks, change requests, and users.
- **Интегрируйте systems**: Connect ServiceNow with your other tools and processes for seamless automation.
- **Maintain a single source of truth**: Keep all your service and operations data organized and accessible.
- **Drive operational efficiency**: Reduce manual work and improve service quality with customizable workflows and automation.

In Sim, the ServiceNow integration enables your agents to interact directly with your ServiceNow instance as part of their workflows. Agents can create, read, update, or delete records in any ServiceNow table and leverage ticket or user data for sophisticated automation and decision-making. This integration bridges your workflow automation and IT operations, empowering your agents to manage service requests, incidents, users, and assets without manual intervention. By connecting Sim with ServiceNow, you can automate service management tasks, improve response times, and ensure consistent, secure access to your organization's vital service data.

## Инструкции по использованию

Интегрируйте ServiceNow в ваш рабочий процесс. Create, read, update, and delete records in any ServiceNow table including incidents, tasks, change requests, users, and more.



## Действия

### `servicenow_create_record`

Создание нового record in a ServiceNow table

#### Вход

| Параметр | Тип | Обязательный | Описание |
| --------- | ---- | -------- | ----------- |
| `instanceUrl` | string | Да | ServiceNow instance URL (например, https://instance.service-now.com\) |
| `username` | string | Да | ServiceNow username |
| `password` | string | Да | ServiceNow password |
| `tableName` | string | Да | Table name (например, incident, task, sys_user\) |
| `fields` | json | Да | Fields to set on the record as JSON object (например, \{"short_description": "Issue title", "priority": "1"\}\) |

#### Выход

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `record` | json | Created ServiceNow record with sys_id and other fields |
| `metadata` | json | Operation metadata |

### `servicenow_read_record`

Read records from a ServiceNow table

#### Вход

| Параметр | Тип | Обязательный | Описание |
| --------- | ---- | -------- | ----------- |
| `instanceUrl` | string | Да | ServiceNow instance URL (например, https://instance.service-now.com\) |
| `username` | string | Да | ServiceNow username |
| `password` | string | Да | ServiceNow password |
| `tableName` | string | Да | Table name (например, incident, task, sys_user, change_request\) |
| `sysId` | string | Нет | Specific record sys_id (например, 6816f79cc0a8016401c5a33be04be441\) |
| `number` | string | Нет | Record number (например, INC0010001\) |
| `query` | string | Нет | Encoded query string (например, "active=true^priority=1"\) |
| `limit` | number | Нет | Maximum number of records to return (например, 10, 50, 100\) |
| `offset` | number | Нет | Number of records to skip for pagination (например, 0, 10, 20\) |
| `fields` | string | Нет | Comma-separated list of fields to return (например, sys_id,number,short_description,state\) |
| `displayValue` | string | Нет | Return display values for reference fields: "true" \(display only\), "false" \(sys_id only\), or "all" \(both\) |

#### Выход

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `records` | array | Array of ServiceNow records |
| `metadata` | json | Operation metadata |

### `servicenow_aggregate`

Compute aggregate statistics (count, sum, average, min, max, group by) over a ServiceNow table

#### Вход

| Параметр | Тип | Обязательный | Описание |
| --------- | ---- | -------- | ----------- |
| `instanceUrl` | string | Да | ServiceNow instance URL (например, https://instance.service-now.com\) |
| `username` | string | Да | ServiceNow username |
| `password` | string | Да | ServiceNow password |
| `tableName` | string | Да | Table name (например, incident, change_request, task\) |
| `query` | string | Нет | Encoded query string to filter records before aggregating (например, "active=true"\) |
| `count` | boolean | Нет | Return the count of matching records |
| `groupBy` | string | Нет | Comma-separated fields to group results by (например, category,priority\) |
| `avgFields` | string | Нет | Comma-separated numeric fields to average (например, reassignment_count\) |
| `sumFields` | string | Нет | Comma-separated numeric fields to sum |
| `minFields` | string | Нет | Comma-separated fields to compute the minimum of |
| `maxFields` | string | Нет | Comma-separated fields to compute the maximum of |
| `having` | string | Нет | Filter on aggregate results (например, "count&gt;5"\) |
| `displayValue` | string | Нет | Return display values for grouped reference fields: "true", "false", or "all" |

#### Выход

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `result` | json | Aggregate result. Ungrouped: \{stats: \{count, sum, avg, min, max\}\}. Grouped: array of \{stats, groupby_fields\}. |
| `count` | number | Total matching record count \(only present for ungrouped count queries\) |
| `metadata` | json | Operation metadata \(grouped, groupCount\) |

### `servicenow_list_attachments`

Получение списка the attachments on a ServiceNow record

#### Вход

| Параметр | Тип | Обязательный | Описание |
| --------- | ---- | -------- | ----------- |
| `instanceUrl` | string | Да | ServiceNow instance URL (например, https://instance.service-now.com\) |
| `username` | string | Да | ServiceNow username |
| `password` | string | Да | ServiceNow password |
| `tableName` | string | Да | Table that owns the record (например, incident, change_request\) |
| `recordSysId` | string | Да | sys_id of the record whose attachments should be listed |
| `limit` | number | Нет | Maximum number of attachments to return |

#### Выход

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `attachments` | array | Attachment metadata records |
|   ↳ `sys_id` | string | Attachment sys_id |
|   ↳ `file_name` | string | File name |
|   ↳ `content_type` | string | MIME type |
|   ↳ `size_bytes` | string | File size in bytes |
|   ↳ `download_link` | string | Direct download URL for the file |
| `metadata` | json | Operation metadata \(recordCount\) |

### `servicenow_download_attachment`

Download an attachment file from ServiceNow by its sys_id

#### Вход

| Параметр | Тип | Обязательный | Описание |
| --------- | ---- | -------- | ----------- |
| `instanceUrl` | string | Да | ServiceNow instance URL (например, https://instance.service-now.com\) |
| `username` | string | Да | ServiceNow username |
| `password` | string | Да | ServiceNow password |
| `attachmentSysId` | string | Да | sys_id of the attachment to download \(from Получение списка Attachments\) |

#### Выход

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `file` | file | Downloaded attachment stored in execution files |
| `content` | string | Base64 encoded file content |

### `servicenow_upload_attachment`

Attach a file to a ServiceNow record

#### Вход

| Параметр | Тип | Обязательный | Описание |

## Триггеры

**Триггер** — это блок, который запускает рабочий процесс при наступлении события в этом сервисе.

### ServiceNow Change Request Created

Запуск рабочего процесса when a new change request is created in ServiceNow

#### Конфигурация

| Параметр | Тип | Обязательный | Описание |
| --------- | ---- | -------- | ----------- |
| `webhookSecret` | string | Да | Required. Use the same value in your ServiceNow Business Rule as Bearer token or X-Sim-Webhook-Secret. |
| `tableName` | string | Нет | Optionally filter to a specific ServiceNow table |

#### Выход

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `sysId` | string | Unique system ID of the record |
| `number` | string | Record number (например, INC0010001, CHG0010001\) |
| `tableName` | string | ServiceNow table name |
| `shortDescription` | string | Short description of the record |
| `description` | string | Full description of the record |
| `state` | string | Current state of the record |
| `priority` | string | Priority level \(1=Critical, 2=High, 3=Moderate, 4=Low, 5=Planning\) |
| `assignedTo` | string | User assigned to this record |
| `assignmentGroup` | string | Group assigned to this record |
| `createdBy` | string | User who created the record |
| `createdOn` | string | When the record was created \(ISO 8601\) |
| `updatedBy` | string | User who last updated the record |
| `updatedOn` | string | When the record was last updated \(ISO 8601\) |
| `type` | string | Change type \(Normal, Standard, Emergency\) |
| `risk` | string | Risk level of the change |
| `impact` | string | Impact level of the change |
| `approval` | string | Approval status |
| `startDate` | string | Planned start date |
| `endDate` | string | Planned end date |
| `category` | string | Change category |
| `record` | json | Full change request record data |
