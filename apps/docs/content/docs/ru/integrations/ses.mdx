[Amazon Simple Email Service (SES)](https://aws.amazon.com/ses/) is a cloud-based email sending service designed for high-volume, transactional, and marketing email delivery. It provides a cost-effective, scalable way to send email without managing your own mail server infrastructure.

With AWS SES, you can:

- **Send simple emails**: Deliver one-off emails with plain text or HTML body content to individual recipients
- **Send templated emails**: Use pre-defined templates with variable substitution (e.g., `{{name}}`, `{{link}}`) for personalized emails at scale
- **Send bulk emails**: Deliver templated emails to large lists of recipients in a single API call, with per-destination data overrides
- **Manage email templates**: Create, retrieve, list, and delete reusable email templates for transactional and marketing campaigns
- **Monitor account health**: Получение your account's sending quota, send rate, and whether sending is currently enabled

In Sim, the AWS SES integration is designed for workflows that need reliable, programmatic email delivery — from access request notifications and approval alerts to bulk outreach and automated reporting. It pairs naturally with the IAM Identity Center integration for TEAM (Temporary Elevated Access Management) workflows, where email notifications are sent when access is provisioned, approved, or revoked.

## Инструкции по использованию

Интегрируйте AWS SES v2 в рабочий процесс. Send simple, templated, and bulk emails. Manage email templates and retrieve account sending quota and verified identity information.



## Действия

### `ses_send_email`

Отправка письма via AWS SES using simple or HTML content

#### Вход

| Параметр | Тип | Обязательный | Описание |
| --------- | ---- | -------- | ----------- |
| `region` | string | Да | Регион AWS (например, us-east-1\) |
| `accessKeyId` | string | Да | Идентификатор ключа доступа AWS |
| `secretAccessKey` | string | Да | Секретный ключ доступа AWS |
| `fromAddress` | string | Да | Verified sender email address |
| `toAddresses` | string | Да | Comma-separated list of recipient email addresses |
| `subject` | string | Да | Email subject line |
| `bodyText` | string | Нет | Plain text email body |
| `bodyHtml` | string | Нет | HTML email body |
| `ccAddresses` | string | Нет | Comma-separated list of CC email addresses |
| `bccAddresses` | string | Нет | Comma-separated list of BCC email addresses |
| `replyToAddresses` | string | Нет | Comma-separated list of reply-to email addresses |
| `configurationSetName` | string | Нет | SES configuration set name for tracking |

#### Выход

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `messageId` | string | SES message ID for the sent email |

### `ses_send_templated_email`

Отправка письма using an SES email template with dynamic template data

#### Вход

| Параметр | Тип | Обязательный | Описание |
| --------- | ---- | -------- | ----------- |
| `region` | string | Да | Регион AWS (например, us-east-1\) |
| `accessKeyId` | string | Да | Идентификатор ключа доступа AWS |
| `secretAccessKey` | string | Да | Секретный ключ доступа AWS |
| `fromAddress` | string | Да | Verified sender email address |
| `toAddresses` | string | Да | Comma-separated list of recipient email addresses |
| `templateName` | string | Да | Name of the SES email template to use |
| `templateData` | string | Да | JSON string of key-value pairs for template variable substitution |
| `ccAddresses` | string | Нет | Comma-separated list of CC email addresses |
| `bccAddresses` | string | Нет | Comma-separated list of BCC email addresses |
| `configurationSetName` | string | Нет | SES configuration set name for tracking |

#### Выход

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `messageId` | string | SES message ID for the sent email |

### `ses_send_bulk_email`

Send emails to multiple recipients using an SES template with per-recipient data

#### Вход

| Параметр | Тип | Обязательный | Описание |
| --------- | ---- | -------- | ----------- |
| `region` | string | Да | Регион AWS (например, us-east-1\) |
| `accessKeyId` | string | Да | Идентификатор ключа доступа AWS |
| `secretAccessKey` | string | Да | Секретный ключ доступа AWS |
| `fromAddress` | string | Да | Verified sender email address |
| `templateName` | string | Да | Name of the SES email template to use |
| `destinations` | string | Да | JSON array of destination objects with toAddresses \(string\[\]\) and optional templateData \(JSON string\); falls back to defaultTemplateData when omitted |
| `defaultTemplateData` | string | Нет | Default JSON template data used when a destination does not specify its own |
| `configurationSetName` | string | Нет | SES configuration set name for tracking |

#### Выход

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `results` | array | Per-destination send results with status and messageId |
| `successCount` | number | Number of successfully sent emails |
| `failureCount` | number | Number of failed email sends |

### `ses_list_identities`

Получение списка verified email identities (email addresses and domains) in your SES account

#### Вход

| Параметр | Тип | Обязательный | Описание |
| --------- | ---- | -------- | ----------- |
| `region` | string | Да | Регион AWS (например, us-east-1\) |
| `accessKeyId` | string | Да | Идентификатор ключа доступа AWS |
| `secretAccessKey` | string | Да | Секретный ключ доступа AWS |
| `pageSize` | number | Нет | Maximum number of identities to return \(1-1000\) |
| `nextToken` | string | Нет | Pagination token from a previous list response |

#### Выход

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `identities` | array | Получение списка of email identities with name, type, sending status, and verification status |
| `nextToken` | string | Pagination token for the next page of results |
| `count` | number | Number of identities returned |
