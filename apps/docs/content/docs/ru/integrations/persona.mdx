[Persona](https://withpersona.com/) is an identity verification platform that helps businesses verify who their users are. Persona handles the full identity lifecycle — collecting government IDs, selfies, and documents through hosted verification flows, screening individuals against global watchlists and sanctions lists, and routing edge cases to human review.

With Persona, you can:

- **Verify identities end to end**: Create inquiries from your verification templates, send customers one-time verification links, and read back collected fields and decision results
- **Automate KYC and compliance decisions**: Approve or decline inquiries programmatically, run watchlist, adverse media, and politically-exposed-person screening reports, and monitor cases that need manual review
- **Manage your verified user base**: Create and look up accounts, bulk-import existing users from CSV, and keep your user model in sync with Persona via reference IDs
- **Keep an audit trail**: Download inquiry summary PDFs and retrieve the underlying verifications and documents behind every decision

In Sim, the Persona block lets your agents drive identity verification as part of real workflows. Trigger verification when a customer signs up, route on approval status, screen names against watchlists before activating accounts, post pending reviews to Slack, or archive verification PDFs to cloud storage — all using your Persona Ключ API and templates.

## Инструкции по использованию

Интегрируйте Persona identity verification в рабочий процесс. Manage the full inquiry lifecycle (create, update, approve, decline, review, resume, expire, redact), generate one-time verification links and PDF summaries, manage accounts including CSV bulk import, run watchlist and adverse media reports, review cases, retrieve verifications and documents, and discover inquiry templates.



## Действия

### `persona_create_inquiry`

Создание нового identity verification inquiry from an inquiry template. Returns the created inquiry, which can then be completed by the individual via a one-time link.

#### Вход

| Параметр | Тип | Обязательный | Описание |
| --------- | ---- | -------- | ----------- |
| `apiKey` | string | Да | Persona Ключ API |
| `inquiryTemplateId` | string | Да | Inquiry template ID \(starts with itmpl_\), inquiry template version ID \(starts with itmplv_\), or legacy template ID \(starts with tmpl_\) |
| `accountId` | string | Нет | Account ID \(starts with act_\) to associate with this inquiry |
| `referenceId` | string | Нет | Reference ID that refers to an entity in your user model. An account is auto-created for it if one does not exist. |
| `fields` | json | Нет | JSON object of field name to field value pairs to pre-fill, as defined by the inquiry template (например, \{"name-first": "Jane"\}\) |
| `note` | string | Нет | Free-form note to attach to the inquiry |
| `redirectUri` | string | Нет | URI to redirect the individual to after completing the inquiry flow |

#### Выход

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `inquiry` | object | The created inquiry |

### `persona_list_inquiries`

Получение списка identity verification inquiries, optionally filtered by status, account ID, reference ID, or creation date range. Results are cursor-paginated.

#### Вход

| Параметр | Тип | Обязательный | Описание |
| --------- | ---- | -------- | ----------- |
| `apiKey` | string | Да | Persona Ключ API |
| `status` | string | Нет | Filter by inquiry status \(created, pending, completed, failed, expired, needs_review, approved, declined\) |
| `accountId` | string | Нет | Filter by account ID \(starts with act_\); comma-separate multiple IDs |
| `referenceId` | string | Нет | Filter by reference ID |
| `createdAtStart` | string | Нет | Filter to inquiries created at or after this ISO 8601 timestamp |
| `createdAtEnd` | string | Нет | Filter to inquiries created at or before this ISO 8601 timestamp |
| `pageSize` | number | Нет | Number of inquiries to return per page \(1-100, по умолчанию 10\) |
| `pageAfter` | string | Нет | Pagination cursor: return inquiries after this inquiry ID |

#### Выход

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `inquiries` | array | Inquiries matching the filters |
| `nextCursor` | string | Cursor for the next page \(pass as pageAfter\), or null on the last page |
