[Apify](https://apify.com/) — это мощная платформа для создания, развёртывания и запуска акторов веб-автоматизации и веб-скрапинга в масштабе. Apify позволяет извлекать полезные данные с любого веб-сайта, автоматизировать рабочие процессы и бесшовно соединять конвейеры данных.

С Apify вы можете:

- **Запускать готовые или пользовательские акторы**: Интегрируйте публичные акторы или разрабатывайте собственные, автоматизируя широкий спектр задач по извлечению веб-данных и работе с браузером.
- **Получать наборы данных**: Доступ к структурированным наборам данных, собранным акторами, и управление ими в реальном времени.
- **Масштабировать веб-автоматизацию**: Используйте облачную инфраструктуру для надёжного выполнения задач, асинхронно или синхронно, с устойчивой обработкой ошибок.

В Sim интеграция с Apify позволяет вашим агентам выполнять основные операции Apify программно:

- **Запуск актора (синхронно)**: Используйте `apify_run_actor_sync` для запуска актора Apify и ожидания его завершения с получением результатов сразу после окончания выполнения.
- **Запуск актора (асинхронно)**: Используйте `apify_run_actor_async` для запуска актора в фоновом режиме и периодического опроса результатов, что подходит для более длительных или сложных задач.

Эти операции позволяют вашим агентам автоматизировать, извлекать данные и оркестрировать задачи сбора данных или браузерной автоматизации непосредственно внутри рабочих процессов — всё с гибкой конфигурацией и обработкой результатов, без необходимости ручного запуска или внешних инструментов. Интегрируйте Apify как динамический движок автоматизации и извлечения данных, который программно обеспечивает работу веб-масштабных рабочих процессов ваших агентов.

## Инструкция по использованию

Интегрируйте Apify в ваш рабочий процесс. Запускайте любой актор Apify или сохранённую задачу с пользовательскими входными данными, получайте элементы набора данных и проверяйте статус выполнения. Поддерживает как синхронное, так и асинхронное выполнение с автоматическим получением набора данных.



## Действия

### `apify_run_actor_sync`

Синхронный запуск актора Apify и получение результатов (макс. 5 минут)

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `apiKey` | string | Yes | Токен API Apify из console.apify.com/account#/integrations |
| `actorId` | string | Yes | ID актора или username/actor-name. Примеры: "apify/web-scraper", "janedoe/my-actor", "moJRLRc85AitArpNN" |
| `input` | string | No | Входные данные актора в виде строки JSON. Пример: \{"startUrls": \[\{"url": "https://example.com"\}\], "maxPages": 10\} |
| `memory` | number | No | Выделенная память в мегабайтах для запуска актора \(128-32768\). Пример: 1024 для 1 ГБ, 2048 для 2 ГБ |
| `timeout` | number | No | Тайм-аут в секундах для запуска актора. Пример: 300 для 5 минут, 3600 для 1 часа |
| `build` | string | No | Сборка актора для запуска. Примеры: "latest", "beta", "1.2.3", "build-tag-name" |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `success` | boolean | Успешно ли завершился запуск актора |
| `runId` | string | ID запуска Apify |
| `status` | string | Статус запуска \(SUCCEEDED, FAILED и т.д.\) |
| `items` | array | Элементы набора данных \(если выполнение завершено\) |

### `apify_run_actor_async`

Асинхронный запуск актора Apify с опросом для долго выполняющихся задач

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `apiKey` | string | Yes | Токен API Apify из console.apify.com/account#/integrations |
| `actorId` | string | Yes | ID актора или username/actor-name. Примеры: "apify/web-scraper", "janedoe/my-actor", "moJRLRc85AitArpNN" |
| `input` | string | No | Входные данные актора в виде строки JSON. Пример: \{"startUrls": \[\{"url": "https://example.com"\}\], "maxPages": 10\} |
| `waitForFinish` | number | No | Начальное время ожидания в секундах \(0-60\) до начала опроса. Пример: 30 |
| `itemLimit` | number | No | Максимальное количество извлекаемых элементов набора данных \(1-250000\). По умолчанию: 100. Пример: 500 |
| `memory` | number | No | Выделенная память в мегабайтах для запуска актора \(128-32768\). Пример: 1024 для 1 ГБ, 2048 для 2 ГБ |
| `timeout` | number | No | Тайм-аут в секундах для запуска актора. Пример: 300 для 5 минут, 3600 для 1 часа |
| `build` | string | No | Сборка актора для запуска. Примеры: "latest", "beta", "1.2.3", "build-tag-name" |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `success` | boolean | Успешно ли завершился запуск актора |
| `runId` | string | ID запуска Apify |
| `status` | string | Статус запуска \(SUCCEEDED, FAILED и т.д.\) |
| `datasetId` | string | ID набора данных, содержащего результаты |
| `items` | array | Элементы набора данных \(если выполнение завершено\) |

### `apify_run_task`

Синхронный запуск сохранённой задачи актора Apify и получение элементов набора данных (макс. 5 минут)

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `apiKey` | string | Yes | Токен API Apify из console.apify.com/account#/integrations |
| `taskId` | string | Yes | ID задачи или username/task-name. Примеры: "janedoe/my-task", "moJRLRc85AitArpNN" |
| `input` | string | No | Строка JSON, переопределяющая сохранённые входные данные задачи. Пример: \{"startUrls": \[\{"url": "https://example.com"\}\]\} |
| `itemLimit` | number | No | Максимальное количество возвращаемых элементов набора данных \(1-250000\). Пример: 500 |
| `memory` | number | No | Выделенная память в мегабайтах для запуска \(128-32768\). Пример: 1024 для 1 ГБ |
| `timeout` | number | No | Тайм-аут в секундах для запуска. Пример: 300 для 5 минут |
| `build` | string | No | Сборка актора для запуска. Примеры: "latest", "beta", "1.2.3" |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `success` | boolean | Успешно ли завершился запуск задачи |
| `status` | string | Статус запуска \(SUCCEEDED, FAILED и т.д.\) |
| `items` | array | Элементы набора данных, созданные при выполнении |

### `apify_get_dataset_items`

Получение элементов, хранящихся в наборе данных Apify

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `apiKey` | string | Yes | Токен API Apify из console.apify.com/account#/integrations |
| `datasetId` | string | Yes | ID набора данных для чтения элементов. Пример: "9RnD3Pql2vGZkc5H5" |
| `itemLimit` | number | No | Максимальное количество возвращаемых элементов \(1-250000\). По умолчанию: все элементы. Пример: 500 |
| `offset` | number | No | Количество пропускаемых элементов в начале. По умолчанию: 0 |
| `fields` | string | No | Список полей для включения через запятую. Пример: "title,url,price" |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `success` | boolean | Успешно ли получены элементы |
| `datasetId` | string | ID набора данных, из которого были прочитаны элементы |
| `items` | array | Элементы, хранящиеся в наборе данных |
| `count` | number | Количество возвращённых элементов |

### `apify_get_run`

Получение статуса и деталей запуска актора Apify

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `apiKey` | string | Yes | Токен API Apify из console.apify.com/account#/integrations |
| `runId` | string | Yes | ID запуска актора для получения. Пример: "HG7ML7M8z78YcAPEB" |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `success` | boolean | Был ли запуск найден |
| `runId` | string | ID запуска Apify |
| `status` | string | Статус запуска \(READY, RUNNING, SUCCEEDED, FAILED и т.д.\) |
| `startedAt` | string | Время начала запуска \(ISO-дата\) |
| `finishedAt` | string | Время завершения запуска \(ISO-дата\) |
| `datasetId` | string | ID набора данных по умолчанию для запуска |
| `keyValueStoreId` | string | ID хранилища ключ-значение по умолчанию для запуска |
| `stats` | json | Статистика запуска \(память, CPU, длительность\) |
