[Apify](https://apify.com/)Apifyhttps://apify.com/является мощной платформой для создания, развертывания и запуска веб-автоматизации и веб-скрейпинга в больших масштабах. Apify позволяет извлекать полезные данные с любого сайта, автоматизировать рабочие процессы и беспрепятственно подключать ваши каналы передачи данных.

С помощью Apify вы можете:

- **Используйте готовые или разработанные акторы:** Интегрируйте существующие публичные акторы или разработайте собственные, автоматизируя широкий спектр задач извлечения данных из веб-сайтов и работы с браузером.- **Получение наборов данных:** Доступ к и управление структурированными наборами данных, собранными участниками в режиме реального времени.- **Автоматизация веб-приложений:** Используйте облачную инфраструктуру для надежного выполнения задач асинхронно или синхронно, с эффективной обработкой ошибок.
В Sim интеграция с Apify позволяет вашим агентам выполнять основные операции Apify программным способом.

- **Запуск актора (синхронный):** Используйте `apify_run_actor_sync`, чтобы запустить актор Apify и дождаться его завершения, получая результаты сразу после завершения запуска.- **Запуск актора (асинхронно)**: Используйте `apify_run_actor_async`, чтобы запустить актор в фоновом режиме и периодически получать результаты, что подходит для более длительных или сложных задач.
Эти операции позволяют вашим агентам автоматизировать, извлекать и координировать задачи сбора данных или автоматизации браузера непосредственно внутри рабочих процессов — все это благодаря гибкой настройке и обработке результатов, без необходимости в ручном выполнении или внешних инструментах. Интегрируйте Apify как динамический движок автоматизации и извлечения данных, который программно обеспечивает работу ваших агентов в масштабе веб-приложений.

## Инструкция по использованию
Интегрируйте Apify в свою рабочую среду. Запускайте любые акторы Apify или сохраненные задачи с использованием пользовательских входных данных, получайте элементы набора данных и проверяйте статус выполнения. Поддерживает как синхронное, так и асинхронное выполнение с автоматическим извлечением наборов данных.



## Действия
### `apify_run_actor_sync`

Запустите актора APIFY синхронно и получите результаты (максимум 5 минут).

#### Ввод
|  | Параметр | Тип | Необходимо | Описание |  || --------- | ---- | -------- | ----------- |
|  | `apiKey` | строка | Да | Токен API APIFY из консоли account.apify.com/integrations |  ||  | `actorId` | строка | Да | Идентификатор актера или имя пользователя/название актора. Примеры: "apify/web-scraper", "janedoe/my-actor", "moJRLRc85AitArpNN" |  ||  | `input` | строка | Нет | Ввод данных актера в виде строки JSON. Пример: `{"startUrls": [{"url": "https://example.com"}] , "maxPages": 10}` |  ||  | `memory` | число | Нет | Объем памяти в мегабайтах, выделенный для работы актора (от 128 до 32768 МБ). Пример: 1024 МБ для 1 ГБ, 2048 МБ для 2 ГБ. |  ||  | `timeout` | число | Нет | Время ожидания в секундах для выполнения задачи. Например, 300 секунд – это 5 минут, 3600 секунд – это 1 час. |  ||  | `build` | строка | Нет | Версия для развертывания. Примеры: «последняя», «бета», «1.2.3», «имя версии» |  |
#### Вывод
|  | Параметр | Тип | Описание |  || --------- | ---- | ----------- |
|  | `success` | логическое значение | Успешно ли прошло действие? |  ||  | `runId` | строка | Идентификатор выполнения APIFY |  ||  | `status` | строка | Статус выполнения (УСПЕШНО, НЕ УСПЕХО, и т.д.) |  ||  | `items` | массив | Элементы набора данных (если они завершены) |  |

Запустите актора APIFY асинхронно с использованием периодического опроса для задач продолжительной работы.

#### Ввод
|  | Параметр | Тип | Необходимо | Описание |  || --------- | ---- | -------- | ----------- |

Примеры: "apify/web-scraper", "janedoe/my-actor", "moJRLRc85AitArpNN" |  ||  | `input` | строка | Нет | Ввод данных актера в виде строки JSON. Пример: `{"startUrls": [{"url": "https://example.com"}] , "maxPages": 10}` |  ||  | `waitForFinish` | число | Нет | Начальное время ожидания в секундах (от 0 до 60) перед началом опроса. Пример: 30 |  ||  | `itemLimit` | число | Нет | Максимальное количество элементов набора данных для извлечения (от 1 до 250 000). Значение по умолчанию: 100. Пример: 500 |  ||  | `memory` | число | Нет | Объем памяти в мегабайтах, выделенный для работы актора (от 128 до 32768 МБ). Пример: 1024 МБ для 1 ГБ, 2048 МБ для 2 ГБ. |  ||  | `timeout` | число | Нет | Время ожидания в секундах для выполнения задачи. Например, 300 секунд – это 5 минут, 3600 секунд – это 1 час. |  ||  | `build` | строка | Нет | Версия для развертывания. Примеры: «последняя», «бета», «1.2.3», «имя версии» |  |
#### Вывод
|  | Параметр | Тип | Описание |  || --------- | ---- | ----------- |
|  | `success` | логическое значение | Успешно ли прошло действие? |  ||  | `runId` | строка | Идентификатор выполнения APIFY |  ||  | `status` | строка | Статус выполнения (УСПЕШНО, НЕ УСПЕХО, и т.д.) |  ||  | `datasetId` | строка | Идентификатор набора данных, содержащий результаты. |  ||  | `items` | массив | Элементы набора данных (если они завершены) |  |
### `apify_run_task`

Запустите сохраненную задачу "APIFY" синхронно и получите элементы набора данных (максимум 5 минут).

#### Ввод
|  | Параметр | Тип | Необходимо | Описание |  || --------- | ---- | -------- | ----------- |
|  | `apiKey` | строка | Да | Токен API APIFY из консоли account.apify.com/integrations |  ||  | `taskId` | строка | Да | Идентификатор задачи или имя пользователя/название задачи. Примеры: «janedoe/my-task», «moJRLRc85AitArpNN» |  ||  | `input` | строка | Нет | Строка JSON, которая переопределяет сохраненные входные данные задачи. Пример: `{"startUrls": [{"url": "https://example.com"}]}` |  ||  | `itemLimit` | число | Нет | Максимальное количество элементов набора данных для возврата (от 1 до 250 000). Пример: 500 |  ||  | `memory` | число | Нет | Объем памяти в мегабайтах, выделенный для выполнения (от 128 до 32768 МБ). Пример: 1024 МБ для 1 ГБ. |  ||  | `timeout` | число | Нет | Время ожидания в секундах для выполнения задачи. Например, 300 секунд – это 5 минут. |  ||  | `build` | строка | Нет | Версия, предназначенная для запуска. Примеры: «последняя», «бета», «1.2.3» |  |
#### Вывод
|  | Параметр | Тип | Описание |  || --------- | ---- | ----------- |
|  | `success` | логическое значение | Удалось ли выполнить задачу? |  ||  | `status` | строка | Статус выполнения (УСПЕШНО, НЕ УСПЕХО, и т.д.) |  ||  | `items` | массив | Элементы набора данных, созданные в результате выполнения. |  |
### `apify_get_dataset_items`

Получить элементы, хранящиеся в наборе данных APIFY.

#### Ввод
|  | Параметр | Тип | Необходимо | Описание |  || --------- | ---- | -------- | ----------- |
|  | `apiKey` | строка | Да | Токен API APIFY из консоли account.apify.com/integrations |  ||  | `datasetId` | строка | Да | Идентификатор набора данных для чтения элементов. Пример: "9RnD3Pql2vGZkc5H5" |  ||  | `itemLimit` | число | Нет | Максимальное количество элементов для возврата (от 1 до 250 000). Значение по умолчанию: все элементы. Пример: 500 |  ||  | `offset` | число | Нет | Количество элементов, которые нужно пропустить в начале. Значение по умолчанию: 0 |  ||  | `fields` | строка | Нет | Список полей, разделенных запятыми, для включения. Пример: «название, URL, цена» |  |
#### Вывод
|  | Параметр | Тип | Описание |  || --------- | ---- | ----------- |
|  | `success` | логическое значение | Были ли эти элементы получены? |  ||  | `datasetId` | строка | Идентификатор набора данных, из которого были прочитаны элементы. |  ||  | `items` | массив | Элементы, хранящиеся в наборе данных |  ||  | `count` | число | Количество возвращенных товаров |  |
### `apify_get_run`

Получите статус и детали выполнения APIFY.

#### Ввод
|  | Параметр | Тип | Необходимо | Описание |  || --------- | ---- | -------- | ----------- |
|  | `apiKey` | строка | Да | Токен API APIFY из консоли account.apify.com/integrations |  ||  | `runId` | строка | Да | Идентификатор выполнения для получения данных. Пример: "HG7ML7M8z78YcAPEB" |  |
#### Вывод
|  | Параметр | Тип | Описание |  || --------- | ---- | ----------- |
|  | `success` | логическое значение | Было ли обнаружено это событие? |  ||  | `runId` | строка | Идентификатор выполнения APIFY |  ||  | `status` | строка | Статус выполнения (ГОТОВО, В РАБОТЕ, УСПЕШНО, НЕУСПЕХ и т.д.) |  ||  | `startedAt` | строка | Когда началась работа (метка времени ISO) |  ||  | `finishedAt` | строка | Когда выполнение завершилось (метка времени ISO) |  ||  | `datasetId` | строка | Стандартный идентификатор набора данных для запуска |  ||  | `keyValueStoreId` | строка | Стандартный идентификатор хранилища ключ-значение для запуска. |  ||  | `stats` | JSON | Запустите статистику (память, процессор, время выполнения) |  |
