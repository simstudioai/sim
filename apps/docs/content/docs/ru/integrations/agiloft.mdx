[Agiloft](https://www.agiloft.com/)Agilofthttps://www.agiloft.com/является платформой для управления жизненным циклом контрактов в масштабе предприятия, которая помогает организациям автоматизировать и управлять контрактами, соглашениями и связанными с ними бизнес-процессами в любой базе знаний.

При интеграции с Agiloft в Sim, вы можете:

- **Создавайте записи:** Добавляйте новые записи в любую таблицу Agiloft с использованием пользовательских значений полей.- **Прочитать записи:** Получить отдельные записи по идентификатору, с возможностью выбора дополнительных полей.- **Обновление записей:** Изменение полей существующих записей в любой таблице.- **Удаление записей:** Удалите записи из вашей базы знаний.- **Поиск записей:** Находите записи с использованием синтаксиса запросов Agiloft, с поддержкой постраничной навигации.- **Выбор записей:** Используйте операторы SQL WHERE для выполнения запросов и фильтрации данных.- **Сохраненные запросы:** Список определений сохранённых запросов для таблицы.- **Прикрепить файлы:** Загрузите и прикрепите файлы к соответствующим полям.- **Получение вложений:** Скачивание файлов из полей записи.- **Удалить вложения:** Удалите прикрепленные файлы из полей записи, используя их позицию.- **Информация об вложениях:** Получите метаданные обо всех файлах, прикрепленных к определенному полю записи.- **Блокировать записи:** Проверять, получать или освобождать блоки на записях для одновременного редактирования.
В системе Sim интеграция с Agiloft позволяет вашим агентам управлять контрактами и документами программно в рамках автоматизированных рабочих процессов. Агенты могут создавать и обновлять документы, искать информацию по таблицам, обрабатывать прикрепленные файлы и управлять блокировками документов — что обеспечивает интеллектуальную автоматизацию жизненного цикла контрактов.

## Инструкция по применению
Интегрируйтесь с системой управления жизненным циклом контрактов Agiloft для создания, чтения, обновления, удаления и поиска записей. Поддерживает прикрепление файлов, выбор на основе SQL, сохраненные запросы и блокировку записей в любой таблице вашей базы знаний.



## Действия
### `agiloft_attach_file`

Прикрепите файл к полю в записи в системе Agiloft.

#### Ввод
|  | Параметр | Тип | Необходимо | Описание |  || --------- | ---- | -------- | ----------- |
|  | `instanceUrl` | строка | Да | URL экземпляра Agiloft (например, https://mycompany.agiloft.com) |  ||  | `knowledgeBase` | строка | Да | Название базы знаний |  ||  | `login` | строка | Да | Имя пользователя для Agiloft |  ||  | `password` | строка | Да | Пароль для Agiloft |  ||  | `table` | строка | Да | Имя таблицы (например, «договоры») |  ||  | `recordId` | строка | Да | Идентификатор записи, к которой нужно прикрепить файл. |  ||  | `fieldName` | строка | Да | Название поля для прикрепленного файла |  ||  | `file` | файл | Нет | File to attach |  ||  | `fileName` | string | No | Name to assign to the file \(defaults to original file name\) |  |
#### Output
|  | Parameter | Type | Description |  || --------- | ---- | ----------- |
|  | `recordId` | string | ID of the record the file was attached to |  ||  | `fieldName` | string | Name of the field the file was attached to |  ||  | `fileName` | string | Name of the attached file |  ||  | `totalAttachments` | number | Total number of files attached in the field after the operation |  |
