[Agiloft](https://www.agiloft.com/)Agilofthttps://www.agiloft.com/является платформой для управления жизненным циклом контрактов в масштабе предприятия, которая помогает организациям автоматизировать и управлять контрактами, соглашениями и связанными с ними бизнес-процессами во всех информационных системах.

При интеграции с Agiloft в Sim, вы можете:

- **Создавайте записи:** Добавляйте новые записи в любую таблицу Agiloft с использованием пользовательских значений полей.- **Прочитать записи:** Получить отдельные записи по идентификатору, с возможностью выбора дополнительных полей.- **Обновление записей:** Изменение полей существующих записей в любой таблице.- **Удаление записей:** Удалите записи из вашей базы знаний.- **Поиск записей:** Находите записи с использованием синтаксиса запросов Agiloft и с поддержкой постраничной навигации.- **Выбор записей:** Используйте операторы SQL WHERE для выполнения запросов и фильтрации данных.- **Сохраненные запросы:** Список определений сохранённых запросов для таблицы.- **Прикрепить файлы:** Загрузите и прикрепите файлы к соответствующим полям.- **Получить вложения:** Скачать файлы, прикрепленные к записям.- **Удалить вложения:** Удалите прикрепленные файлы из полей записи, используя их позицию.- **Информация об вложениях:** Получите метаданные обо всех файлах, прикрепленных к определенному полю записи.- **Блокировать записи:** Проверять, получать или освобождать блоки на записях для одновременного редактирования.
В системе Sim интеграция с Agiloft позволяет вашим агентам управлять контрактами и документами программно в рамках автоматизированных рабочих процессов. Агенты могут создавать и обновлять документы, искать информацию по таблицам, обрабатывать прикрепленные файлы и управлять блокировками документов — что обеспечивает интеллектуальную автоматизацию жизненного цикла контрактов.

## Инструкция по использованию
Интегрируйтесь с системой управления жизненным циклом контрактов Agiloft для создания, чтения, обновления, удаления и поиска записей. Поддерживает прикрепление файлов, выбор на основе SQL, сохраненные запросы и блокировку записей в любой таблице вашей базы знаний.

### `agiloft_attach_file`

Прикрепите файл к полю в записи в системе Agiloft.

#### Ввод
|  | Параметр | Тип | Необходимо | Описание |  || --------- | ---- | -------- | ----------- |
|  | `instanceUrl` | строка | Да | URL экземпляра Agiloft (например, https://mycompany.agiloft.com) |  ||  | `knowledgeBase` | строка | Да | Название базы знаний |  ||  | `login` | строка | Да | Имя пользователя для Agiloft |  ||  | `password` | строка | Да | Пароль для Agiloft |  ||  | `table` | строка | Да | Имя таблицы (например, «договоры») |  ||  | `recordId` | строка | Да | Идентификатор записи, к которой нужно прикрепить файл. |  ||  | `fieldName` | строка | Да | Название поля для прикрепленного файла |  ||  | `file` | файл | Нет | Файл для прикрепления |  ||  | `fileName` | строка | Нет | Имя для присвоения файлу (по умолчанию используется имя исходного файла) |  |

|  | Параметр | Тип | Описание |  || --------- | ---- | ----------- |
|  | `recordId` | строка | Идентификатор записи, к которой был прикреплен файл. |  ||  | `fieldName` | строка | Название поля, к которому был прикреплен файл. |  ||  | `fileName` | строка | Имя прикрепленного файла |  ||  | `totalAttachments` | число | Общее количество файлов, прикрепленных в поле после операции. |  |
