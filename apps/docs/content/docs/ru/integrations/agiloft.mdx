[Agiloft](https://www.agiloft.com/) — это корпоративная платформа управления жизненным циклом контрактов (CLM), которая помогает организациям автоматизировать и управлять контрактами, соглашениями и связанными бизнес-процессами в любой базе знаний.

С помощью интеграции Agiloft в Sim вы можете:

- **Создавать записи**: Добавлять новые записи в любую таблицу Agiloft с пользовательскими значениями полей
- **Читать записи**: Получать отдельные записи по ID с опциональным выбором полей
- **Обновлять записи**: Изменять существующие поля записей в любой таблице
- **Удалять записи**: Удалять записи из вашей базы знаний
- **Искать записи**: Находить записи с использованием синтаксиса запросов Agiloft с поддержкой пагинации
- **Выбирать записи**: Запрашивать записи с использованием SQL WHERE для расширенной фильтрации
- **Сохранённые поиски**: Просматривать определения сохранённых поисков, доступных для таблицы
- **Прикреплять файлы**: Загружать и прикреплять файлы к полям записей
- **Получать вложения**: Скачивать прикреплённые файлы из полей записей
- **Удалять вложения**: Удалять прикреплённые файлы из полей записей по позиции
- **Информация о вложениях**: Получать метаданные обо всех файлах, прикреплённых к полю записи
- **Блокировать записи**: Проверять, устанавливать или снимать блокировки записей для параллельного редактирования

В Sim интеграция Agiloft позволяет вашим агентам программно управлять контрактами и записями в рамках автоматизированных рабочих процессов. Агенты могут создавать и обновлять записи, выполнять поиск по таблицам, обрабатывать вложения файлов и управлять блокировками записей — обеспечивая интеллектуальную автоматизацию жизненного цикла контрактов.

## Usage Instructions

Интегрируйтесь с управлением жизненным циклом контрактов Agiloft для создания, чтения, обновления, удаления и поиска записей. Поддерживает вложения файлов, выборку на основе SQL, сохранённые поиски и блокировку записей в любой таблице вашей базы знаний.



## Actions

### `agiloft_attach_file`

Прикрепить файл к полю в записи Agiloft.

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `instanceUrl` | string | Yes | URL экземпляра Agiloft \(например, https://mycompany.agiloft.com\) |
| `knowledgeBase` | string | Yes | Имя базы знаний |
| `login` | string | Yes | Имя пользователя Agiloft |
| `password` | string | Yes | Пароль Agiloft |
| `table` | string | Yes | Имя таблицы \(например, "contracts"\) |
| `recordId` | string | Yes | ID записи, к которой нужно прикрепить файл |
| `fieldName` | string | Yes | Имя поля вложения |
| `file` | file | No | Файл для прикрепления |
| `fileName` | string | No | Имя для присвоения файлу \(по умолчанию — исходное имя файла\) |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `recordId` | string | ID записи, к которой был прикреплён файл |
| `fieldName` | string | Имя поля, к которому был прикреплён файл |
| `fileName` | string | Имя прикреплённого файла |
| `totalAttachments` | number | Общее количество файлов, прикреплённых в поле после операции |

| Parameter | Type | Description |

| `records` | json | Массив совпадающих записей со значениями их полей |
| `totalCount` | number | Общее количество совпадающих записей |
| `page` | number | Текущий номер страницы |
| `limit` | number | Записей на странице |
