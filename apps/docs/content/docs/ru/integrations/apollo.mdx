[Apollo.io](https://apollo.io/) — это ведущая платформа разведывательной информации о продажах и вовлечения, которая позволяет пользователям находить, обогащать и вовлекать контакты и компании в масштабе. Apollo.io объединяет обширную базу контактов с надёжными инструментами обогащения и автоматизации рабочих процессов, помогая командам продаж, маркетинга и рекрутинга ускорять рост.

С помощью Apollo.io вы можете:

- **Искать миллионы контактов и компаний**: Находить точные лиды с помощью расширенных фильтров
- **Обогащать лиды и аккаунты**: Заполнять недостающие детали проверенными данными и актуальной информацией
- **Управлять и организовывать записи CRM**: Поддерживать точность и актуальность данных о людях и компаниях
- **Автоматизировать охват**: Добавлять контакты в последовательности и создавать задачи для последующих действий непосредственно из Apollo.io

В Sim интеграция Apollo.io позволяет вашим агентам программно выполнять основные операции Apollo:

- **Поиск людей и компаний**: Используйте `apollo_people_search` для поиска новых лидов с помощью гибких фильтров.
- **Обогащение данных людей**: Используйте `apollo_people_enrich` для дополнения контактов проверенной информацией.
- **Массовое обогащение людей**: Используйте `apollo_people_bulk_enrich` для крупномасштабного обогащения нескольких контактов одновременно.
- **Поиск и обогащение компаний**: Используйте `apollo_company_search` и `apollo_company_enrich` для поиска и обновления ключевой информации о компаниях.

Это позволяет вашим агентам создавать мощные рабочие процессы для поиска потенциальных клиентов, обогащения CRM и автоматизации без ручного ввода данных или переключения вкладок. Интегрируйте Apollo.io как динамический источник данных и CRM-движок — позволяя вашим агентам выявлять, квалифицировать и охватывать лиды seamlessly в рамках их повседневных операций.

## Usage Instructions

Интегрирует Apollo.io в рабочий процесс. Поиск людей и компаний, обогащение контактных данных, управление контактами и аккаунтами CRM, добавление контактов в последовательности и создание задач.



## Actions

### `apollo_people_search`

Поиск людей в базе данных Apollo с использованием демографических фильтров

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `apiKey` | string | Yes | API-ключ Apollo |
| `person_titles` | array | No | Должности для поиска \(например, \["CEO", "VP of Sales"\]\) |
| `include_similar_titles` | boolean | No | Возвращать ли людей с должностями, похожими на person_titles |
| `person_locations` | array | No | Местоположения для поиска \(например, \["San Francisco, CA", "New York, NY"\]\) |
| `person_seniorities` | array | No | Уровни seniority: owner, founder, c_suite, partner, vp, head, director, manager, senior, entry, intern |
| `organization_ids` | array | No | ID организаций Apollo для фильтрации \(например, \["5e66b6381e05b4008c8331b8"\]\) |
| `organization_names` | array | No | Названия компаний для поиска \(устаревший фильтр\) |
| `organization_locations` | array | No | Местоположения штаб-квартир текущего работодателя людей \(например, \['texas', 'tokyo', 'spain'\]\) |
| `q_organization_domains_list` | array | No | Доменные имена работодателей \(например, \["apollo.io", "microsoft.com"\]\) — до 1 000, без www. или @ |
| `organization_num_employees_ranges` | array | No | Диапазоны количества сотрудников текущего работодателя. Каждая запись в формате \"min,max\" \(например, \["1,10", "250,500", "10000,20000"\]\) |
| `contact_email_status` | array | No | Статусы эл. почты для фильтрации: \"verified\", \"unverified\", \"likely to engage\", \"unavailable\" |
| `q_keywords` | string | No | Ключевые слова для поиска |
| `page` | number | No | Номер страницы для пагинации, по умолчанию 1 \(например, 1, 2, 3\) |
| `per_page` | number | No | Результатов на странице, по умолчанию 25, макс. 100 \(например, 25, 50, 100\) |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `people` | json | Массив людей, соответствующих критериям поиска |
| `page` | number | Текущий номер страницы |
| `per_page` | number | Результатов на странице |
| `total_entries` | number | Общее количество совпадающих записей |

### `apollo_people_enrich`

Обогатить данные одного человека с помощью Apollo

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `apiKey` | string | Yes | API-ключ Apollo |
| `first_name` | string | No | Имя человека |
| `last_name` | string | No | Фамилия человека |
| `name` | string | No | Полное имя человека \(альтернатива first_name/last_name\) |
| `id` | string | No | Apollo ID человека |
| `hashed_email` | string | No | MD5 или SHA-256 хеш эл. почты |
| `email` | string | No | Адрес эл. почты человека |
| `organization_name` | string | No | Название компании, где работает человек |
| `domain` | string | No | Домен компании \(например, \"apollo.io\", \"acme.com\"\) |
| `linkedin_url` | string | No | URL профиля LinkedIn |
| `reveal_personal_emails` | boolean | No | Раскрыть личные адреса эл. почты \(расходует кредиты\) |
| `reveal_phone_number` | boolean | No | Раскрыть номера телефонов \(расходует кредиты, требует webhook_url\) |
| `webhook_url` | string | No | URL webhook для асинхронной доставки номера телефона \(обязателен, когда reveal_phone_number = true\) |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `person` | json | Обогащённые данные человека из Apollo |
| `enriched` | boolean | Был ли человек успешно обогащён |

### `apollo_people_bulk_enrich`

Обогатить данные до 10 человек одновременно с помощью Apollo

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `apiKey` | string | Yes | API-ключ Apollo |
| `people` | array | Yes | Массив людей для обогащения \(макс. 10\) |
| `reveal_personal_emails` | boolean | No | Раскрыть личные адреса эл. почты \(расходует кредиты\) |
| `reveal_phone_number` | boolean | No | Раскрыть номера телефонов \(расходует кредиты, требует webhook_url\) |
| `webhook_url` | string | No | URL webhook для асинхронной доставки номера телефона \(обязателен, когда reveal_phone_number = true\) |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `matches` | json | Массив обогащённых людей \(null означает отсутствие совпадения\) |
| `total_requested_enrichments` | number | Общее количество записей, отправленных на обогащение |
| `unique_enriched_records` | number | Количество успешно обогащённых записей |
| `missing_records` | number | Количество записей, которые не удалось обогатить |
| `credits_consumed` | number | Количество кредитов Apollo, израсходованных этим запросом |

### `apollo_contact_create`

Создать новый контакт в вашей базе данных Apollo

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `apiKey` | string | Yes | API-ключ Apollo |
| `first_name` | string | Yes | Имя контакта |
| `last_name` | string | Yes | Фамилия контакта |
| `email` | string | No | Адрес эл. почты контакта |
| `title` | string | No | Должность \(например, "VP of Sales", "Software Engineer"\) |
| `account_id` | string | No | ID аккаунта Apollo для связывания \(например, "acc_abc123"\) |
| `owner_id` | string | No | ID пользователя-владельца контакта \(принимается Apollo, но не задокументирован официально для POST /contacts\) |
| `organization_name` | string | No | Название работодателя контакта \(например, "Apollo"\) |
| `website_url` | string | No | URL корпоративного веб-сайта \(например, "https://www.apollo.io/"\) |
| `label_names` | array | No | Списки/метки для добавления контакта \(например, \["Prospects"\]\) |
| `contact_stage_id` | string | No | Apollo ID стадии контакта |
| `present_raw_address` | string | No | Личное местоположение контакта \(например, "Atlanta, United States"\) |
| `direct_phone` | string | No | Основной номер телефона |
| `corporate_phone` | string | No | Рабочий/офисный номер телефона |
| `mobile_phone` | string | No | Мобильный номер телефона |
| `home_phone` | string | No | Домашний номер телефона |
| `other_phone` | string | No | Альтернативный номер телефона |
| `typed_custom_fields` | json | No | Значения пользовательских полей, сгруппированные по ID поля |
| `run_dedupe` | boolean | No | При true Apollo выполняет дедупликацию с существующими контактами |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `contact` | json | Данные созданного контакта из Apollo |
| `created` | boolean | Был ли контакт успешно создан |

### `apollo_contact_search`

Поиск контактов в аккаунте вашей команды Apollo. Лимит отображения: 50 000 записей (100 записей на странице, макс. 500 страниц). Используйте фильтры для сужения результатов. Требуется мастер-ключ.

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `apiKey` | string | Yes | API-ключ Apollo \(требуется мастер-ключ\) |
| `q_keywords` | string | No | Поиск по ключевым словам в контактах |
| `contact_stage_ids` | array | No | Фильтр по ID стадий контактов |
| `sort_by_field` | string | No | Поле сортировки: "contact_last_activity_date", "contact_created_at", "contact_updated_at", "contact_sanford_potential" |
| `sort_ascending` | boolean | No | Сортировка по возрастанию при true. По умолчанию — по убыванию. |
| `page` | number | No | Номер страницы для пагинации \(например, 1, 2, 3\) |
| `per_page` | number | No | Результатов на странице, макс. 100 \(например, 25, 50, 100\) |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `contacts` | json | Массив контактов, соответствующих критериям поиска |
| `pagination` | json | Информация о пагинации \(page, per_page, total_entries, total_pages\) |

### `apollo_contact_bulk_update`

Обновить до 500 существующих контактов одновременно в вашей базе данных Apollo. Каждый контакт должен включать поле id. Требуется мастер-ключ.

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `apiKey` | string | Yes | API-ключ Apollo \(требуется мастер-ключ\) |
| `contact_ids` | array | No | Массив ID контактов для обновления одинаковыми значениями \(макс. 500\). Используйте с owner_id/label_names для единообразных обновлений. |
| `owner_id` | string | No | При использовании contact_ids — применить этого владельца ко всем контактам |
| `label_names` | array | No | При использовании contact_ids — применить эти метки ко всем контактам |
| `contact_attributes` | json | No | Массив объектов контактов с индивидуальными обновлениями \(каждый должен включать id\) |
| `async` | boolean | No | При true обрабатывает обновление асинхронно. Поддерживается только с contact_ids. |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `contacts` | json | Обновлённые контакты \(синхронный ответ\) |
| `contact_ids` | json | ID обновлённых контактов |
| `entity_progress_job` | json | Дескриптор асинхронной задачи \(когда async=true\) |
| `job_id` | string | ID асинхронной задачи, извлечённый из entity_progress_job |
| `message` | string | Опциональное сообщение подтверждения от Apollo |
