[Номер телефона агента](https://agentphone.to/)Номер телефона агентаhttps://agentphone.to/является платформой для голосовой и текстовой связи, разработанной специально для ИИ-агентов, основанной на API. AgentPhone позволяет получать реальные телефонные номера, совершать исходящие голосовые звонки с использованием ИИ, отправлять SMS и сообщения iMessage, управлять разговорами и контактами, а также отслеживать использование – все через простой REST API, предназначенный для программного доступа.

**Почему стоит выбрать AgentPhone?**
- **Телефонная связь, разработанная специально для ИИ-агентов:** Предназначена для использования с ИИ-агентами – позволяет получать номера телефонов, совершать звонки и отправлять сообщения без необходимости заключения договоров с операторами связи или управления телефонной инфраструктурой.- **Голосовой и текстовый обмен в одном API:** Запускайте голосовые звонки с использованием искусственного интеллекта вместе со SMS, MMS и iMessage, используя один и тот же аккаунт и телефонные номера.- **Управление разговорами и транскриптами:** Каждый звонок возвращает упорядоченный транскрипт; каждая переписка отслеживается как разговор с полным историей и метаданными.- **Контакты уже есть:** Создавайте, ищите, обновляйте и удаляйте контакты в учетной записи, чтобы ваши сотрудники могли обращаться к людям по имени, а не просто по телефонным номерам.- **Видимость использования:** Проверьте лимиты плана, текущие значения и ежедневную/ежемесячную агрегацию, чтобы обеспечить соответствие рабочих процессов установленным правилам.
**Использование AgentPhone в Sim**

Интеграция Sim's AgentPhone позволяет напрямую связывать ваши рабочие процессы с AgentPhone с использованием ключа API. Благодаря 22 операциям, охватывающим номера телефонов, звонки, разговоры, контакты и использование, вы можете создавать мощные голосовые и текстовые автоматизации без написания кода на стороне сервера.

**Основные преимущества использования AgentPhone в Sim:**
- **Динамическое выделение номеров:** Выделяйте номера в США или Канаде по требованию – для каждого сотрудника, для каждого клиента или для каждой рабочей группы – и отзывайте их, когда они больше не нужны.- **Входящие голосовые звонки с использованием ИИ:** Возможность совершать звонки от имени агента с опциональным приветствием, переопределением голоса или системным запросом, а также чтение полного текста разговора в структурированном формате после завершения звонка.- **Двусторонняя связь:** Отправка SMS, MMS или iMessage, получение истории переписки и реагирование на входящие iMessage – все это можно делать прямо из вашего рабочего процесса.- **Управление контактами и метаданными:** Ведите список контактов на уровне аккаунта и добавляйте пользовательские метаданные в формате JSON к разговорам, чтобы последующие блоки могли реагировать на состояние.- **Оперативный анализ:** Получите текущие данные об использовании и ежедневные/ежемесячные отчеты, чтобы отслеживать потребление и соблюдать лимиты плана до совершения следующего звонка.
Независимо от того, создаете ли вы AI-агента для исходящих звонков, автоматизируете отправку SMS-сообщений, управляете двусторонними разговорами с клиентами или контролируете использование телефонов во всей вашей организации, AgentPhone в Sim предоставляет вам прямой и безопасный доступ ко всему API AgentPhone — без необходимости использования промежуточного программного обеспечения. Просто настройте свой ключ API, выберите необходимую операцию, и Sim позаботится обо всем остальном.

## Инструкция по использованию
Предоставьте вашему рабочему процессу возможность общаться по телефону. Настройте номера с поддержкой SMS и голосовых вызовов, отправляйте сообщения и реакции "tapback", совершайте исходящие голосовые звонки, управляйте разговорами и контактами, а также отслеживайте использование – все это с помощью одного ключа API AgentPhone.



## Действия
### `agentphone_create_call`

Запустить исходящий голосовой звонок от агента, использующего платформу AgentPhone.

#### Ввод
|  | Параметр | Тип | Необходимо | Описание |  || --------- | ---- | -------- | ----------- |
|  | `apiKey` | строка | Да | Ключ API для AgentPhone |  ||  | `agentId` | строка | Да | Оператор, который будет отвечать на звонок. |  ||  | `toNumber` | строка | Да | Номер телефона в формате E.164 (например, +14155551234) |  ||  | `fromNumberId` | строка | Нет | Идентификатор номера телефона для использования в качестве идентификатора звонящего. Этот номер должен принадлежать агенту. Если он не указан, используется первый номер, который был назначен агенту. |  ||  | `initialGreeting` | строка | Нет | Необязательное приветствие, которое произносится при ответе получателя. |  ||  | `voice` | строка | Нет | Переопределение идентификации голоса для этого звонка (по умолчанию используется голос, настроенный оператором) |  ||  | `systemPrompt` | строка | Нет | В случае необходимости использует встроенную языковую модель для общения вместо перенаправления к вашему веб-хуку. |  |
#### Вывод
|  | Параметр | Тип | Описание |  || --------- | ---- | ----------- |
|  | `id` | строка | Уникальный идентификатор звонка |  ||  | `agentId` | строка | Оператор, который принимает звонок |  ||  | `status` | строка | Статус первого звонка |  ||  | `toNumber` | строка | Номер телефона получателя |  ||  | `fromNumber` | строка | Информация о вызывающем абоненте, использованная для звонка. |  ||  | `phoneNumberId` | строка | Идентификатор номера телефона, используемого в качестве идентификатора звонящего. |  ||  | `direction` | строка | Направление звонка (входящее) |  ||  | `startedAt` | строка | Временная метка по стандарту ISO 8601 |  |
