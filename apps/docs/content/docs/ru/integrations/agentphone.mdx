[AgentPhone](https://agentphone.to/) — это голосовая и messaging-платформа, ориентированная на API, созданная для ИИ-агентов. AgentPhone позволяет выделять реальные телефонные номера, совершать исходящие ИИ-голосовые вызовы, отправлять SMS и iMessage, управлять перепиской и контактами, а также отслеживать использование — всё через простой REST API, разработанный для программного доступа.

**Почему AgentPhone?**
- **Нативная телефония для агентов:** Специально создана для ИИ-агентов — выделение номеров, совершение вызовов и отправка сообщений без контрактов с операторами и телефонной инфраструктуры.
- **Голос + сообщения в одном API:** Исходящие ИИ-голосовые вызовы вместе с SMS, MMS и iMessage с одного аккаунта и одних телефонных номеров.
- **Управление перепиской и транскриптами:** Каждый вызов возвращает упорядоченный транскрипт; каждая цепочка сообщений отслеживается как переписка с полной историей и метаданными.
- **Встроенные контакты:** Создание, поиск, обновление и удаление контактов в аккаунте, чтобы ваши агенты могли ссылаться на людей по имени, а не по номерам телефонов.
- **Видимость использования:** Проверка лимитов плана, текущих показателей и дневной/месячной агрегации, чтобы рабочие процессы оставались в рамках ограничений.

**Использование AgentPhone в Sim**

Интеграция Sim с AgentPhone соединяет ваши агентные рабочие процессы напрямую с AgentPhone с помощью API-ключа. С 22 операциями, охватывающими номера, вызовы, переписку, контакты и использование, вы можете создавать мощные голосовые и messaging-автоматизации без написания серверного кода.

**Ключевые преимущества использования AgentPhone в Sim:**
- **Динамическое выделение номеров:** Резервируйте номера США или Канады на лету — для каждого агента, клиента или рабочего процесса — и освобождайте их, когда они больше не нужны.
- **Исходящие ИИ-голосовые вызовы:** Совершайте вызовы от агента с опциональным приветствием, переопределением голоса или системным промптом и получайте полный транскрипт в виде структурированных данных после завершения вызова.
- **Двусторонний обмен сообщениями:** Отправляйте SMS, MMS или iMessage, получайте историю переписки и реагируйте на входящие iMessage — всё внутри вашего рабочего процесса.
- **Управление контактами и метаданными:** Ведите список контактов на уровне аккаунта и прикрепляйте пользовательские JSON-метаданные к перепискам, чтобы последующие блоки могли ветвиться в зависимости от состояния.
- **Операционная аналитика:** Получайте текущую статистику использования и дневные/месячные разбивки для мониторинга потребления и соблюдения лимитов плана перед следующим вызовом.

Создаёте ли вы исходящего ИИ-голосового агента, выполняете автоматизированные SMS-рассылки, управляете двусторонней перепиской с клиентами или отслеживаете использование телефонии в вашей организации — AgentPhone в Sim предоставляет вам прямой и безопасный доступ к полному API AgentPhone, без промежуточного ПО. Просто настройте ваш API-ключ, выберите нужную операцию, и Sim сделает всё остальное.

## Usage Instructions

Дайте вашему рабочему процессу телефон. Выделяйте номера с поддержкой SMS и голоса, отправляйте сообщения и реакции tapback, совершайте исходящие голосовые вызовы, управляйте перепиской и контактами, а также отслеживайте использование — всё через единый API-ключ AgentPhone.



## Actions

### `agentphone_create_call`

Инициировать исходящий голосовой вызов от агента AgentPhone

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `apiKey` | string | Yes | API-ключ AgentPhone |
| `agentId` | string | Yes | Агент, который будет обрабатывать вызов |
| `toNumber` | string | Yes | Номер телефона для вызова в формате E.164 \(например, +14155551234\) |
| `fromNumberId` | string | No | ID номера телефона для использования в качестве Caller ID. Должен принадлежать агенту. Если не указан, используется первый назначенный номер агента. |
| `initialGreeting` | string | No | Опциональное приветствие, произносимое при ответе получателя |
| `voice` | string | No | ID голоса для переопределения в этом вызове \(по умолчанию используется настроенный голос агента\) |
| `systemPrompt` | string | No | При указании используется встроенная LLM для разговора вместо пересылки на ваш webhook |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `id` | string | Уникальный идентификатор вызова |
| `agentId` | string | Агент, обрабатывающий вызов |
| `status` | string | Начальный статус вызова |
| `toNumber` | string | Номер телефона назначения |
| `fromNumber` | string | Caller ID, использованный для вызова |
| `phoneNumberId` | string | ID номера телефона, использованного в качестве Caller ID |
| `direction` | string | Направление вызова \(outbound\) |
| `startedAt` | string | Метка времени ISO 8601 |
