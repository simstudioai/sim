[AgentMail](https://agentmail.to/) — это почтовая платформа, ориентированная на API, созданная для агентов и автоматизации. AgentMail позволяет создавать почтовые ящики на лету, отправлять и получать сообщения, отвечать в цепочках, управлять черновиками и организовывать переписку с помощью меток — всё через простой REST API, разработанный для программного доступа.

**Почему AgentMail?**
- **Нативная почта для агентов:** Специально создана для ИИ-агентов и автоматизации — создание ящиков, отправка сообщений и управление цепочками без пользовательского интерфейса.
- **Полный жизненный цикл писем:** Отправка новых сообщений, ответы в цепочках, пересылка писем, управление черновиками и планирование отправки — всё через единый API.
- **Управление цепочками и перепиской:** Организация писем в цепочки с полной поддержкой чтения, ответа, пересылки и меток для структурированного отслеживания переписки.
- **Рабочий процесс с черновиками:** Создание черновиков, их обновление, планирование отправки и отправка после утверждения — идеально для рабочих процессов с проверкой перед отправкой.
- **Организация с помощью меток:** Пометка цепочек и сообщений пользовательскими метками для фильтрации, маршрутизации и последующей автоматизации.

**Использование AgentMail в Sim**

Интеграция Sim с AgentMail соединяет ваши агентные рабочие процессы напрямую с AgentMail с помощью API-ключа. С 20 операциями, охватывающими ящики, цепочки, сообщения и черновики, вы можете создавать мощные почтовые автоматизации без написания серверного кода.

**Ключевые преимущества использования AgentMail в Sim:**
- **Динамическое создание ящиков:** Создавайте новые ящики на лету для каждого агента, рабочего процесса или клиента — идеально для многопользовательской обработки почты.
- **Автоматизированная обработка писем:** Просматривайте и читайте входящие сообщения, затем запускайте последующие действия на основе содержимого, отправителя или меток.
- **Диалоговая почта:** Отвечайте в цепочках и пересылайте сообщения, чтобы переписка продолжалась естественным образом в рамках ваших автоматизированных рабочих процессов.
- **Рабочие процессы с черновиками и проверкой:** Создавайте черновики, обновляйте их с помощью контента, сгенерированного ИИ, и отправляйте после утверждения — идеально для паттернов с участием человека.
- **Организация почты:** Применяйте метки к цепочкам и сообщениям для категоризации, фильтрации и маршрутизации писем через ваш конвейер автоматизации.

Создаёте ли вы ИИ-почтового ассистента, автоматизируете ответы службы поддержки, обрабатываете входящие лиды или управляете многоагентными почтовыми рабочими процессами — AgentMail в Sim предоставляет вам прямой и безопасный доступ к полному API AgentMail, без промежуточного ПО. Просто настройте ваш API-ключ, выберите нужную операцию, и Sim сделает всё остальное.

## Usage Instructions

Интегрируйте AgentMail в ваш рабочий процесс. Создавайте и управляйте почтовыми ящиками, отправляйте и получайте сообщения, отвечайте в цепочках, управляйте черновиками и организуйте цепочки с помощью меток. Требуется API-ключ.



## Actions

### `agentmail_create_draft`

Создать новый черновик письма в AgentMail

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `apiKey` | string | Yes | API-ключ AgentMail |
| `inboxId` | string | Yes | ID ящика, в котором создать черновик |
| `to` | string | No | Адреса получателей \(через запятую\) |
| `subject` | string | No | Тема черновика |
| `text` | string | No | Текст черновика в простом формате |
| `html` | string | No | Тело черновика в HTML |
| `cc` | string | No | Адреса CC-получателей \(через запятую\) |
| `bcc` | string | No | Адреса BCC-получателей \(через запятую\) |
| `inReplyTo` | string | No | ID сообщения, на которое создаётся ответ |
| `sendAt` | string | No | Метка времени ISO 8601 для планирования отправки |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `draftId` | string | Уникальный идентификатор черновика |
| `inboxId` | string | Ящик, которому принадлежит черновик |
| `subject` | string | Тема черновика |
| `to` | array | Адреса получателей |
| `cc` | array | Адреса CC |
| `bcc` | array | Адреса BCC |
| `text` | string | Текстовое содержимое |
| `html` | string | HTML-содержимое |
| `preview` | string | Текст предпросмотра черновика |
| `labels` | array | Метки, назначенные черновику |
| `inReplyTo` | string | ID сообщения, на которое отвечает черновик |
| `sendStatus` | string | Статус отправки \(scheduled, sending, failed\) |
| `sendAt` | string | Запланированное время отправки |
| `createdAt` | string | Метка времени создания |
| `updatedAt` | string | Метка времени последнего обновления |
