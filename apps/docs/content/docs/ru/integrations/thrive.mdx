## Инструкции по использованию

Интегрируйте Thrive Learning в рабочий процесс. Manage user lifecycle, audiences and their members and managers, content assignments and enrolments, learning completions, content and activity records, CPD, tags, and skills.



## Действия

### `thrive_create_user`

Создание нового user in Thrive.

#### Вход

| Параметр | Тип | Обязательный | Описание |
| --------- | ---- | -------- | ----------- |
| `tenantId` | string | Да | Thrive Tenant ID \(used as the Basic auth username\) |
| `apiKey` | string | Да | Thrive Ключ API \(used as the Basic auth password\) |
| `host` | string | Нет | Region-specific API host |
| `ref` | string | Да | Your organisation's unique identifier for this individual |
| `firstName` | string | Да | The given name of the individual |
| `lastName` | string | Да | The family name of the individual |
| `email` | string | Нет | The email address for the user \(required unless loginMethod is 'ref'\) |
| `loginMethod` | string | Нет | How the user logs in: 'email' or 'ref' \(defaults to 'email'\) |
| `role` | string | Нет | Role assigned: 'administrator', 'learneradmin', or 'learner' \(defaults to 'learner'\) |
| `jobTitle` | string | Нет | Name of this individual's role in your organisation |
| `managerRef` | string | Нет | Your organisation's unique identifier for this individual's line manager |
| `startDate` | string | Нет | Date this individual started with your organisation \(ISO 8601\) |
| `endDate` | string | Нет | Date this individual left your organisation \(ISO 8601\) |
| `timeZone` | string | Нет | The user's preferred timezone \(tenant по умолчанию if omitted\) |
| `languageCode` | string | Нет | The user's preferred language (например, 'en-gb'\) |
| `sso` | boolean | Нет | Указывает,  account is managed by an authentication provider |
| `domain` | string | Нет | Domain this individual is associated with |
| `additionalFields` | string | Нет | JSON object of custom field key-value pairs. Example: \{"department":"Sales"\} |

#### Выход

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `user` | object | The created user |
|   ↳ `id` | string | The user ID |
|   ↳ `loginMethod` | string | How the user logs in |
|   ↳ `ref` | string | Your organisation's unique identifier for the user |
|   ↳ `email` | string | The email address for the user |
|   ↳ `firstName` | string | The given name of the individual |
|   ↳ `lastName` | string | The family name of the individual |
|   ↳ `role` | string | Role assigned to this individual |
|   ↳ `jobTitle` | string | Name of this individual's role |
|   ↳ `managerRef` | string | The line manager's ref |
|   ↳ `startDate` | string | Date started with the organisation |
|   ↳ `endDate` | string | Date left the organisation |
|   ↳ `timeZone` | string | The user's preferred timezone |
|   ↳ `languageCode` | string | The user's preferred language |
|   ↳ `active` | boolean | Указывает,  account is active or suspended |
|   ↳ `createdAt` | string | Date/time the user was created |
|   ↳ `updatedAt` | string | Date/time the user was last modified |
|   ↳ `sso` | boolean | Указывает,  account is managed by an auth provider |
|   ↳ `domain` | string | Domain this individual is associated with |
|   ↳ `additionalFields` | json | Custom field values for this user |

| `perPage` | number | Нет | Number of results per page \(1-1000, по умолчанию 100\) |

| `omitStatuses` | string | Нет | Comma-separated statuses to exclude: active, inactive, expired, new |

|   ↳ `lastLogin` | string | Last login timestamp \(ISO 8601\) |
|   ↳ `tags` | json | Tag membership (например, skills\) |
