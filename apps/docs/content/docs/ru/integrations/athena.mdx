[Amazon Athena](https://aws.amazon.com/athena/) — это интерактивный сервис запросов от AWS, который позволяет легко анализировать данные непосредственно в Amazon S3 с использованием стандартного SQL. Athena является бессерверным сервисом, поэтому вам не нужно управлять инфраструктурой, и вы платите только за выполненные запросы.

С помощью Athena вы можете:

- **Выполнять запросы к данным в S3**: запускайте SQL-запросы непосредственно к данным, хранящимся в Amazon S3, без необходимости загрузки в базу данных
- **Поддерживать множество форматов**: запрашивайте данные в CSV, JSON, Parquet, ORC, Avro и других распространённых форматах
- **Интегрироваться с AWS Glue**: используйте каталог данных AWS Glue для управления метаданными таблиц и схемами
- **Автоматически масштабироваться**: обрабатывайте запросы любого размера без выделения серверов или кластеров
- **Сохранять и переиспользовать запросы**: создавайте именованные запросы для часто используемых SQL-выражений

В Sim интеграция с Athena позволяет вашим агентам выполнять SQL-запросы к данным в S3, проверять статус выполнения запросов, получать результаты и управлять сохранёнными запросами — всё в рамках ваших агентных рабочих процессов. Поддерживаемые операции включают:

- **Запуск запроса**: выполнение SQL-запросов к вашим данным в S3
- **Получение статуса выполнения запроса**: проверка статуса и деталей выполняющегося или завершённого запроса
- **Получение результатов запроса**: извлечение результатов завершённого запроса
- **Остановка запроса**: отмена выполняющегося запроса
- **Список выполненных запросов**: просмотр недавних идентификаторов выполненных запросов
- **Создание именованного запроса**: сохранение запроса для повторного использования
- **Получение именованного запроса**: получение деталей сохранённого запроса
- **Список именованных запросов**: просмотр всех сохранённых идентификаторов запросов

Эта интеграция позволяет агентам Sim автоматизировать задачи анализа данных с помощью AWS Athena, обеспечивая рабочие процессы, которые запрашивают, отслеживают и управляют крупномасштабными данными в S3 без ручных усилий или управления инфраструктурой.

## Инструкция по использованию

Интегрируйте AWS Athena в рабочие процессы. Выполняйте SQL-запросы к данным в S3, проверяйте статус запросов, получайте результаты, управляйте именованными запросами и просматривайте список выполненных запросов. Требуется ключ доступа AWS и секретный ключ доступа.



## Действия

### `athena_start_query`

Запуск выполнения SQL-запроса в AWS Athena

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `awsRegion` | string | Yes | Регион AWS (например, us-east-1) |
| `awsAccessKeyId` | string | Yes | Идентификатор ключа доступа AWS |
| `awsSecretAccessKey` | string | Yes | Секретный ключ доступа AWS |
| `queryString` | string | Yes | Строка SQL-запроса для выполнения |
| `database` | string | No | Имя базы данных в каталоге |
| `catalog` | string | No | Имя каталога данных (по умолчанию: AwsDataCatalog) |
| `outputLocation` | string | No | Расположение в S3 для результатов запроса (например, s3://bucket/path/) |
| `workGroup` | string | No | Рабочая группа для выполнения запроса (по умолчанию: primary) |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `queryExecutionId` | string | Уникальный идентификатор запущенного выполнения запроса |

### `athena_get_query_execution`

Получение статуса и деталей выполнения запроса Athena

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `awsRegion` | string | Yes | Регион AWS (например, us-east-1) |
| `awsAccessKeyId` | string | Yes | Идентификатор ключа доступа AWS |
| `awsSecretAccessKey` | string | Yes | Секретный ключ доступа AWS |
| `queryExecutionId` | string | Yes | Идентификатор выполнения запроса для проверки |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `queryExecutionId` | string | Идентификатор выполнения запроса |
| `query` | string | Строка SQL-запроса |
| `state` | string | Состояние запроса (QUEUED, RUNNING, SUCCEEDED, FAILED, CANCELLED) |
| `stateChangeReason` | string | Причина изменения состояния (например, сообщение об ошибке) |
| `statementType` | string | Тип выражения (DDL, DML, UTILITY) |
| `database` | string | Имя базы данных |
| `catalog` | string | Имя каталога данных |
| `workGroup` | string | Имя рабочей группы |
| `submissionDateTime` | number | Время отправки запроса (Unix эпоха, мс) |
| `completionDateTime` | number | Время завершения запроса (Unix эпоха, мс) |
| `dataScannedInBytes` | number | Объём просканированных данных в байтах |
| `engineExecutionTimeInMillis` | number | Время выполнения движка в миллисекундах |
| `queryPlanningTimeInMillis` | number | Время планирования запроса в миллисекундах |
| `queryQueueTimeInMillis` | number | Время нахождения запроса в очереди в миллисекундах |
| `totalExecutionTimeInMillis` | number | Общее время выполнения в миллисекундах |
| `outputLocation` | string | Расположение результатов запроса в S3 |

### `athena_get_query_results`

Получение результатов завершённого выполнения запроса Athena

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `awsRegion` | string | Yes | Регион AWS (например, us-east-1) |
| `awsAccessKeyId` | string | Yes | Идентификатор ключа доступа AWS |
| `awsSecretAccessKey` | string | Yes | Секретный ключ доступа AWS |
| `queryExecutionId` | string | Yes | Идентификатор выполнения запроса для получения результатов |
| `maxResults` | number | No | Максимальное количество строк для возврата (1–999) |
| `nextToken` | string | No | Токен пагинации из предыдущего запроса |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `columns` | array | Метаданные столбцов (имя и тип) |
| `rows` | array | Строки результатов в виде объектов ключ-значение |
| `nextToken` | string | Токен пагинации для следующей страницы результатов |
| `updateCount` | number | Количество затронутых строк (для выражений INSERT/UPDATE) |
