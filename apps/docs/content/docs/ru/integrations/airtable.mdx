[Airtable](https://airtable.com/) — это мощная облачная платформа, сочетающая функциональность базы данных с простотой электронной таблицы. Она позволяет пользователям создавать гибкие базы данных для организации, хранения и совместной работы с информацией.

С Airtable вы можете:

- **Создавать пользовательские базы данных**: Стройте индивидуальные решения для управления проектами, контент-календарей, отслеживания запасов и многого другого
- **Визуализировать данные**: Просматривайте информацию в виде сетки, канбан-доски, календаря или галереи
- **Автоматизировать рабочие процессы**: Настраивайте триггеры и действия для автоматизации повторяющихся задач
- **Интегрироваться с другими инструментами**: Подключайтесь к сотням других приложений через встроенные интеграции и API

В Sim интеграция с Airtable позволяет вашим агентам программно взаимодействовать с вашими базами Airtable. Это обеспечивает бесшовные операции с данными, такие как получение информации, создание новых записей и обновление существующих данных — всё в рамках рабочих процессов ваших агентов. Используйте Airtable как динамический источник или приёмник данных для ваших агентов, позволяя им получать доступ к структурированной информации и манипулировать ею в рамках процессов принятия решений и выполнения задач.

## Инструкция по использованию

Интегрирует Airtable в рабочий процесс. Позволяет перечислять базы, таблицы (со схемой), а также создавать, получать, перечислять или обновлять записи. Также может использоваться в режиме триггера для запуска рабочего процесса при обновлении таблицы Airtable.



## Действия

### `airtable_list_bases`

Перечисление всех баз, к которым имеет доступ аутентифицированный пользователь

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `offset` | string | No | Смещение пагинации для получения дополнительных баз |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `bases` | array | Массив баз Airtable с id, name и permissionLevel |
|   ↳ `id` | string | ID базы \(начинается с "app"\) |
|   ↳ `name` | string | Название базы |
|   ↳ `permissionLevel` | string | Уровень доступа \(none, read, comment, edit, create\) |
| `metadata` | json | Метаданные пагинации и количества |
|   ↳ `offset` | string | Смещение для следующей страницы результатов |
|   ↳ `totalBases` | number | Количество возвращённых баз |

### `airtable_list_tables`

Перечисление всех таблиц и их схемы в базе Airtable

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `baseId` | string | Yes | ID базы Airtable \(начинается с "app", например, "appXXXXXXXXXXXXXX"\) |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `tables` | array | Список таблиц в базе с их схемой |
|   ↳ `id` | string | ID таблицы \(начинается с "tbl"\) |
|   ↳ `name` | string | Название таблицы |
|   ↳ `description` | string | Описание таблицы |
|   ↳ `primaryFieldId` | string | ID первичного поля |
|   ↳ `fields` | array | Список полей в таблице |
|     ↳ `id` | string | ID поля \(начинается с "fld"\) |
|     ↳ `name` | string | Название поля |
|     ↳ `type` | string | Тип поля \(singleLineText, multilineText, number, checkbox, singleSelect, multipleSelects, date, dateTime, attachment, linkedRecord и др.\) |
|     ↳ `description` | string | Описание поля |
|     ↳ `options` | json | Специфичные для поля параметры \(варианты выбора и т.д.\) |
| `metadata` | json | Информация о базе и метаданные количества |
|   ↳ `baseId` | string | ID запрошенной базы |
|   ↳ `totalTables` | number | Количество таблиц в базе |

### `airtable_list_records`

Чтение записей из таблицы Airtable

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `baseId` | string | Yes | ID базы Airtable \(начинается с "app", например, "appXXXXXXXXXXXXXX"\) |
| `tableId` | string | Yes | ID таблицы \(начинается с "tbl"\) или название таблицы |
| `maxRecords` | number | No | Максимальное количество возвращаемых записей \(по умолчанию: все записи\) |
| `filterFormula` | string | No | Формула для фильтрации записей \(например, "\(\{Имя поля\} = \'Значение\'\)"\) |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `records` | array | Массив полученных записей Airtable |
|   ↳ `id` | string | ID записи |
|   ↳ `createdTime` | string | Временная метка создания записи |
|   ↳ `fields` | json | Значения полей записи |
| `metadata` | json | Метаданные операции, включая смещение пагинации и общее количество записей |
|   ↳ `offset` | string | Смещение пагинации для следующей страницы |
|   ↳ `totalRecords` | number | Количество возвращённых записей |

### `airtable_get_record`

Получение одной записи из таблицы Airtable по её ID

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `baseId` | string | Yes | ID базы Airtable \(начинается с "app", например, "appXXXXXXXXXXXXXX"\) |
| `tableId` | string | Yes | ID таблицы \(начинается с "tbl"\) или название таблицы |
| `recordId` | string | Yes | ID записи для получения \(начинается с "rec", например, "recXXXXXXXXXXXXXX"\) |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `record` | json | Полученная запись Airtable |
|   ↳ `id` | string | ID записи |
|   ↳ `createdTime` | string | Временная метка создания записи |
|   ↳ `fields` | json | Значения полей записи |
| `metadata` | json | Метаданные операции |
|   ↳ `recordCount` | number | Количество возвращённых записей \(всегда 1\) |

### `airtable_create_records`

Запись новых записей в таблицу Airtable

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `baseId` | string | Yes | ID базы Airtable \(начинается с "app", например, "appXXXXXXXXXXXXXX"\) |
| `tableId` | string | Yes | ID таблицы \(начинается с "tbl"\) или название таблицы |
| `records` | json | Yes | Массив записей для создания, каждая с объектом `fields` |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `records` | array | Массив созданных записей Airtable |
|   ↳ `id` | string | ID записи |
|   ↳ `createdTime` | string | Временная метка создания записи |
|   ↳ `fields` | json | Значения полей записи |
| `metadata` | json | Метаданные операции |
|   ↳ `recordCount` | number | Количество созданных записей |

### `airtable_update_record`

Обновление существующей записи в таблице Airtable по ID

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `baseId` | string | Yes | ID базы Airtable \(начинается с "app", например, "appXXXXXXXXXXXXXX"\) |
| `tableId` | string | Yes | ID таблицы \(начинается с "tbl"\) или название таблицы |
| `recordId` | string | Yes | ID записи для обновления \(начинается с "rec", например, "recXXXXXXXXXXXXXX"\) |
| `fields` | json | Yes | Объект, содержащий названия полей и их новые значения |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `record` | json | Обновлённая запись Airtable |
|   ↳ `id` | string | ID записи |
|   ↳ `createdTime` | string | Временная метка создания записи |
|   ↳ `fields` | json | Значения полей записи |
| `metadata` | json | Метаданные операции |
|   ↳ `recordCount` | number | Количество обновлённых записей \(всегда 1\) |
|   ↳ `updatedFields` | array | Список названий полей, которые были обновлены |

### `airtable_update_multiple_records`

Обновление нескольких существующих записей в таблице Airtable

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `baseId` | string | Yes | ID базы Airtable \(начинается с "app", например, "appXXXXXXXXXXXXXX"\) |
| `tableId` | string | Yes | ID таблицы \(начинается с "tbl"\) или название таблицы |
| `records` | json | Yes | Массив записей для обновления, каждая с `id` и объектом `fields` |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `records` | array | Массив обновлённых записей Airtable |
|   ↳ `id` | string | ID записи |
|   ↳ `createdTime` | string | Временная метка создания записи |
|   ↳ `fields` | json | Значения полей записи |
| `metadata` | json | Метаданные операции |
|   ↳ `recordCount` | number | Количество обновлённых записей |
|   ↳ `updatedRecordIds` | array | Список ID обновлённых записей |

### `airtable_get_base_schema`

Получение схемы всех таблиц, полей и представлений в базе Airtable

#### Input

| Parameter | Type | Required | Description |

| `baseId` | string | Yes | ID базы Airtable \(начинается с "app", например, "appXXXXXXXXXXXXXX"\) |

#### Output

| Parameter | Type | Description |

## Триггеры

**Триггер** — это блок, который запускает рабочий процесс при наступлении события в этом сервисе.

### Airtable Webhook

Запуск рабочего процесса при изменениях записей Airtable, таких как создание, обновление и удаление (требуются учётные данные Airtable)

#### Configuration

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `triggerCredentials` | string | Yes | Для этого триггера требуются учётные данные Airtable для доступа к вашей учётной записи. |
| `baseId` | string | Yes | ID базы Airtable, которую будет отслеживать этот вебхук. |
| `tableId` | string | Yes | ID таблицы в базе, которую будет отслеживать вебхук. |
| `includeCellValues` | boolean | No | Включите для получения полных данных записи в полезной нагрузке, а не только изменений. |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `payloads` | array | Полезные нагрузки изменений Airtable |
|   ↳ `timestamp` | string | Временная метка изменения |
|   ↳ `baseTransactionNumber` | number | Номер транзакции |
| `latestPayload` | object | Самая последняя полезная нагрузка от Airtable |
|   ↳ `timestamp` | string | Временная метка изменения в формате ISO 8601 |
|   ↳ `baseTransactionNumber` | number | Номер транзакции |
|   ↳ `payloadFormat` | string | Версия формата полезной нагрузки \(например, v0\) |
|   ↳ `actionMetadata` | object | Метаданные о том, кто внёс изменение |
|     ↳ `source` | string | Источник изменения \(например, client, publicApi\) |
|     ↳ `sourceMetadata` | object | Метаданные источника, включая информацию о пользователе |
|       ↳ `user` | object | Пользователь, внёсший изменение |
|         ↳ `id` | string | ID пользователя |
|         ↳ `email` | string | Email пользователя |
|         ↳ `name` | string | Имя пользователя |
|         ↳ `permissionLevel` | string | Уровень доступа пользователя |
|   ↳ `changedTablesById` | object | Таблицы, которые были изменены \(по ID таблицы\) |
|     ↳ `changedRecordsById` | object | Изменённые записи по ID записи |
|       ↳ `current` | object | Текущее состояние записи |
|         ↳ `cellValuesByFieldId` | object | Значения ячеек по ID поля |
|     ↳ `createdRecordsById` | object | Созданные записи по ID |
|     ↳ `destroyedRecordIds` | array | Массив ID удалённых записей |
| `airtableChanges` | array | Изменения, внесённые в таблицу Airtable |
|   ↳ `tableId` | string | ID таблицы |
|   ↳ `recordId` | string | ID записи |
|   ↳ `changeType` | string | Тип изменения \(created, changed, destroyed\) |
|   ↳ `cellValuesByFieldId` | object | Значения ячеек по ID поля |
