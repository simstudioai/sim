[Wiza](https://wiza.co/) is a B2B contact data platform that finds and verifies emails and phone numbers for sales, recruiting, and marketing teams. Wiza pairs a global prospect database with real-time enrichment, so the data you act on stays accurate and deliverable.

With Wiza, you can:

- **Search a global prospect database**: Find people using person, company, and financial filters
- **Enrich companies**: Resolve firmographic data from a name, domain, or LinkedIn URL
- **Reveal individual contacts**: Получение verified work emails, personal emails, and mobile phone numbers
- **Track credit usage**: Check remaining email, phone, export, and API credits at any time

In Sim, the Wiza integration lets your agents drive prospecting and enrichment workflows end-to-end:

- **Prospect Search**: Use `wiza_prospect_search` to discover prospects matching detailed filters.
- **Company Enrichment**: Use `wiza_company_enrichment` to enrich a company from name, domain, LinkedIn ID, or LinkedIn slug.
- **Start Individual Reveal**: Use `wiza_start_individual_reveal` to begin enrichment for a contact via LinkedIn URL, name + company, or email — with configurable enrichment depth (`none`, `partial`, `phone`, `full`).
- **Получение Individual Reveal**: Use `wiza_get_individual_reveal` to poll a reveal and retrieve verified emails, phones, and full company context.
- **Получение Credits**: Use `wiza_get_credits` to monitor remaining credit balances before scaling up runs.

Individual reveals are asynchronous: start a reveal, then poll `wiza_get_individual_reveal` until `is_complete` is `true`. Combine these tools to build agents that source, enrich, and qualify leads on demand — without leaving your workflow.

## Инструкции по использованию

{/* MANUAL-CONTENT-START:usage */}
### Wiza Ключ API Setup

Wiza authenticates via Ключ API. To get yours:

1. Log in to your [Wiza account](https://app.wiza.co/).
2. Open **Settings → API** and generate a new Ключ API (or copy an existing one).
3. In Sim, open the Wiza block and paste the key into the **Wiza Ключ API** field.

The same key is used for every Wiza operation. Wiza enforces a rate limit of 30 requests per minute (43,200 per day) per key.

### Individual Reveals Are Asynchronous

`wiza_start_individual_reveal` returns immediately with a reveal `id` and a `status` of `queued` or `resolving`. To retrieve the enriched contact data, call `wiza_get_individual_reveal` with that `id` and check `is_complete`. Possible statuses are `queued`, `resolving`, `finished`, and `failed`.

For real-time delivery without polling, pass a `callback_url` when starting the reveal — Wiza will POST the completed payload to that URL.

### Enrichment Levels and Credits

When starting an individual reveal, choose the enrichment level that matches the data you need:

- **`none`** — Identity only, no contact info (no credit spend)
- **`partial`** — Verified work email (email credits)
- **`phone`** — Mobile phone (phone credits)
- **`full`** — Email + phone (email + phone credits)

Use `wiza_get_credits` to monitor remaining email, phone, export, and API credits before running large batches.

## Действия

### `wiza_prospect_search`

Search Wiza's database of prospects using person, company, and financial filters

#### Вход

| Параметр | Тип | Обязательный | Описание |
| --------- | ---- | -------- | ----------- |
| `apiKey` | string | Да | Wiza Ключ API |
| `size` | number | Нет | Number of sample profiles to return \(0-30, по умолчанию 0\) |
| `filters` | object | Нет | Full filters object \(overrides individual filter params if provided\) |
| `first_name` | array | Нет | Exact first names to match (например, \["John", "Jane"\]\) |
| `last_name` | array | Нет | Exact last names to match |
| `job_title` | array | Нет | Job titles to include/exclude (например, \[\{"v":"CEO","s":"i"\},\{"v":"CTO","s":"e"\}\]\) |
| `job_title_level` | array | Нет | Seniority levels (например, \["cxo", "director", "manager"\]\) |
| `job_role` | array | Нет | Job role categories (например, \["sales", "engineering", "marketing"\]\) |
| `job_sub_role` | array | Нет | Detailed role categories (например, \["software", "product"\]\) |
| `location` | array | Нет | Person's location filters \(city/state/country with include/exclude\) |
| `skill` | array | Нет | Professional skills (например, \["python", "marketing"\]\) |
| `school` | array | Нет | Educational institutions |
| `major` | array | Нет | Field of study |
| `linkedin_slug` | array | Нет | LinkedIn profile slugs |
| `job_company` | array | Нет | Current company filters \(include/exclude\) |
| `past_company` | array | Нет | Past company filters |
| `company_location` | array | Нет | Company HQ location filters |
| `company_industry` | array | Нет | Company industry filters \(include/exclude\) |
| `company_size` | array | Нет | Company headcount brackets (например, \["1-10", "11-50", "51-200"\]\) |
| `company_type` | array | Нет | Company type (например, \["private", "public", "educational"\]\) |

#### Выход

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `total` | number | Total number of matching prospects |
| `profiles` | array | Sample profiles matching the filter criteria |

### `wiza_individual_reveal`

Reveal a contact via LinkedIn URL, name + company/domain, or email. Starts the reveal and polls until it resolves. Uses 2 credits per valid email and 5 credits per phone, charged only on success.

#### Вход

| Параметр | Тип | Обязательный | Описание |
| --------- | ---- | -------- | ----------- |
| `apiKey` | string | Да | Wiza Ключ API |
| `enrichment_level` | string | Да | Enrichment depth: none, partial, phone, or full |
| `profile_url` | string | Нет | LinkedIn profile URL (например, https://linkedin.com/in/johndoe\) |
| `full_name` | string | Нет | Full name \(used with company or domain\) |
| `company` | string | Нет | Company name \(used with full_name\) |
| `domain` | string | Нет | Company domain \(used with full_name\) |
| `email` | string | Нет | Email address \(use alone or with other identifiers\) |
| `accept_work` | boolean | Нет | Whether to accept work emails \(email_options\) |
| `accept_personal` | boolean | Нет | Whether to accept personal emails \(email_options\) |

#### Выход

| Parameter | Type | Description |

| `company_size` | number | Employee count |

| `company_type` | string | Company type |

| `company_locality` | string | City |

Получение the remaining credits on your Wiza account

#### Вход

| Параметр | Тип | Обязательный | Описание |
| --------- | ---- | -------- | ----------- |
| `apiKey` | string | Да | Wiza Ключ API |

#### Выход

| Parameter | Type | Description |
| --------- | ---- | ----------- |

| `phone_credits` | json | Remaining phone credits \(number or "unlimited"\) |

| `api_credits` | number | Remaining API credits |
