[WhatsApp](https://www.whatsapp.com/) — это глобально популярная платформа обмена сообщениями, обеспечивающая безопасную и надёжную связь между людьми и компаниями.

WhatsApp Business API предоставляет организациям мощные возможности для:

- **Вовлечения клиентов**: Отправляйте персонализированные сообщения, уведомления и обновления непосредственно в предпочитаемое приложение для обмена сообщениями клиентов
- **Автоматизации бесед**: Создавайте интерактивных чат-ботов и системы автоматических ответов для типовых запросов
- **Улучшения поддержки**: Обеспечивайте обслуживание клиентов в реальном времени через знакомый интерфейс с поддержкой мультимедиа
- **Увеличения конверсии**: Содействуйте транзакциям и последующим взаимодействиям с клиентами в безопасной, соответствующей требованиям среде

В Sim интеграция с WhatsApp позволяет вашим агентам использовать эти возможности обмена сообщениями в рамках своих рабочих процессов. Это создаёт возможности для сложных сценариев взаимодействия с клиентами, таких как напоминания о встречах, коды подтверждения, оповещения и интерактивные беседы. Интеграция устраняет разрыв между вашими AI-рабочими процессами и каналами связи с клиентами, позволяя вашим агентам доставлять своевременную, актуальную информацию непосредственно на мобильные устройства пользователей. Подключив Sim к WhatsApp, вы можете создавать интеллектуальных агентов, которые взаимодействуют с клиентами через предпочитаемую ими платформу обмена сообщениями, улучшая пользовательский опыт и автоматизируя рутинные задачи по обмену сообщениями.

## Инструкции по использованию

Интегрируйте WhatsApp в рабочий процесс. Можно отправлять сообщения.



## Действия

### `whatsapp_send_message`

Отправка текстового сообщения через WhatsApp Cloud API.

#### Вход

| Параметр | Тип | Обязательный | Описание |
| --------- | ---- | -------- | ----------- |
| `phoneNumber` | string | Да | Номер телефона получателя с кодом страны \(например, +14155552671\) |
| `message` | string | Да | Текстовое содержимое сообщения для отправки |
| `phoneNumberId` | string | Да | Идентификатор номера телефона WhatsApp Business \(из Meta Business Suite\) |
| `previewUrl` | boolean | Нет | Должен ли WhatsApp пытаться отобразить предпросмотр ссылки для первого URL в сообщении |

#### Выход

| Параметр | Тип | Описание |
| --------- | ---- | ----------- |
| `success` | boolean | Статус успешности отправки сообщения WhatsApp |
| `messageId` | string | Уникальный идентификатор сообщения WhatsApp |
| `messageStatus` | string | Начальное состояние доставки, возвращённое API |
| `messagingProduct` | string | Продукт обмена сообщениями, возвращённый API |
| `inputPhoneNumber` | string | Номер телефона получателя, возвращённый WhatsApp |
| `whatsappUserId` | string | Идентификатор пользователя WhatsApp, определённый для получателя |
| `contacts` | array | Записи контактов получателя, возвращённые WhatsApp |
|   ↳ `input` | string | Входной номер телефона, отправленный в API |
|   ↳ `wa_id` | string | Идентификатор пользователя WhatsApp, связанный с получателем |



## Триггеры

**Триггер** — это блок, который запускает рабочий процесс при наступлении события в этом сервисе.

### WhatsApp Webhook

Запуск рабочего процесса от вебхуков входящих сообщений WhatsApp и статусов сообщений

#### Конфигурация

| Параметр | Тип | Обязательный | Описание |
| --------- | ---- | -------- | ----------- |
| `verificationToken` | string | Да | Введите любой безопасный токен здесь. Вы |
| `appSecret` | string | Да | Требуется для проверки подписи POST-запросов WhatsApp. Sim использует его для проверки заголовка X-Hub-Signature-256 при каждой доставке вебхука. |

#### Выход

| Параметр | Тип | Описание |
| --------- | ---- | ----------- |
| `eventType` | string | Классификация вебхука, например incoming_message, message_status или mixed |
| `messageId` | string | Первый идентификатор сообщения WhatsApp \(wamid\), найденный в пакете вебхука |
| `from` | string | Номер телефона отправителя из первого входящего сообщения в пакете |
| `recipientId` | string | Номер телефона получателя из первого обновления статуса в пакете |
| `phoneNumberId` | string | Идентификатор бизнес-номера телефона из первого сообщения или элемента статуса в пакете |
| `displayPhoneNumber` | string | Отображаемый бизнес-номер телефона из первого сообщения или элемента статуса в пакете |
| `text` | string | Текстовое содержимое из первого входящего текстового сообщения в пакете |
| `timestamp` | string | Временная метка из первого сообщения или элемента статуса в пакете |
| `messageType` | string | Тип первого входящего сообщения в пакете \(text, image, system и т.д.\) |
| `status` | string | Первый статус исходящего сообщения в пакете, например sent, delivered, read или failed |
| `contact` | json | Первый контакт отправителя в пакете \(wa_id, profile.name\) |
| `webhookContacts` | json | Все профили контактов отправителей из пакета вебхука |
| `messages` | json | Все объекты входящих сообщений из пакета вебхука, развёрнутые по entries/changes |
| `statuses` | json | Все объекты статусов сообщений из пакета вебхука, развёрнутые по entries/changes |
| `conversation` | json | Метаданные беседы из первого обновления статуса в пакете \(id, expiration_timestamp, origin.type\) |
| `pricing` | json | Метаданные ценообразования из первого обновления статуса в пакете \(billable, pricing_model, category\) |
| `raw` | json | Полная структурированная полезная нагрузка вебхука от WhatsApp |
