Повторение одних и тех же шагов над целой коллекцией — одна из самых фундаментальных идей в программировании. Sim предоставляет для этого два блока — блок **Loop** и блок **Parallel**. Они делают одну и ту же работу и отличаются только *тем, как* они её выполняют.

## Блок, содержащий блоки

Loop (или Parallel) — это **контейнер** — блок, внутрь которого вы помещаете другие блоки. Вы передаёте ему коллекцию, и он выполняет всё, что внутри, по одному разу для каждого элемента этой коллекции, превращая один шаг во множество.

## Что знает каждый запуск

Внутри контейнера каждая итерация получает **текущий элемент** для работы (и его индекс). Именно так одна дорожка блоков становится запуском на каждый лид, на каждую строку или на каждый файл — шаги остаются теми же, данные меняются с каждым проходом.

## По одному за раз или всё сразу

В этом вся разница между двумя блоками:

- **Loop** выполняет элементы **последовательно** — один завершается, прежде чем начинается следующий. Используйте его, когда каждый шаг строится на предыдущем, или когда нужно соблюдать порядок или ограничения частоты запросов.
- **Parallel** выполняет их **все сразу** — каждый элемент параллельно. Используйте его, когда элементы независимы; это значительно быстрее на большой коллекции.

## Замена контейнера

Поскольку оба блока имеют одинаковую форму, вы можете **заменить Loop на Parallel** (или обратно), не перестраивая ничего внутри. Та же логика, разное выполнение — последовательное или параллельное — выбирается тем, в какой контейнер вы её обернёте.
