## Развилка в потоке

Большинство воркфлоу, которые вы видели, выполняются по прямой линии. Ветвление добавляет **развилку**: в какой-то момент запуск смотрит на результат шага и идёт по одному или другому пути. Пути могут выполнять совершенно разную работу.

Вот форма развилки — один шаг решает, и запуск идёт по одному или другому пути:

## Блок Condition — решение с помощью правила

Блок **Condition** ветвится на основе **явного правила** — сравнения над выводом предыдущего блока, например, приоритет равен «high». Совпавшая ветвь выполняется; остальные нет. Это детерминированно и предсказуемо.

## Блок Router — решение моделью

Иногда правило размыто: «направляй вопросы по биллингу одним путём, всё остальное другим». Блок **Router** передаёт решение **модели** — она читает входные данные, выбирает ветвь, соответствующую намерению, и запуск продолжается по ней. Та же развилка, но решение принимается рассуждением, а не фиксированным правилом.

## Всё ещё один воркфлоу

Каким бы образом ни происходило ветвление, это **всё ещё один воркфлоу**, и каждый запуск записывается. Откройте логи, и вы сможете точно увидеть, какой путь выбрал конкретный запуск и почему.
