Model Context Protocol ([MCP](https://modelcontextprotocol.com/)) — это открытый стандарт для подключения ИИ к внешним инструментам и данным. Добавьте MCP-сервер в ваше рабочее пространство, и его инструменты станут доступны вашим агентам — способ интегрировать сервисы, для которых у Sim нет встроенной интеграции.

## Добавление MCP-сервера как инструмента

MCP-серверы предоставляют коллекции инструментов, которые могут использовать ваши агенты.

Чтобы добавить один:

1. Перейдите в **Settings → MCP Tools**

4. Добавьте любые необходимые **Headers** (например, API-ключи)
5. Нажмите **Add MCP**, чтобы сохранить

### Параметры настройки сервера

| Поле | Описание |
|-------|-------------|
| **Name** | Отображаемое имя сервера |
| **URL** | Конечная точка MCP-сервера |
| **Transport** | В настоящее время поддерживается `streamable-http` |
| **Headers** | Пары ключ-значение для аутентификации или пользовательских заголовков |
| **Timeout** | Таймаут соединения в миллисекундах (по умолчанию: 30 000) |

### Переменные окружения в настройке

URL серверов и заголовки поддерживают подстановку переменных окружения с использованием синтаксиса `{{VAR_NAME}}`. Это позволяет хранить чувствительные значения, такие как API-ключи, вне конфигурации сервера.

```
URL: https://api.example.com/mcp
Authorization: Bearer {{MCP_API_TOKEN}}
```

Когда вы вводите `{{` в полях URL или заголовков, появляется выпадающий список с доступными переменными окружения рабочего пространства.

### Тестирование и проверка

Нажмите **Test Connection** перед сохранением, чтобы убедиться, что сервер доступен, и обнаружить доступные инструменты. Ответ теста показывает количество найденных инструментов и версию протокола.

После сохранения каждый сервер отображает свои доступные инструменты с именами параметров, типами и флагами обязательности. Если инструменты сервера изменяются (например, после обновления сервера), нажмите **Refresh**, чтобы получить последние схемы. Это автоматически обновляет все блоки агентов, использующие эти инструменты.

Значки проверки инструментов появляются на серверах с проблемами — например, если инструмент был удалён с сервера, но всё ещё используется в воркфлоу. Нажмите на значок, чтобы увидеть, какие воркфлоу затронуты.

### Разрешение доменов (Domain Allowlisting)

Самостоятельные развёртывания могут ограничить, какие домены MCP-серверов разрешены, установив переменную окружения `ALLOWED_MCP_DOMAINS` (список, разделённый запятыми). Когда она установлена, можно добавлять только серверы на одобренных доменах. Когда не установлена, разрешены все домены.

## Использование MCP-инструментов в агентах

После настройки MCP-серверов их инструменты становятся доступны в ваших блоках агентов:

<div className="flex justify-center">
  <Image
    src="/static/blocks/mcp-agent-dropdown.png"
    alt="Использование MCP-инструмента в блоке Agent"
    width={700}
    height={450}
    className="my-6"
  />
</div>

1. Откройте блок **Agent**
2. В секции **Tools** нажмите **Add tool…**
3. В разделе **MCP Servers** нажмите на сервер, чтобы увидеть его инструменты

<div className="flex justify-center">
  <Image
    src="/static/blocks/mcp-agent-tools.png"
    alt="Список MCP-инструментов для выбранного сервера"
    width={400}
    height={400}
    className="my-6"
  />
</div>

4. Выберите отдельные инструменты или выберите **Use all N tools**, чтобы добавить все инструменты с этого сервера
5. Агент теперь может использовать эти инструменты во время выполнения

<Callout type="info">
Если вы ещё не настроили сервер, нажмите **Add MCP Server** в верхней части выпадающего списка, чтобы открыть модальное окно настройки, не покидая блок.
</Callout>

## Отдельный блок MCP Tool

Для более детального контроля вы можете использовать специальный блок MCP Tool для выполнения конкретных MCP-инструментов:

<div className="flex justify-center">
  <Image
    src="/static/blocks/mcp-tool-block.png"
    alt="Отдельный блок MCP Tool"
    width={700}
    height={450}
    className="my-6"
  />
</div>

Блок MCP Tool выполняет один настроенный инструмент с параметрами, которые вы задаёте явно, и его вывод доступен для чтения последующими блоками, как и любой другой.

## Когда использовать MCP Tool против агента

| | **Агент с MCP-инструментами** | **Блок MCP Tool** |
|---|---|---|
| **Выполнение** | ИИ решает, какие инструменты вызывать | Детерминированное — выполняет выбранный вами инструмент |
| **Параметры** | ИИ выбирает во время выполнения | Вы задаёте их явно |
| **Лучше всего для** | Динамических, диалоговых потоков | Структурированных, повторяемых шагов |
| **Рассуждение** | Обрабатывает сложную многошаговую логику | Один инструмент, один вызов |

## Требования к разрешениям

Функциональность MCP требует определённых разрешений рабочего пространства:

| Действие | Требуемое разрешение |
|--------|-------------------|
| Создание или обновление MCP-серверов | **Write** или **Admin** |
| Удаление MCP-серверов | **Admin** |
| Использование MCP-инструментов в агентах | **Write** или **Admin** |
| Просмотр доступных MCP-инструментов | **Read**, **Write** или **Admin** |
| Выполнение блоков MCP Tool | **Read**, **Write** или **Admin** |

<Callout type="warn">
MCP-серверы работают с разрешениями пользователя, который их настроил — подключайте только те серверы, которым вы доверяете, и проверяйте инструменты сервера перед тем, как передавать их агентам.
</Callout>
