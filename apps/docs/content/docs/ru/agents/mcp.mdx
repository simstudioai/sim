Протокол контекста модели ([MCP](https://modelcontextprotocol.com/)MCPhttps://modelcontextprotocol.com/) – это открытый стандарт для подключения ИИ к внешним инструментам и данным. Добавьте сервер MCP в свою рабочую среду, и его инструменты станут доступны вашим агентам – это способ интегрировать сервисы Sim, для которых нет встроенной интеграции.

Серверы MCP предоставляют наборы инструментов, которые ваши агенты могут использовать.

### Варианты конфигурации сервера
|  | Поле | Описание |  ||-------|-------------|
|  | **Название** | Имя сервера для отображения |  ||  | **URL** | Конечная точка сервера MCP |  ||  | **Транспорт** | В настоящее время поддерживает `streamable-http` |  ||  | **Заголовки** | Пары ключ-значение для аутентификации или пользовательских заголовков. |  ||  | **Превышение времени ожидания** | Время ожидания соединения в миллисекундах (по умолчанию: 30 000) |  |
### Переменные окружения в конфигурации
URL-ы и заголовки сервера поддерживают замену переменных окружения с использованием синтаксиса ``{{VAR_NAME}}``. Это позволяет хранить конфиденциальные значения, такие как ключи API, вне конфигурации сервера.

```
URL: https://api.example.com/mcp
Authorization: Bearer {{MCP_API_TOKEN}}
```

Когда вы вводите `{{` в адресной строке или полях заголовков, появляется раскрывающийся список с доступными переменными окружения рабочего пространства.

### Тестирование и проверка
Нажмите кнопку «Проверить подключение» перед сохранением, чтобы убедиться, что сервер доступен, и узнать, какие инструменты доступны. Результат теста показывает количество найденных инструментов и версию протокола.

После сохранения каждый сервер отображает доступные инструменты с указанием их параметров, типов и необходимых флагов. Если инструменты на сервере изменяются (например, после обновления сервера), нажмите кнопку «Обновить», чтобы получить последнюю версию схем. Это автоматически обновит любые блоки агентов, использующие эти инструменты.

Значки проверки инструментов отображаются на серверах с проблемами – например, если инструмент был удален с сервера, но все еще упоминается в рабочем процессе. Нажмите на значок, чтобы узнать, какие рабочие процессы затронуты.

### Разрешение доменов
Развертывание на собственном сервере может ограничивать список доменов серверов MCP, которые разрешены, путем установки переменной окружения `ALLOWED_MCP_DOMAINS` (список, разделенный запятыми). При установке разрешено добавлять только серверы из одобренных доменов. Если переменная не установлена, все домены разрешены.

## Использование инструментов MCP в агентах
После настройки серверов MCP, их инструменты становятся доступны в ваших блоках агентов.

<Изображение

высота = {450}
className="my-6"

1. Откройте блок "**Agent**"
2. В разделе «Инструменты» нажмите кнопку «Добавить инструмент…».
3. В разделе "**Серверы MCP**" нажмите на сервер, чтобы увидеть его инструменты.

<div className="flex justify-center">
<Изображение

4. Выберите отдельные инструменты или выберите опцию "**Использовать все N инструментов**", чтобы добавить каждый инструмент с этого сервера.
5. Теперь агент может использовать эти инструменты во время выполнения.

<Callout type="info">
If you haven't configured a server yet, click **Add MCP Server** at the top of the dropdown to open the setup modal without leaving the block.
</Callout>

Для более точного управления, вы можете использовать специализированный блок "MCP Tool" для выполнения конкретных задач MCP:

## Когда использовать инструмент MCP вместо агента?
|  |  | **Агент с использованием инструментов MCP** | Блок "Инструмент MCP" |  ||---|---|---|
|  | **Выполнение** | Искусственный интеллект определяет, какие инструменты использовать. | Определённый режим – позволяет использовать выбранный инструмент. |  ||  | **Параметры** | Искусственный интеллект выбирает в процессе выполнения. | Вы устанавливаете их вручную. |  ||  | **Идеально подходит для:** | Динамичные, ориентированные на диалог сценарии. | Четко определенные и повторяющиеся шаги |  ||  | **Обоснование** | Обрабатывает сложные многоступенчатые логические операции. | Одно решение, один телефонный звонок. |  |
## Требования к разрешениям
Функциональность MCP требует определенных разрешений для доступа к рабочему пространству:

|  | Действие | Необходимое разрешение |  ||--------|-------------------|
|  | Создать или обновить серверы MCP. | **Написать** или **Администрировать** |  ||  | Удалить серверы MCP | **Администратор** |  ||  | Используйте инструменты MCP в агентах. | **Написать** или **Администрировать** |  ||  | Просмотреть доступные инструменты для мониторинга и управления. | **Читать**, **писать** или **управлять** |  ||  | Выполните блоки инструмента MCP | **Читать**, **писать** или **управлять** |  |
