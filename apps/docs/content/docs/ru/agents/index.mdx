**Агент** – это…[рабочий процесс](/workflows)рабочий процесс/рабочие процессычтобы оно принимало решения и действовало самостоятельно. Оно получает входные данные, определяет, что нужно сделать, и выполняет это, используя инструменты и ваши данные. Вы создаете специализированного агента в Sim, составляя рабочий процесс, который включает в себя одну или несколько этапов.[Агент блокирует](/workflows/blocks/agent)Агент блокирует/рабочие процессы/блоки/агент.

**Блок "Агент"** – это этап рассуждения внутри этого рабочего процесса: модель считывает доступные ей значения, принимает решение и возвращает результат, который затем считывается другими блоками по ссылке. Простой агент – это один блок "Агент"; более сложный – это несколько блоков, соединенных вместе с другими блоками. Блок "Агент" является местом, где модель принимает решения; остальная часть рабочего процесса реализует эти решения.

## Блок «Агент»
Вы настраиваете этап рассуждения, предоставляя блоку "Agent" **модель** и **подсказку**. Модель – это большая языковая модель (LLM), которая его питает; вы выбираете одну из доступных провайдеров, а по умолчанию используется `claude-sonnet-4-6`. Подсказка – это системное сообщение, которое определяет, кто такой агент и как он должен себя вести, плюс пользовательское сообщение, содержащее вводные данные, обычно ссылку, например, `<start.input>`.

Когда он работает, блок "Agent" анализирует данные, вызывает необходимые инструменты и сохраняет результат под своим именем. По умолчанию этот результат представляет собой текстовое поле в `content`, которое затем будет прочитано другим блоком как `<agent.content>`, вместе с информацией о выполнении, такой как используемая модель, количество токенов, вызовы инструментов и стоимость. Все настройки и поля вывода находятся в[Ссылка на блок агента](/workflows/blocks/agent)Ссылка на блок агента/рабочие процессы/блоки/агент.

## Что вы предоставляете агенту
Самостоятельно блок "Агент" может только рассуждать и генерировать текст. Вы расширяете его функциональность, чтобы он мог действовать, следовать вашим правилам, использовать ваши данные, запоминать информацию и возвращать результаты, на которые могут опираться другие блоки. Каждая функция соответствует тому, что вы хотите, чтобы агент делал.

### Примите решение: инструменты
Чтобы агент мог что-то сделать в мире, ему нужны **инструменты**. Инструмент – это действие, которое агент может вызвать, например, отправить электронное письмо, выполнить поиск в интернете, обновить запись в CRM или запустить другой рабочий процесс. Вы прикрепляете инструменты к блоку "Агент", и агент решает, какой из них использовать для выполнения задачи. В инструменте "Lead scorer" у агента есть инструмент поиска для сбора информации о потенциальном клиенте и инструмент для отправки электронного письма для установления контакта.

Инструменты поступают из нескольких источников:

- **Интеграции** – это каталог сторонних сервисов: Gmail, Slack, Airtable, Linear и многие другие.- **[Настраиваемые инструменты](/agents/custom-tools)** – это инструменты, которые вы определяете один раз, используя схему и фрагмент кода, а затем используете повторно.- **Инструменты MCP** предоставляются сторонним поставщиком, с которым вы взаимодействуете через протокол Model Context.- **«Рабочий процесс как инструмент» (/workflows)** позволяет вызывать другой рабочий процесс, таким образом, агент выполняет всю процедуру в рамках одного шага.
<Callout type="info">
The same integration can run two ways. As a [block](/workflows#blocks) it runs because the path reached it. As an agent tool it runs because the agent chose it. Per-tool usage controls (force a tool, or disable one) are in the [Agent block reference](/workflows/blocks/agent).
</Callout>

### Следуйте определенному процессу: навыки
Чтобы предоставить агенту инструкции, которые он может выполнить, напишите…[навык](/agents/skills)навык/навыки агентов. Навык – это переиспользуемый шаблон с коротким названием и описанием, который агент всегда видит, а также более длинное описание, которое загружается только тогда, когда навык применяется. В примере "лидер", навык `lead-scoring-rubric` определяет критерии и исключения, а агент читает полное руководство только в случае неоднозначности ситуации. Инструмент – это действие, которое выполняет агент; навык – это инструкция, которую он читает.

### Используйте свои документы: знания
Чтобы агент мог отвечать от вашего имени, необходимо подключить[база знаний](/knowledgebase)база знаний/база знанийАгент ищет информацию и основывает свои ответы на том, что он находит, вместо того чтобы полагаться только на общие знания модели. Предоставьте ведущему агенту базу данных предыдущих сделок, и он сможет сравнить новый потенциальный клиент с теми, с которыми вы уже успешно работали.

### Не забывайте о памяти при переключении между рабочими процессами.
Чтобы агент мог использовать информацию из одной сессии для следующей, предоставьте ему...[память](/integrations/memory)память/интеграции/память, которая хранит и восстанавливает значения, связанные с конкретной беседой. Без нее каждое новое взаимодействие начинается с нуля; при ее использовании агент в чате может использовать информацию, полученную ранее, в последующих сообщениях.

### Вернуть полезный результат: структурированный вывод
Чтобы результат работы агента мог использоваться последующими блоками, необходимо предоставить ему структурированный вывод: объект определенного типа вместо произвольного текста. В примере с лидером по количеству голов агент возвращает ``{ score, tier, reason }``, а последующие...[Состояние](/workflows/blocks/condition)Состояние/рабочие процессы/блоки/условиеблок читает данные из `<agent.tier>`, чтобы перейти к следующему разделу. Смотрите[Как блоки передают данные](/workflows/data-flow)Как блоки передают данные/рабочие процессы/поток данныхдля чтения полей.

## Выбор того, что использовать
Начните с одного блока "Агент" и запроса, а затем добавляйте только то, что необходимо для выполнения задачи: инструмент, когда агент должен действовать, навык, когда требуется письменные инструкции, базу знаний, когда нужно отвечать на вопросы из ваших документов, память, когда нужно запоминать информацию, и структурированный вывод, когда последующий блок должен читать его результат.
