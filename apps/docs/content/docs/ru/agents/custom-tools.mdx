Пользовательские инструменты позволяют вам писать собственные функции на JavaScript и делать их доступными как вызываемые инструменты в блоках Agent. Это полезно, когда вам нужна функциональность, не покрываемая встроенными интеграциями Sim — например, вызов внутреннего API, выполнение пользовательского вычисления или преобразование данных специфическим образом.

## Как работают пользовательские инструменты

Пользовательский инструмент состоит из двух частей:

1. **Схема** — JSON-определение, описывающее имя, описание и параметры инструмента (в формате вызова функций OpenAI). Это сообщает ИИ-агенту, что делает инструмент и какие входные данные он ожидает.
2. **Код** — Тело функции JavaScript, которое выполняется, когда агент вызывает инструмент. Параметры, определённые в схеме, доступны как переменные в вашем коде.

Когда блок Agent имеет доступ к пользовательскому инструменту, ИИ-модель решает, когда его вызвать, на основе описания схемы и контекста разговора — так же, как и встроенные инструменты.

## Создание пользовательского инструмента

<Steps>
<Step>

### Откройте настройки пользовательских инструментов

Перейдите в **Settings → Custom Tools** в вашем рабочем пространстве и нажмите **Add**.

</Step>
<Step>

### Определите схему

На вкладке **Schema** определите ваш инструмент, используя JSON в формате вызова функций OpenAI:

```json
{
  "type": "function",
  "function": {
    "name": "get_weather",
    "description": "Получить текущую погоду для города",
    "parameters": {
      "type": "object",
      "properties": {
        "city": {
          "type": "string",
          "description": "Название города"
        },
        "units": {
          "type": "string",
          "enum": ["celsius", "fahrenheit"],
          "description": "Единицы измерения температуры"
        }
      },
      "required": ["city"]
    }
  }
}
```

<Callout type="info">
  Вы можете использовать кнопку AI-волшебной палочки, чтобы сгенерировать схему из описания на естественном языке того, что должен делать инструмент.
</Callout>

</Step>
<Step>

### Напишите код

Переключитесь на вкладку **Code** и напишите тело функции JavaScript. Параметры из вашей схемы доступны напрямую как переменные:

```javascript
const response = await fetch(
  `https://api.openweathermap.org/data/2.5/weather?q=${city}&units=${units === 'celsius' ? 'metric' : 'imperial'}&appid={{OPENWEATHER_API_KEY}}`
);

const data = await response.json();

return {
  temperature: data.main.temp,
  description: data.weather[0].description,
  humidity: data.main.humidity
};
```

<Callout type="info">
  Вы также можете использовать AI-волшебную палочку для генерации кода из описания. Переменные окружения указываются с помощью синтаксиса `&#123;&#123;KEY&#125;&#125;`.
</Callout>

</Step>
<Step>

### Сохраните

Нажмите **Save**, чтобы создать инструмент. Теперь он доступен для использования в любом блоке Agent в вашем рабочем пространстве.

</Step>
</Steps>

## Использование пользовательских инструментов в воркфлоу

После создания пользовательские инструменты отображаются вместе со встроенными инструментами при настройке блока Agent:

1. Откройте блок Agent
2. Нажмите **Add Tools**
3. Найдите ваш пользовательский инструмент в списке инструментов
4. Агент вызовет инструмент, когда определит, что он релевантен задаче

## Среда выполнения кода

### Доступные возможности

- **Async/await** — Ваш код выполняется в асинхронном контексте, так что вы можете использовать `await` напрямую
- **fetch()** — Выполняйте HTTP-запросы к внешним API
- **Встроенные модули Node.js** — Доступ к `crypto`, `Buffer` и другим стандартным модулям
- **Переменные окружения** — Используйте синтаксис `{{KEY}}` для внедрения секретов

### Ограничения

- **Нет npm-пакетов** — Внешние библиотеки, такие как `axios` или `lodash`, недоступны. Используйте встроенные API
- **Параметры по имени** — Параметры схемы доступны напрямую как переменные (например, `city`), а не через объект `params`

### Возврат результатов

Верните значение из вашего кода, чтобы отправить его обратно агенту:

```javascript
const result = await fetch(`https://api.example.com/data?q=${query}`);
const data = await result.json();
return data;
```

Возвращённое значение становится выводом инструмента, который агент видит и может использовать в своём ответе.

## Управление пользовательскими инструментами

В разделе **Settings → Custom Tools** вы можете:

- **Искать** инструменты по имени, имени функции или описанию
- **Редактировать** схему или код любого инструмента
- **Удалять** инструменты, которые больше не нужны

<Callout type="warn">
  Удаление пользовательского инструмента удаляет его из всех блоков Agent, которые на него ссылаются. Убедитесь, что ни один активный воркфлоу не зависит от инструмента перед удалением.
</Callout>

## Разрешения

| Действие | Требуемое разрешение |
|--------|-------------------|
| Просмотр пользовательских инструментов | **Read**, **Write** или **Admin** |
| Создание или редактирование инструментов | **Write** или **Admin** |
| Удаление инструментов | **Admin** |
