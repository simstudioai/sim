Навыки агентов (Agent Skills) — это переиспользуемые пакеты инструкций, которые дают вашим ИИ-агентам специализированные возможности. Основанные на открытом формате [Agent Skills](https://agentskills.io), навыки позволяют вам зафиксировать экспертизу в предметной области, воркфлоу и лучшие практики, которые агенты могут загружать по требованию.

## Как работают навыки

Навыки используют **прогрессивное раскрытие**, чтобы контекст агента оставался компактным:

1. **Обнаружение** — Только имена и описания навыков включаются в системный промпт агента (~50-100 токенов каждый)
2. **Активация** — Когда агент решает, что навык релевантен, он вызывает инструмент `load_skill` для загрузки полных инструкций в контекст
3. **Выполнение** — Агент следует загруженным инструкциям для выполнения задачи

## Создание навыков

Навыки находятся на странице **Integrations**: нажмите **Integrations** на боковой панели рабочего пространства, затем переключитесь на вкладку **Skills**. Она показывает все навыки в рабочем пространстве с возможностью поиска по имени.

![Вкладка Skills на странице Integrations](/static/skills/skills-tab.png)

Нажмите **+ Add to Sim**, чтобы открыть диалог **Add Skill**. Вкладка **Create** принимает три поля:

![Диалог Add Skill, вкладка Create](/static/skills/add-skill-create.png)

| Поле | Описание |
|-------|-------------|
| **Name** | Идентификатор в kebab-case (например, `sql-expert`, `code-reviewer`). Макс. 64 символа. |
| **Description** | Краткое объяснение того, что делает навык и когда его использовать. Это то, что агент читает, чтобы решить, активировать ли навык. Макс. 1024 символа. |
| **Content** | Полные инструкции навыка в markdown. Загружаются, когда агент активирует навык. |

<Callout type="info">
  Описание критически важно — это единственное, что агент видит перед тем, как решить загрузить навык. Будьте конкретны в отношении того, когда и почему навык следует использовать.
</Callout>

### Импорт навыков

Вкладка **Import** позволяет импортировать существующий навык в открытом формате [SKILL.md](https://agentskills.io/specification) тремя способами:

![Диалог Add Skill, вкладка Import](/static/skills/add-skill-import.png)

- **Загрузить файл** — файл `.md` с YAML-фронтматтером или `.zip`, содержащий `SKILL.md`.
- **Импорт из GitHub** — вставьте URL GitHub к `SKILL.md` и нажмите **Fetch**.
- **Вставить содержимое** — вставьте `SKILL.md` напрямую. Фронтматтер содержит `name` и `description`; тело markdown — это содержимое.

Страницы интеграций предлагают **курируемые навыки** для их сервиса — откройте одну (например, HubSpot) и добавьте предложенный навык одним кликом.

### Как писать хорошее содержимое навыков

Содержимое навыков следует тем же соглашениям, что и файлы [SKILL.md](https://agentskills.io/specification):

```markdown
# SQL Expert

## Когда использовать этот навык
Используйте, когда пользователь просит написать, оптимизировать или отладить SQL-запросы.

## Инструкции
1. Всегда спрашивайте, какой движок базы данных (PostgreSQL, MySQL, SQLite)
2. Используйте CTE вместо подзапросов для читаемости
3. Добавляйте рекомендации по индексам, где уместно
4. Объясняйте планы запросов для запросов на оптимизацию

## Типовые паттерны
...
```

**Рекомендуемая структура:**
- **Когда использовать** — Конкретные триггеры и сценарии
- **Инструкции** — Пошаговое руководство с нумерованными списками
- **Примеры** — Образцы вход/выход, показывающие ожидаемое поведение
- **Типовые паттерны** — Переиспользуемые подходы для частых задач
- **Крайние случаи** — Подводные камни и особые соображения

Делайте навыки сфокусированными и не более 500 строк. Если навык становится слишком большим, разделите его на несколько специализированных навыков.

## Добавление навыков к агенту

Откройте любой блок **Agent** и найдите выпадающий список **Skills** под секцией инструментов. Выберите навыки, к которым вы хотите дать агенту доступ.

![Добавление навыка](/static/skills/add-skill.png)

Выбранные навыки отображаются как карточки, на которые можно нажать для редактирования или удаления.

### Что происходит во время выполнения

Когда воркфлоу выполняется:

1. Системный промпт агента включает секцию `<available_skills>`, перечисляющую имя и описание каждого навыка
2. Инструмент `load_skill` автоматически добавляется в доступные инструменты агента
3. Когда агент определяет, что навык релевантен текущей задаче, он вызывает `load_skill` с именем навыка
4. Полное содержимое навыка возвращается как ответ инструмента, давая агенту подробные инструкции

Это работает со всеми поддерживаемыми LLM-провайдерами — инструмент `load_skill` использует стандартный вызов инструментов, поэтому специфичная для провайдера настройка не требуется.

## Типовые случаи использования

Навыки наиболее ценны, когда агентам нужны специализированные знания или многошаговые воркфлоу:

**Экспертиза в предметной области**
- `api-integration-expert` — Лучшие практики вызова конкретных API (аутентификация, ограничение частоты, обработка ошибок)
- `data-transformation` — ETL-паттерны, очистка данных и правила валидации
- `code-reviewer` — Руководства по ревью кода, специфичные для стандартов вашей команды

**Шаблоны воркфлоу**
- `bug-investigation` — Пошаговая методология отладки (воспроизвести → изолировать → протестировать → исправить)
- `feature-implementation` — Воркфлоу разработки от требований до развёртывания
- `document-generator` — Шаблоны и правила форматирования для технической документации

**Знания, специфичные для компании**
- `our-architecture` — Диаграммы архитектуры системы, зависимости сервисов и процессы развёртывания
- `style-guide` — Руководства по бренду, тон письма, UI/UX паттерны
- `customer-onboarding` — Стандартные процедуры и частые вопросы клиентов

**Когда использовать навыки против инструкций агента:**
- Используйте **навыки** для знаний, которые применяются в нескольких воркфлоу или часто меняются
- Используйте **инструкции агента** для специфичного для задачи контекста, уникального для одного агента

## Лучшие практики

**Написание эффективных описаний**
- **Будьте конкретны и насыщены ключевыми словами** — Вместо «Помогает с SQL», напишите «Пишет оптимизированные SQL-запросы для PostgreSQL, MySQL и SQLite, включая рекомендации по индексам и анализ плана запроса»
- **Включайте триггеры активации** — Упоминайте конкретные слова или фразы, которые должны вызвать навык (например, «Используйте, когда пользователь упоминает PDF, формы или извлечение документов»)
- **Держите не более 200 слов** — Агенты быстро сканируют описания; каждое слово должно быть весомым

**Объём и организация навыков**
- **Один навык на область** — Сфокусированный навык `sql-expert` работает лучше, чем широкий навык `database-everything`
- **Ограничьте 5-10 навыками на агента** — Больше навыков = больше накладных расходов на решение; начните с малого и добавляйте по мере необходимости
- **Разделяйте большие навыки** — Если навык превышает 500 строк, разбейте его на сфокусированные поднавыки

**Структура содержимого**
- **Используйте форматирование markdown** — Заголовки, списки и блоки кода помогают агентам разбирать и следовать инструкциям
- **Предоставляйте примеры** — Показывайте пары вход/выход, чтобы агенты понимали ожидаемое поведение
- **Будьте явны в отношении крайних случаев** — Не предполагайте, что агенты сами догадаются об особой обработке

**Тестирование и итерация**
- **Тестируйте активацию** — Запустите воркфлоу и проверьте, что агент загружает навык, когда ожидается
- **Проверяйте ложные срабатывания** — Убедитесь, что навыки не активируются, когда не должны
- **Уточняйте описания** — Если навык не загружается, когда нужно, добавьте больше ключевых слов в описание
