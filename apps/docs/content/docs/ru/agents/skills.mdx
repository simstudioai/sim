Навыки агента – это переиспользуемые наборы инструкций, которые предоставляют вашим ИИ-агентам специализированные возможности. Основаны на открытых[Навыки агента](https://agentskills.io)Навыки агентаhttps://agentskills.ioформат и возможности позволяют извлекать знания о предметной области, рабочие процессы и лучшие практики, которые могут быть использованы агентами по мере необходимости.

## Как работают навыки
Инструсы Skills используют принцип «постепенного раскрытия информации», чтобы минимизировать объем данных, доступных агенту.

1. **Обнаружение** – В системном запросе агента включаются только названия и описания навыков (примерно по 50–100 токенов каждый).
2. **Активация** — Когда агент определяет, что определенная функция полезна, он вызывает инструмент `load_skill`, чтобы загрузить полные инструкции в контекст.
3. **Выполнение:** Агент следует инструкциям, загруженным в систему, для выполнения задачи.

### Импорт навыков
Вкладка "**Импорт**" позволяет импортировать существующие навыки из открытого источника.[Навыки.md](https://agentskills.io/specification)Навыки.mdhttps://agentskills.io/спецификацияформат, три способа:

![Диалоговое окно «Добавить навык», вкладка «Импорт»](/static/skills/add-skill-import.png)Диалоговое окно «Добавить навык», вкладка «Импорт»/static/skills/добавить_навык_импорт.png

- **Загрузите файл** – это файл с YAML-метаданными или файл, содержащий `SKILL.md`.- **Импорт из GitHub** – вставьте URL репозитория GitHub в поле `SKILL.md` и нажмите кнопку «Получить».- **Вставьте содержимое** – вставьте текст непосредственно. Заголовок (frontmatter) содержит информацию о `name` и `description`, а основной текст на формате Markdown – это само содержимое.
Страницы интеграции предлагают **специализированные навыки**, которые можно добавить к своей услуге – откройте одну из них (например, HubSpot), и добавьте предложенный навык одним кликом.

### Создание качественного контента о навыках письма
Содержание навыков соответствует тем же правилам, что и...[Файлы SKILL.md](https://agentskills.io/specification)Файлы SKILL.mdhttps://agentskills.io/спецификация:

```markdown
# SQL Expert

## When to use this skill
Use when the user asks you to write, optimize, or debug SQL queries.

## Instructions
1. Always ask which database engine (PostgreSQL, MySQL, SQLite)
2. Use CTEs over subqueries for readability
3. Add index recommendations when relevant
4. Explain query plans for optimization requests

## Common Patterns
...
```

**Рекомендуемая структура:**
- **Когда использовать:** – Конкретные триггеры и сценарии- **Инструкция:** – Пошаговые рекомендации с использованием нумерованных списков.- **Примеры** – Образцы входных и выходных данных, демонстрирующие ожидаемое поведение.- **Распространенные шаблоны** – Переиспользуемые подходы для часто выполняемых задач.- **Случаи на краю** – Возможные проблемы и особые соображения
Сосредоточьтесь на конкретных навыках и ограничьте их размер до 500 строк. Если навык становится слишком большим, разделите его на несколько специализированных навыков.

## Добавление навыков для агента
Откройте любой блок "**Agent**" и найдите выпадающий список "**Skills**" под разделом "Инструменты". Выберите навыки, к которым должен иметь доступ агент.

![Добавить навык](/static/skills/add-skill.png)Добавить навык/static/skills/добавить_навык.png

Выбранные навыки отображаются в виде карточек, которые можно нажать для редактирования или удаления.

### Что происходит во время выполнения
Когда выполняется рабочий процесс:

1. В системе агента есть раздел `<available_skills>`, в котором перечислены названия и описания каждой из навыков.
2. Инструмент `load_skill` автоматически добавляется в список доступных инструментов агента.
3. Когда агент определяет, что определенный навык актуален для текущей задачи, он вызывает функцию `load_skill`, передавая ей имя навыка.
4. Полный набор навыков возвращается в виде ответа инструмента, предоставляя агенту подробные инструкции.

Это работает со всеми поддерживаемыми провайдерами больших языковых моделей – инструмент `load_skill` использует стандартный механизм вызова инструментов, поэтому не требуется настройка, специфичная для каждого провайдера.

## Типичные сценарии использования
Навыки наиболее ценны, когда агентам требуется специализированные знания или многоэтапные процессы.

**Специализированные знания**
- `api-integration-expert` – Лучшие практики для вызова конкретных API (аутентификация, ограничение скорости запросов, обработка ошибок)- `data-transformation` – шаблоны ETL, правила очистки и проверки данных.- `code-reviewer` – Руководство по проверке кода, разработанное с учетом стандартов вашей команды.
**Шаблоны рабочих процессов**
- `bug-investigation` – Метод отладки «шаг за шагом» (воспроизведение проблемы → выявление причины → тестирование → исправление)- `feature-implementation` – Процесс разработки от определения требований до развертывания.- `document-generator` – Шаблоны и правила форматирования для технической документации
Знания, специфичные для компании
- `our-architecture` – Схемы архитектуры системы, зависимости между сервисами и процессы развертывания.- `style-guide` – Руководство по бренду, стиль написания, шаблоны пользовательского интерфейса и UX.- `customer-onboarding` – Стандартные процедуры и часто задаваемые вопросы клиентов.
**Когда использовать навыки, а когда – инструкции для агента?**
- Используйте **навыки** для информации, которая применима в различных рабочих процессах или часто меняется.- Используйте **инструкции для агентов**, чтобы предоставить конкретный контекст, который является уникальным для каждого отдельного агента.
## Лучшие практики
**Написание эффективных описаний**
- **Будьте конкретны и используйте ключевые слова** – Вместо "Помогает с SQL", напишите: "Написание оптимизированных запросов на SQL для PostgreSQL, MySQL и SQLite, включая рекомендации по индексам и анализ плана выполнения запроса".- **Укажите триггеры активации** – Перечислите конкретные слова или фразы, которые должны вызывать работу навыка (например, «Использовать, когда пользователь упоминает PDF-файлы, формы или извлечение документов»).- **Ограничьте объем до 200 слов** – Агенты быстро анализируют описания; используйте каждое слово эффективно.
**Область и организация навыков**
- **Один навык в каждой области** – Специализированный навык работает лучше, чем широкий.- **Ограничьте количество навыков для каждого сотрудника до 5-10 – Больше навыков означает больше нагрузки на принятие решений. Начните с небольшого количества и добавляйте по мере необходимости.**- **Разделите сложные навыки** – Если навык состоит из более чем 500 строк кода, разбейте его на отдельные, более простые поднавыки.
**Структура контента**
- **Используйте форматирование Markdown:** Заголовки, списки и блоки кода помогают агентам понимать и следовать инструкциям.- **Приведите примеры** – Покажите пары входных и выходных данных, чтобы агенты понимали ожидаемое поведение.- **Четко указывайте на крайние случаи** – Не предполагайте, что агенты сами определят необходимость специальной обработки.
**Тестирование и итерация**
- **Активация теста** – Запустите свой рабочий процесс и убедитесь, что агент загружает необходимый навык в ожидаемое время.- **Проверьте на наличие ложных срабатываний** – Убедитесь, что навыки не активируются в неправильных ситуациях.- **Уточните описания** – Если навык не отображается при необходимости, добавьте больше ключевых слов в описание.
